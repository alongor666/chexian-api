# 驾意险推介率可视化页面开发总结

## 需求概述

用户要求基于 `车险保单综合明细表0212.parquet` 数据文件，开发一个名为"驾意险推介率.html"的可视化网页，展示车驾意推介率分析功能。

### 核心需求
1. **第一层**：四川分公司汇总（单行数据）
2. **下钻维度**：用户可自由选择（不固定顺序）
   - 三级机构、销售团队、业务员、客户类别
   - 是否新车、是否过户、是否新能源、是否电销、是否续保
3. **展示字段**：
   - 车险件数、驾意件数
   - 单交/交三/主全的车险件数和驾意件数
   - 各分类推介率（驾意件数/车险件数）

### 计算逻辑
- 车险件数 = COUNT(DISTINCT policy_no)
- 驾意件数 = COUNT(DISTINCT CASE WHEN is_cross_sell = true THEN policy_no END)
- 推介率 = 驾意件数 / 车险件数（按险别组合分别计算）

## 技术方案探索

### 方案一：调用后端API
**尝试**：使用已有的 `/api/query/cross-sell` 接口
**问题**：
- API需要JWT认证（localStorage中的token）
- 用户希望直接读取Parquet文件，不依赖认证

### 方案二：DuckDB-WASM浏览器端处理
**尝试**：在浏览器中使用DuckDB-WASM直接读取Parquet文件
**技术栈**：
- Vue 3 + CDN引入
- DuckDB-WASM (@duckdb/duckdb-wasm@1.28.1)
- ES Module方式加载

**遇到的问题**：
1. DuckDB-WASM无法直接读取本地文件路径
2. 跨域请求（CORS）限制
3. Worker加载需要正确的配置

## 关键代码片段

### SQL查询逻辑
```sql
-- 汇总查询
SELECT 
    COUNT(DISTINCT 保单号) as total_policy_count,
    COUNT(DISTINCT CASE WHEN 交叉销售标识 = true THEN 保单号 END) as total_driver_count,
    COUNT(DISTINCT CASE WHEN 险别组合 = '单交' THEN 保单号 END) as danjiao_policy_count,
    COUNT(DISTINCT CASE WHEN 险别组合 = '单交' AND 交叉销售标识 = true THEN 保单号 END) as danjiao_driver_count,
    -- 交三、主全同理
FROM policy_data
WHERE 签单日期 >= DATE '2025-12-01'
GROUP BY dimension_field
```

### DuckDB-WASM初始化
```javascript
import * as duckdb from 'https://cdn.jsdelivr.net/npm/@duckdb/duckdb-wasm@1.28.1/+esm';

const bundle = {
    mainWorker: new Worker('.../duckdb-browser-mvp.worker.js'),
    main: '.../duckdb-browser-mvp.wasm'
};

const db = new duckdb.AsyncDuckDB(logger, bundle.mainWorker);
await db.instantiate(bundle.main);
```

## 结论

由于浏览器安全策略限制（跨域、CORS），DuckDB-WASM无法直接通过HTTP URL读取Parquet文件。最终方案需要：

1. **推荐方案**：使用后端API（需要先登录获取token）
2. **备选方案**：使用Python/FastAPI搭建本地服务，绕过CORS
3. **生产方案**：将Parquet转为DuckDB数据库文件，使用WASM直接加载
