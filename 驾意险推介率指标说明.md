# 驾意险推介率板块指标说明

本文档对“驾意险推介率”板块所涉及的核心分析指标、计算公式、底层因子（参数）的意义及数据范围进行了系统说明。所有计算均基于后端 SQL 引擎直接在数据湖中（如 DuckDB 聚合后）进行统计。

---

## 1. 核心计算因子（参数）说明

这些因子是各项推介率、件均指标计算的基础分子与分母。

| 因子字段              | 中文含义         | 因子意义                                                                                                               | 数据范围                                       |
| :-------------------- | :--------------- | :--------------------------------------------------------------------------------------------------------------------- | :--------------------------------------------- |
| `auto_count`          | 车险承保车辆数   | 指代一定维度下，购买了商业险或交强险的**去重车架号**数量。是计算推介率的**分母**，解决一车多单导致的分子虚高问题。                               | `≥ 0` 的整数                                   |
| `driver_count`        | 驾意险承保车辆数 | 在 `auto_count` 的基础上，同时购买了驾意险的**去重车架号**数量。是计算推介率的**分子**。                                           | `≥ 0` 的整数，通常 `driver_count ≤ auto_count` |
| `driver_policy_count` | 驾意险保单件数   | 该维度下所产生的所有驾意险**保单件数**。用于计算驾意险件均保费的**分母**。                                                       | `≥ 0` 的整数                                   |
| `driver_premium`      | 驾意险保费       | 该维度下所产生的所有驾意险保费总金额。用于计算件均保费与计划达成率的**分子**。                                                       | `≥ 0.00` 的金额数值（单位：元）                |
| `auto_premium`        | 车险保费         | 该维度下所产生的所有商业险与交强险保费总金额。                                                                         | `≥ 0.00` 的金额数值（单位：元）                |
| `coverage_combination`| 险别组合         | 代表车主的车险购买形式，作为分类维度进行交叉下钻。                                                                     | 包含：`'整体'`、`'主全'`、`'交三'`、`'单交'`   |
| `plan_premium_wan`    | KPI 计划保费     | 配置的机构或业务员驾意险销售目标（万元）。在动态周期计算时会按天/周/月折算。                                             | `≥ 0` 的金额数值（单位：万元）                 |

## 2. 核心指标（Metrics）与计算公式

### 2.1 推介率 (Recommendation Rate, `rate`)

- **计算公式**：
  ```sql
  CASE 
    WHEN auto_count = 0 THEN 0
    ELSE ROUND(driver_count * 100.0 / auto_count, 2)
  END
  ```
- **指标意义**：
  办理了基础车险（交强险/商业险）的客户中，有多少比例同时被营销并购买了驾意险。该指标直观反映了业务员、团队或机构的“交叉销售”推荐转化能力。基于车架号去重计算，能更科学准确地呈现渗透率。
- **数据范围**：
  正常情况下为 `0.00% ~ 100.00%`。

### 2.2 驾意件均保费 (Average Driver Premium, `avg_premium`)

- **计算公式**：
  ```sql
  CASE 
    WHEN driver_policy_count = 0 THEN 0
    ELSE ROUND(driver_premium / driver_policy_count, 2)
  END
  ```
- **指标意义**：
  平均每**笔**（件）成功购买的驾意险保单所支付的金额。**注意**：这里分母必须是驾意险保单的绝对件数 (`driver_policy_count`)，而不是车架号去重的数量。即便在“车上责任不分保额”的情况下，驾意险件均依然遵循这一保单统计底层逻辑。
- **数据范围**：
  `≥ 0.00`（常见值为几百元至一两千元人民币）。

### 2.3 车险件均保费 (Auto Average Premium, `auto_avg_premium`)

- **计算公式**：
  ```sql
  CASE 
    WHEN auto_count = 0 THEN 0
    ELSE ROUND(auto_premium / auto_count, 2)
  END
  ```
- **指标意义**：
  平均每辆车所贡献的基础车险保费金额。
- **数据范围**：
  `≥ 0.00` 的金额数值。

### 2.4 计划达成率 (Plan Achievement Rate, `achievement_rate`)

- **计算公式**：
  ```sql
  CASE
    WHEN plan_premium_wan <= 0 THEN NULL
    ELSE ROUND( (driver_premium / 10000.0) / (plan_premium_wan / denominator) * 100.0, 2 )
  END
  ```
  > *注：`denominator` 根据不同时间粒度动态换算目标：日=`365`、周=`52`、月=`12`、季=`4`、年=`1`。*
- **指标意义**：
  特定时间段内，实际贡献的驾意险保费占**折算后的时段目标KPI**的比例。常用于热力图视图，评估业务进度是否在“红线”或超越目标。
- **数据范围**：
  `0.00% ~ 无穷大`（超过100%代表超额完成该阶段目标）。

## 3. 分析维度与口径

本板块不仅计算上述整体指标，还在后端 SQL 实现了多层多维度的交叉切片，计算口径完全一致，主要的切片维度包含：

1. **时间维度**：日、自然周、自然月、季度、当年至今（包含环比 `prev_` 系列同期对比计算）。
2. **险别组合切片**：针对 "整体", "主全", "交三", "单交" 4个组合分别独立计算四套指标。
3. **车辆类别**：客车、货车、摩托车等过滤。
4. **组织架构层级下钻**：从分公司 -> 三级机构 -> 销售团队 -> 业务员 层层下钻分析推介率。
5. **其它业务属性**：新/旧车、过户车、新能源、电销、续保等状态标识。
