# 会话管理器使用指南

**版本**: 2.0.0 | **更新**: 2025-01-10

管理 Claude Code CLI 的对话历史：查看、搜索、重命名、导出会话。

## 🚀 快速开始

```bash
# 1. 查看所有会话
bun run scripts/session-manager.mjs --list

# 2. 搜索会话
bun run scripts/session-manager.mjs --search "KPI"

# 3. 重命名会话（使用前8位UUID）
bun run scripts/session-manager.mjs --rename 4f9fa1cf --summary "新标题"

# 4. 导出会话
bun run scripts/session-manager.mjs --export 4f9fa1cf --output "会话.md"
```

## 📋 完整命令参考

| 命令 | 说明 | 示例 |
|------|------|------|
| `--locate` | 查找会话目录 | `session-manager --locate` |
| `--list` | 列出所有会话 | `session-manager --list --page 2` |
| `--detail <id>` | 查看会话详情 | `session-manager --detail 4f9fa1cf` |
| `--search <keyword>` | 搜索会话 | `session-manager --search "KPI"` |
| `--rename <id>` | 重命名会话 | `session-manager --rename 4f9fa1cf --summary "新标题"` |
| `--delete <id>` | 删除会话 | `session-manager --delete 4f9fa1cf --force` |
| `--export <id>` | 导出会话 | `session-manager --export 4f9fa1cf --output "会话.md"` |
| `--stats` | 显示统计 | `session-manager --stats` |
| `--help` | 显示帮助 | `session-manager --help` |

**选项参数**：
- `--page <num>` - 分页页码（默认: 1）
- `--format <fmt>` - 导出格式：markdown/json/jsonl（默认: markdown）
- `--output <path>` - 输出文件路径
- `--project <name>` - 按项目名称筛选
- `--date-from <date>` - 起始日期（YYYY-MM-DD）
- `--date-to <date>` - 结束日期（YYYY-MM-DD）
- `--min-messages <num>` - 最小消息数量
- `--force` - 跳过确认提示

## 💡 典型使用场景

### 场景1：查找昨天的讨论

```bash
# 搜索昨天的会话
bun run scripts/session-manager.mjs --search "自然周" \
  --date-from "2025-01-09" --date-to "2025-01-09"

# 查看详情（使用前8位UUID）
bun run scripts/session-manager.mjs --detail <前8位>

# 导出备份
bun run scripts/session-manager.mjs --export <前8位> --output "讨论.md"
```

### 场景2：批量重命名会话

```bash
# 列出所有会话
bun run scripts/session-manager.mjs --list

# 重命名（添加项目前缀）
bun run scripts/session-manager.mjs --rename 4f9fa1cf \
  --summary "车险KPI_数据分析"
```

### 场景3：清理旧会话

```bash
# 查看30天前的会话
bun run scripts/session-manager.mjs --date-to "2024-12-11" --list

# 导出重要会话
bun run scripts/session-manager.mjs --export <id> --output "备份.md"

# 删除不重要会话
bun run scripts/session-manager.mjs --delete <id> --force
```

### 场景4：导出原始JSONL格式

```bash
# 导出为JSONL（与Claude Code兼容）
bun run scripts/session-manager.mjs --export 4f9fa1cf \
  --format jsonl --output "session.jsonl"
```

## 🔧 重要说明

### 重命名规则（v2.0）

**✅ 正确方式**：
```bash
session-manager --rename 4f9fa1cf --summary "新标题"
```

- 使用 `--summary` 参数（符合官方规范）
- 修改第一行的 `summary` 字段
- 文件名（UUID）保持不变

**❌ 错误方式**（v1.0）：
```bash
session-manager --rename 4f9fa1cf --title "新标题"  # 不再支持
```

### 会话ID格式

**完整UUID**（36字符）：
```
4f9fa1cf-6cff-44ac-bbd4-4413201e4402
```

**简化输入**（前8位）：
```
4f9fa1cf
```

两种方式都可以工作！

### 文件格式（JSONL）

```jsonl
{"type": "summary", "summary": "会话标题", "leafUuid": "..."}
{"type": "user", "uuid": "...", "timestamp": "...", "message": {...}}
{"type": "assistant", "uuid": "...", "timestamp": "...", "message": {...}}
```

- 每行一个独立的JSON对象
- 第一行是 `summary`（标题）
- 后续行是消息内容

### 存储位置

```
~/.claude/projects/[normalized-project-path]/[uuid].jsonl
```

**路径规范化**：
- `/Users/xxx/project` → `-Users-xxx-project`
- 自动处理，无需手动转换

## 🐛 故障排除

### 问题1：找不到会话目录

**症状**：`--locate` 显示目录不存在

**原因**：
1. 从未使用 Claude Code CLI 创建过会话
2. 会话存储在其他位置

**解决方案**：
```bash
# 检查正确的位置
ls -la ~/.claude/projects/

# 如果不存在，先使用 Claude Code CLI 创建一些会话
```

### 问题2：无法读取会话文件

**症状**：显示 "读取会话文件失败"

**解决方案**：
```bash
# 检查文件权限
ls -l ~/.claude/projects/*/

# 确认文件是 JSONL 格式
head -1 ~/.claude/projects/*/[uuid].jsonl
```

### 问题3：导出的文件乱码

**症状**：Markdown 或 JSON 文件包含乱码

**解决方案**：
1. 使用支持 UTF-8 的编辑器打开
2. 检查原始 JSONL 文件是否包含特殊字符

## 📊 技术细节

### JSONL vs JSON

**为什么使用 JSONL？**
1. **流式处理**：可以逐行读取，不需要一次性加载整个文件
2. **性能**：大型会话文件处理更高效
3. **可靠性**：单行损坏不影响其他行

### 会话文件结构

```javascript
{
  type: "summary",           // 第一行
  summary: "人类可读的标题",
  leafUuid: "最后消息的UUID"
}
{
  type: "user",              // 用户消息
  uuid: "...",
  timestamp: "2025-01-10T...",
  message: {
    role: "user",
    content: "用户输入"
  }
}
{
  type: "assistant",         // 助手回复
  uuid: "...",
  timestamp: "2025-01-10T...",
  message: {
    model: "claude-sonnet-4-5",
    role: "assistant",
    content: [
      { type: "text", text: "回复内容" },
      { type: "tool_use", id: "...", name: "Bash", input: {...} }
    ]
  }
}
```

### 重命名实现原理

```javascript
// 1. 读取 JSONL 文件
const entries = readJSONL(filePath);

// 2. 找到 summary 条目
const summaryEntry = entries.find(e => e.type === 'summary');

// 3. 修改 summary 字段
summaryEntry.summary = "新标题";

// 4. 重新写入文件
writeJSONL(filePath, entries);
```

**关键点**：
- ✅ 文件名（UUID）不变
- ✅ 只修改第一行的 `summary` 字段
- ✅ 保持 JSONL 格式

## 📚 变更日志

### v2.0.0 (2025-01-10)

**重大变更**：
- ✅ 支持 **JSONL 格式**（而非 JSON）
- ✅ 正确的存储位置：`~/.claude/projects/`
- ✅ 重命名使用 `--summary` 参数（而非 `--title`）
- ✅ 会话 ID 是 UUID 格式
- ✅ 支持项目筛选和列表

**主要修正**：
- 文件格式：JSON → **JSONL**
- 存储位置：`~/Library/Application Support/...` → **`~/.claude/projects/`**
- 重命名参数：`--title` → **`--summary`**
- 会话ID：`session-YYYYMMDD-*` → **UUID格式**
