# 数据管道全面重构 — 8 域分域架构

## Context

数据源从"一张大表嵌入所有字段"切换为"8 个独立清单各司其职"。同时赔付、交叉销售字段从签单清单中移出，需要从旧的"大表查一切"架构迁移到"分域查询 + 按需 JOIN"架构，实现：
- VPS 内存减半（每个查询只扫描所需域）
- 交叉销售/赔付查询提速 10-50x
- 新增 4 个业务域（报价/续保/维修/客户来源去向）

## Phase 2: DuckDB 分域加载

### 2.1 PolicyFact（瘦身版）

不再包含 `claim_cases`、`reported_claims`、`is_cross_sell`、`cross_sell_premium_driver`。
仍包含 `fee_amount`（费用金额留在签单清单）。

```sql
-- policy/current/ 的多分片加载（21-23 + 剔摩24+ + 限摩24+）
CREATE TABLE raw_parquet AS
SELECT * FROM read_parquet([分片列表], union_by_name=true)
```

### 2.2 ClaimsDetail VIEW（已有，不变）

```sql
CREATE VIEW ClaimsDetail AS SELECT * FROM read_parquet('claims_detail/latest.parquet')
```

### 2.3 ClaimsAgg TABLE（新增 — 保单级赔付聚合）

```sql
CREATE TABLE ClaimsAgg AS
SELECT policy_no,
       COUNT(DISTINCT claim_no) AS claim_cases,
       SUM(COALESCE(settled_amount,0) + COALESCE(pending_amount,0)) AS reported_claims
FROM ClaimsDetail
GROUP BY policy_no
```

供 cost.ts 赔付率计算使用。在 `loadClaimsDetail()` 末尾自动创建。

### 2.4 CrossSellFact VIEW（新增 — 替代内嵌字段）

```sql
CREATE VIEW CrossSellFact AS SELECT * FROM read_parquet('cross_sell/latest.parquet')
```

### 2.5 CrossSellDailyAgg TABLE（重构 — 从 CrossSellFact 构建）

当前从 PolicyFact 构建。改为从 CrossSellFact 构建（数据量从 380 万→几十万，构建速度提升 10x）。

## Phase 3: SQL 生成器适配（7 个文件）

### 3.1 cost.ts — 赔付率从 ClaimsAgg JOIN

**当前**：`SUM(reported_claims) FROM PolicyFact`
**改为**：
```sql
SELECT p.*, COALESCE(c.claim_cases, 0), COALESCE(c.reported_claims, 0)
FROM PolicyFact p
LEFT JOIN ClaimsAgg c ON p.policy_no = c.policy_no
WHERE ...
```

影响函数：`generateClaimRatioQuery()`、`generateComprehensiveCostQuery()`、`generateVariableCostQuery()`

### 3.2 cross-sell.ts + 5 个衍生 — 从 CrossSellFact/CrossSellDailyAgg 查询

**当前**：从 PolicyFact 读取 `is_cross_sell`
**改为**：从 CrossSellDailyAgg（已重构为从 CrossSellFact 构建）

影响文件：
- `server/src/sql/cross-sell.ts`
- `server/src/sql/cross-sell-summary.ts`
- `server/src/sql/cross-sell-trend.ts`
- `server/src/sql/cross-sell-org-trend.ts`
- `server/src/sql/cross-sell-heatmap.ts`
- `server/src/sql/cross-sell-top-salesman.ts`

**注意**：如果 CrossSellDailyAgg 的 schema 不变（仍是 auto_count/driver_count/driver_premium），则前端和大部分 SQL 生成器**不需要改**。只需改 `createCrossSellRealtimeView()` 的数据源从 PolicyFact → CrossSellFact。

## Phase 4: 新域前端

### 4.1 维修资源子页面

- 新增路由 `/repair` 或 `/specialty?tab=repair`
- 查看维修厂合作状态、核损金额、换件折扣
- 可按机构/4S店/区域筛选
- 需新增 API 端点 + SQL 生成器

### 4.2 客户来源去向分析页面

- 新增路由 `/customer-flow`
- 核心分析维度：
  - 转入分析：`上年承保主体 IS NOT NULL AND 上年承保主体 != '华安'` → 从哪家公司转入
  - 流失分析：`次年保险公司 IS NOT NULL AND 次年保险公司 != '华安'` → 流向哪家公司
  - 时间特征：按月份分析流失集中度
  - 归因分析：JOIN PolicyFact 获取机构/业务员/风险评分/NCD 系数
- 需新增 API 端点 + SQL 生成器 + 前端页面

### 4.3 厂牌维度激活

- duckdb.ts 加载 BrandDim
- 可在成本分析/KPI 等页面增加"按品牌"维度
- 优先级低，可后续迭代
