# 系统提示词问题反思

> 创建日期：2026-01-31
> 触发场景：删除 AI 洞察与 AI 仪表盘功能时的反思

## 问题清单

### 1. 功能模块数量过多导致 Token 浪费

**问题**：CLAUDE.md 中列出了 13 个功能模块，包括已废弃的 AI Dashboard 和 AI Insights，导致：
- 每次对话都要加载这些无用信息
- Claude 需要理解和记忆这些模块的用途
- 增加了上下文窗口的压力

**建议**：
- 定期清理已废弃功能的文档记录
- 建立功能模块的生命周期管理（活跃/维护/废弃）

### 2. 命令索引过于庞大

**问题**：`.claude/commands/README.md` 索引了 23 个命令，但：
- 很多命令从未被使用
- 命令间有功能重叠
- 维护成本高

**建议**：
- 建立命令使用频率统计
- 合并功能相近的命令
- 移除长期未使用的命令

### 3. 智能加载协议未被有效执行

**问题**：CLAUDE.md 定义了"智能加载协议"（简单任务只读 §1-2），但：
- 实际对话中 Claude 经常全量加载
- 没有强制执行机制
- 任务复杂度判断主观性强

**建议**：
- 简化 CLAUDE.md 结构，减少章节数量
- 将详细信息移到独立文件，按需加载
- 建立更清晰的任务分类标准

### 4. AI 功能开发过度

**问题**：AI 相关功能（AI Dashboard、AI Insights、Query Assistant）占用了大量开发资源，但：
- 性能极差（外部 API 调用延迟高）
- 实际业务价值低（用户几乎不使用）
- 增加了代码复杂度和维护成本

**教训**：
- 新功能开发前应进行用户需求验证
- AI 功能应先做 MVP 验证，再决定是否全面开发
- 避免为了"技术先进性"而忽视实用性

### 5. 文档与代码同步滞后

**问题**：删除 AI 功能后，发现多处文档未同步更新：
- `features/INDEX.md` 仍有 AI Dashboard 描述
- 多个 Page 组件仍引用已删除模块
- 导致清理工作量增加

**建议**：
- 建立文档更新检查清单
- 代码删除时强制要求同步更新文档
- 使用脚本自动检测文档与代码的一致性

### 6. 模块耦合过深

**问题**：删除 `shared/ai` 目录后，需要级联删除：
- `features/ai-insights` 整个目录
- `features/ai-dashboard` 整个目录
- `features/query-assistant` 整个目录
- 8 个 Page 组件中的相关代码

**教训**：
- 实验性功能应与核心模块松耦合
- 使用特性开关控制功能启用/禁用
- 避免将实验性功能深度集成到页面组件

## 已删除的 AI 相关模块

| 模块 | 路径 | 删除原因 |
|------|------|----------|
| AI Insights | `src/features/ai-insights/` | 性能差，无实际业务价值 |
| AI Dashboard | `src/features/ai-dashboard/` | 性能差，无实际业务价值 |
| Query Assistant | `src/features/query-assistant/` | 依赖 AI 模块，功能复杂但使用率低 |
| Shared AI | `src/shared/ai/` | AI Provider 和约束系统，已无使用场景 |
