## 2. 现状与「要替代的」（#753，已合并 commit 6858ee8d）

`branch_code` 现状有两块，本方案分别替代：

| # | 现状（脆弱点） | 替代为 |
|---|---------------|--------|
| 2.1 | **`branch_code` = ETL 常量标签**：`fields.json:550-562` derivation `type=constant` + `envVar=BRANCH_CODE`（默认 `SC`）；`transform.py` 按 env 贴；`daily.mjs` 按 `BRANCH_CODE` + `branchSourceDir(staging/<省>)` 路由。**文件喂错省 → 整列静默贴错，测不出混省。** | **Phase A**：从 `policy_no` 第 2-3 位派生 + 自校验 |
| 2.2 | **sync 分省 = 扁平目录 + 省份前缀 + rsync filter**：`current/` 扁平，SC 裸名 / SX `SX_` 前缀，`sync-vps.mjs` 用 `R/P` filter + `--exclude` 区分省。**靠文件名前缀做隔离，~6 处代码假设扁平目录，filter 语义易错（本次踩过删四川数据 P0）。** | **Phase B**：分省子目录 `current/<省>/`，每省独立 sync |

> #753 的前缀方案已经过独立 codex 把关、单省字节安全、当前休眠（山西数据未进生产）。**它不是有 bug 要修，而是「未来 cutover 前要不要重构得更稳健」的选择。** 本方案是这个选择的执行规划。

## 3. 方案对比与选定

| 方案 | 内容 | 爆炸半径 | 收益 |
|------|------|---------|------|
| **A 派生化**（选定·先做） | `branch_code` 从保单号派生 + ETL 自校验 | **小**（只改 ETL 派生层，**不触碰任何扁平目录站点/RLS/loader**） | 防贴错、检测混省、自校验；替代 2.1 |
| **B 分省子目录**（选定·后置立项） | `current/<省>/` 子目录 + 每省独立 sync，退役 #753 前缀+filter | **大**（改 6 站点 + 2 装载依赖，有「生死点」） | 根治 sync 复杂度、物理删不错省；替代 2.2 |
| C 维持 #753 前缀 | 不动 | 0 | 但留下「派生 + 前缀」两套省份机制并存 |

**选定渐进式 A → B**：A 独立先落地（替代最脆弱的常量标签，且让 B 的省份切分变可靠）；B 后置，需用户确认是否值得为根治 sync 复杂度付大爆炸半径（否则 #753 前缀方案已 codex 验证、可继续用，但需在文档标注「branch_code 已派生化，前缀仅作 sync 文件区分」避免两套机制语义冲突）。

## 4. 分阶段单任务（供新会话按 Loop v2 逐个 evidence-loop 执行）

### Phase A — branch_code 派生化 + 自校验（低爆炸半径，先做）

**A1 · 省份码映射表（单一事实源）**
- 范围：新建 `数据管理/config/province-code-map.json`（或并入现有 config）：`{"10":{"branch_code":"SC","name":"四川分公司"},"18":{"branch_code":"SX","name":"山西分公司"}}` + 「新增省份 SOP」。派生逻辑与未来加省都从此读。
- oracle：映射覆盖现有 10/18；governance 不破。
- 依赖：无（先做）。

**A2 · transform.py 派生化 + 自校验**
- 范围：`数据管理/pipelines/transform.py` 把 `branch_code` 从 constant 改为「`substring(policy_no,2,2)` → 查 A1 映射 → branch_code」；**保留 `BRANCH_CODE` env 作为「期望值」做自校验断言**：派生值 ≠ env 标签 → `sys.exit(1)` 报错（抓 ETL 喂错省/混省）。**NULL/异常前缀必须 fail-fast**（rls_loader 测绘风险：NULL branch_code 会让 RLS `branch_code='SC'` 等值过滤漏掉这些行，SC-only 开 RLS 才暴露）。
- oracle：`duckdb` 直查 `current/` 派生分布全 SC、`validation/SX` 全 SX，与 `branch_code` 列逐省一致（万分之一）；ETL 自校验在 SC + SX 数据上通过；故意喂一行异常 policy_no 应 fail-fast；`bun run governance`。
- 依赖：A1。
- 替代 #753：替代 2.1 的常量标签语义。

**A3 · fields.json 定义更新 + codegen**
- 范围：`fields.json:550-562` 把 `branch_code` derivation 从 `constant` 改为派生/computed，描述注明来源 = `policy_no[2:4]`；跑 `node scripts/field-registry/generate.mjs` 更新 mapping/validator/etl_fields；`bun run governance`（#17 字段一致性）。
- oracle：codegen 产物一致；governance 全过。
- 依赖：A2。

**A4 · backfill 派生化 + 存量校验**
- 范围：`数据管理/pipelines/backfill_branch_code.py` 从「贴常量」改为「从 policy_no 派生 + 校验存量 parquet」，对现有 `current/` 全量验证（抓任何历史贴错行）。
- oracle：`duckdb` 验证 `current/` 全部派生 = SC；输出校验报告；SC parquet 字节不变（只读校验，除非发现错行才改）。
- 依赖：A2/A3。

**A5 · 报价/续保/派生域号码体系确认（对抗审查补）**
- 范围：确认 `quotes`（报价单号）、`renewal`、派生域（cross_sell/customer_flow/new_energy）各自主键号码能否派生省份；不能则明确「这些域 branch_code 继续走 loader 注入/继承 PolicyFact」并兜底；登记结论。
- oracle：duckdb 抽查各域号码前缀；文档化结论。
- 依赖：A1（可与 A2 并行）。

### Phase B — 分省子目录 `current/<省>/` 替代 #753 前缀（高爆炸半径，后置立项）

> ⚠️ B 阶段动工前需用户确认「是否值得为根治 sync 复杂度付大爆炸半径」。以下任务每个都要保证 **BRANCH_RLS_ENABLED=false + 仅 SC 数据时逐字节/逐语义与现状一致**（单省短路 = 不引入子目录层，沿用 ADR §11「单源短路=历史逐字节一致」范式）。

**B1 · 装载层子目录发现（🔴 生死点，最高优先级）**
- 范围：`server/src/services/data-bootstrapper.ts:150-164` `discoverParquetFiles` 非递归 `readdirSync` → 下钻 `current/<省>/` 枚举（这是服务端**唯一** PolicyFact 文件发现入口，漏改则顶层 readdir 返回空 → PolicyFact **静默 0 行**、全站空白）；`server/src/config/paths.ts:28-32` `getCandidateDataDirs`；`deduplicateOverlapping`(`:200-243`) **去重分组键必须纳入 branch 维度**（否则 SC/SX 同起期文件按 8 位日期分到同组、互补剔除 → 丢一省全量数据）。`read_parquet` 用显式文件数组（`duckdb-parquet-loader.ts:134/141`），对子目录透明，但**文件须由 JS 端 readdir 显式列出，DuckDB 不自动递归**。
- oracle：duckdb `raw_parquet` 行数 == 子目录文件行数之和；PolicyFact 行数正确；`duckdb` 直查 `raw_parquet` vs `PolicyFact` 的 branch_code 分布逐省核对（**golden-baseline 在 E2E_PASSWORD 缺失时 BLOCKED，须用此 duckdb 兜底证 SC 零差异**）；保留 legacy fallback(`applyLegacyFallback:170-194`)。
- 替代 #753：为退役前缀做装载层准备。

**B2 · ETL 落盘子目录 + 5 处扁平 readdir 改造**
- 范围：`daily.mjs`/`transform.py` 按**派生省份**把产物落 `current/<省>/`；改 5 处扁平 `readdirSync` 站点下钻子目录并**配套测试断言「子目录里的分片被正确枚举」**（防沉默失败——`parquet-overlap-check.mjs:52`、`quick_reference.mjs:96` 若失明于子目录会返回 0 分片 → 重叠检测假「通过」、shardCount 0，比报错更危险）：`scripts/lib/parquet-overlap-check.mjs:30-68`（已部分 branch-aware 可参考 `parseBranchFromFilename`）、`数据管理/pipelines/quick_reference.mjs:94-123`、`数据管理/daily.mjs:1760-1771`（行数汇总）、`daily.mjs:1679-1690`（归档）、`daily.mjs:1319-1323/1831-1842`（完整性/门禁）。
- oracle：子目录枚举测试；`node scripts/check-data-readiness.mjs`；governance。
- 依赖：B1。

**B3 · sync-vps 退役前缀方案**
- 范围：`scripts/sync-vps.mjs` 用「每省独立同步 `current/<省>/` 子目录」**整段退役** #753 的 `branchFilePatterns`/`buildRsyncBranchFilterArgs`/`R/P` filter + `--exclude` 体操；**改写/废弃对应测试**。🔴 GATED：`rsync -a` 会递归同步子目录 → `current/SX/` 若有数据会直接推生产，违反 D5 顺序铁律；**cutover 前 `current/SX/` 须为空 / 显式排除 / 仍走 validation/ 隔离**。
- oracle：真实本地临时目录 rsync 测试验证每省隔离 + SC-only 字节安全；#753 测试退役/替换有回归保护。
- 依赖：B1/B2。替代 #753：退役 2.2 前缀+filter。

**B4 · governance 闸：禁混省文件 + web 上传子目录化**
- 范围：新增 governance 检查「单个 parquet 文件内所有行派生省份一致」（读派生省份）；`server/src/routes/data.ts:55/361-393` web 上传归档正则子目录化（子目录方案天然规避前缀方案 `data.ts:361` 把 `SX_` 吃进 prefix 段的已知缺陷）。
- oracle：故意造混省文件应被闸拦；governance。
- 依赖：B2。

**B5 · RLS/loader/cutover 交互 + 性能**
- 范围：确认 RLS（`permission.ts:88-99`）/loader 对子目录无感（branch_code 仍是查询期列）；`current/SX/` 与 `validation/SX/` 严格区分；bench `PolicyFactRealtime` 是否需补 `branch_code` 索引（`materialization.ts:203-208` 现 4 索引无 branch_code，多省合表数据翻倍后 RLS 等值过滤可能拖慢 p95）；更新 Day-1 SOP cutover 序列适配子目录。
- oracle：`scripts/benchmark-key-routes.mjs` p95 不退化；RLS 隔离测试；GATED 序列文档更新。
- 依赖：B1-B4。

## 5. 关键风险与防护（测绘实证）

| 风险 | 后果 | 防护 |
|------|------|------|
| **NULL/异常省码** | RLS `branch_code='SC'` 漏掉这些行，SC-only 开 RLS 才暴露 | A2 ETL 自校验 **fail-fast**（宁缺勿错，同 permission.ts fail-closed 理念） |
| **discoverParquetFiles 漏改子目录** | PolicyFact **静默 0 行**、全站空白（非报错） | B1 最高优先级单点，配 duckdb 行数核对 |
| **去重跨省误删** | SC/SX 同起期文件互补剔除丢一省 | B1 去重分组键纳入 branch 维度 |
| **沉默失败（子目录失明）** | 重叠检测假「通过」、shardCount 0，门禁永远放行 | B2 每处改造配「子目录分片被枚举」断言测试 |
| **SC 字节安全无自动门** | golden-baseline BLOCKED on E2E_PASSWORD | duckdb 直查 raw_parquet vs PolicyFact branch_code 逐省核对兜底 + 单源短路按构造证明 |
| **GATED 越界** | rsync -a 递归把 current/SX/ 推生产、或 validation/SX 并入发现路径致跨省串读 | current/SX/ cutover 前为空 + 与 validation/SX 严格区分 + rsync 显式排除 |

## 6. 迁移 / 回滚 / 边界

- **分阶段独立可回滚**：Phase A 独立交付（替代标签，不依赖 B）；B 后置，每任务单源短路保证 SC 字节安全。
- **GATED 红线不变**：本方案任一任务**严禁触发把山西数据推生产**（current/SX/ 保持空、validation/SX 隔离）。山西真正上线仍是独立的 GATED cutover（RLS-on → SX 进 current/ → sync → 发账号），需用户拍板。
- **与 #753 的关系**：A 落地后 #753 的「常量标签」部分被替代；B 落地后 #753 的「前缀+filter」部分被退役。若只做 A 不做 B，须在文档标注「branch_code 已派生化，SX_ 前缀仅作 sync 文件区分、非省份真相来源」，避免两套机制语义冲突。
