## 3. 关键决策门与方案选定

### 3.1 🔴 派生轴决策门（Phase 0 必先定）

现有派生引擎 `transform.py:1113` = `df[source].astype(str).str[:prefix_len].map(mapping)` —— **只能从字符串开头切 N 位**。两条路：

| 选项 | 派生键 | 引擎改动 | 风险 |
|------|--------|---------|------|
| **A（推荐·零代码）** | `policy_no[:3]` → `{"610":"SC","618":"SX"}` | **无**（现成 prefix_map） | 依赖「首位恒为 `6`」——须 Phase 0 用 duckdb 验证 `substring(policy_no,1,1)` 全 `6`，并 grep 已知机构编码确认无未来省份撞 `610`/`618` |
| B（更精准·需改引擎） | `policy_no[1:3]` → `{"10":"SC","18":"SX"}` | 扩展引擎支持 offset/substring | 改动量增加，但语义最准（直取二级机构码，免首位假设） |

**默认走 A**（零代码 + Phase 0 撞车检查）；Phase 0 发现首位非恒 `6` 或撞车风险 → 升级 B（条件分支，引擎加 offset）。

### 3.2 方案选定：渐进式 A（检测层）→ 治理 → B（隔离层）

| 阶段 | 内容 | 爆炸半径 | 交付的能力 |
|------|------|---------|-----------|
| **Phase A**（先做） | branch_code 派生化 + 全域自校验 + 回填 + 治理闸 | **小-中**（不触碰装载/RLS/loader） | **检测层根治**：防贴错、检测混省、自校验 fail-fast |
| **Phase B**（后置立项·需用户确认） | `current/<省>/` 子目录 + 每省独立 sync，退役 #753 前缀+filter | **高**（6 站点 + 生死点） | **隔离层根治**：物理删不错省 |

> ⚠️ **过渡债务（须明确告知用户）**：Phase A 只给「检测混省」不给「物理删不错省」；隔离层在 Phase B 交付前**仍靠 #753 前缀+filter**。即「本期是检测层根治、隔离层维持 #753 现状」。

## 4. 分阶段单任务（供新会话按 Loop v2 逐个 evidence-loop 执行）

### Phase 0 — 基线锚定 + 铁证复验 + 派生轴决策门（爆炸半径=零，纯证据/决策）

- **依赖**：无。**替代 #753**：不替代，锚定 + 决策。

### Phase 1 — premium 域派生化 + ETL 自校验（改注册表一处 + codegen）
- **范围**：① 改 `fields.json:554-558` `branch_code.derivation`：`type:constant`→`type:prefix_map`，`source:"policy_no"`，`prefixLength:3`，`mapping:{"610":"SC","618":"SX"}`；**移除/改造 `defaultValue`**（现状 `transform.py:1114-1115` 对 prefix_map 也 `fillna(defaultValue)` → NULL 省码会被静默填，与自校验冲突；改为不填默认 + NULL 触发 fail-fast）。② 跑 `node scripts/field-registry/generate.mjs` 重生成 mapping/validator/etl_fields。③ `transform.py` 派生块末尾加**自校验断言**：派生省码集合 ⊆ `{SC,SX}` 且若 `env BRANCH_CODE` 声明了省则派生值必须全等声明值，否则 `sys.exit(1)`（抓喂错省/混省）；NULL 派生值 fail-fast（否则 RLS `branch_code='SC'` 等值过滤漏行，rls_loader 测绘风险）。
- **oracle**：`bun run governance`（#17 字段一致性 + #23 体积）；本地 premium ETL（不传 BRANCH_CODE 走 SC 默认链路）后 duckdb 直查 `branch_code` 全 `SC` 且与 `substring(policy_no,1,3)` map 逐行相等；故意用 SC 的 policy_no 传 `--branch-code SX` 应 `sys.exit(1)`。
- **依赖**：Phase 0。**替代 #753**：替代 2.1 premium 域常量标签语义。

### Phase 2 — SC-only 字节安全验证（重构性质·行为不变 oracle）
- **范围**：premium 全量重跑 ETL（SC 默认链路），逐分片 duckdb 对账新旧 parquet `branch_code` 列逐行相等（旧=常量 `SC`，新=派生 `610`→`SC`，值相同）；`golden-baseline.mjs --compare` 71 端点零差异（**BLOCKED on E2E_PASSWORD 时以 duckdb 逐省核对为主 oracle**）；`bun run verify:full`。
- **oracle**：差异行数==0；零端点回退。**依赖**：Phase 1。**替代**：不替代（Phase 1 安全门）。

### Phase 3 — 全域派生覆盖（各域差异化·测绘实证）
- **范围**（逐域，按号码体系差异化处理）：
  - `claims_detail`（`convert_claims_detail.py:302-305`）：policy_no 全 `610` 零例外 → **hard-fail** 自校验。
  - `cross_sell`/`customer_flow`（`base_converter.py:177-181`）：派生 + 自校验。
  - `renewal`（`convert_renewal_tracker.py:83-117`）：用**当年号 `renewed_policy_no`**（非上年号 `source_policy_no`，避免取错省）。
  - `quotes`（`quote_etl.py:277-281`）：policy_no 可空 → 按 Phase 0 策略 **warn**（不 hard-fail）或 VIN→policy JOIN。
  - `new_energy_claims`（`convert_new_energy_claims.py:99-139`）：**policy_no 100% NULL → 无法 prefix_map**，须 VIN JOIN 取省 或显式划独立后续任务。

- **依赖**：Phase 1（A5 quotes 确认可与 Phase 1 并行）。**替代 #753**：扩展替代范围（#753 的 branch_code 仅 premium 有意义）。

### Phase 4 — 存量回填 + governance「单文件不混省」闸
- **范围**：① `backfill_derived_fields.py --force` 把历史 premium + claims_detail + 已覆盖派生域 parquet 的 `branch_code` 从常量重算为派生（已支持 prefix_map，改 fields.json 后自动按 policy_no 派生）；SC 存量变更数应==0（字节安全）或变更行逐条登记为数据质量发现。② `check-governance.mjs` 新增闸「单文件不混省」：每个 parquet duckdb 直查 `COUNT(DISTINCT branch_code)`>1 或「派生省≠列省」→ exit 1。③ `parquet-overlap-check.mjs:30-33,65-67` 的 `parseBranchFromFilename` 从文件名轴（`^([A-Z]{2})_`）切到**派生轴**（读 parquet 派生省份）。
- **oracle**：backfill 日志 SC 变更==0；故意造混省/贴错标签（`branch_code='SX'` 但 policy_no `610`）parquet 被闸检出 exit 1；单省通过。
- **依赖**：Phase 1-3。**替代 #753**：替代「文件名前缀=省份契约」为「parquet 内派生省份=唯一事实源」。

## 5. 关键风险与防护（测绘实证）

| 风险 | 后果 | 防护 |
|------|------|------|
| **`fillna(defaultValue)` 静默填 NULL** | NULL 省码被填成 SC，RLS 漏行不可见 | Phase 1 移除 branch_code 的 defaultValue + NULL fail-fast |
| **派生轴用前3位的撞车** | 未来某省非 `6` 开头或撞 `610/618` → 映射崩 | §3.1 Phase 0 决策门:验首位恒 `6` + grep 撞车,否则走 B(offset) |
| **discoverParquetFiles 漏改子目录** | PolicyFact 静默 0 行、全站空白 | B1 最高优先级,duckdb 行数核对 |
| **去重跨省误删** | SC/SX 同起期文件互补剔除丢一省 | B1 去重分组键纳 branch 维度 |
| **沉默失败（子目录失明）** | 重叠检测假「通过」、shardCount 0 | B2 每处配「子目录分片被枚举」断言 |
| **SC 字节安全无自动门** | golden-baseline BLOCKED on E2E_PASSWORD | duckdb 直查 raw_parquet vs PolicyFact branch_code 逐省核对兜底 |
| **GATED 越界** | rsync -a 把 current/SX/ 推生产 / validation/SX 并入发现路径致跨省串读 | current/SX/ cutover 前为空 + 与 validation/SX 严格区分 |
| **双注入路径过渡债务** | premium 纯派生 / claims+quotes 贴常量+断言,语义不齐 | 统一为纯派生登记后续任务,文档标注过渡态 |
| **quotes 可信度未验** | 本地无 quotes parquet,自校验只能 warn | Phase 0 生产抽样后才能升级 hard-fail |

## 6. 迁移 / 回滚 / GATED 边界

- **分阶段独立可回滚**：Phase 0-5（检测层）独立交付；B（隔离层）后置，每任务单源短路保 SC 字节安全。
