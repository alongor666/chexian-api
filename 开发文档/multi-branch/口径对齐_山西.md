## 2. 字段值域确认 checklist

山西首批 xlsx 到位后，跑以下检查（与 SC 对照）：

```bash
# 字段枚举
duckdb -c "SELECT DISTINCT plate_province FROM '数据管理/warehouse/fact/policy/current/*.parquet' WHERE branch_code='SX' ORDER BY 1"

# 客户类别枚举
duckdb -c "SELECT DISTINCT customer_category FROM '数据管理/warehouse/fact/policy/current/*.parquet' WHERE branch_code='SX' ORDER BY 1"

# 风险等级枚举
duckdb -c "SELECT DISTINCT insurance_grade FROM '数据管理/warehouse/fact/policy/current/*.parquet' WHERE branch_code='SX' GROUP BY insurance_grade"

# 险类枚举
duckdb -c "SELECT DISTINCT insurance_type FROM '数据管理/warehouse/fact/policy/current/*.parquet' WHERE branch_code='SX'"

# 非空率
duckdb -c "SELECT
  ROUND(COUNT(*) FILTER(WHERE customer_category IS NULL) * 100.0 / COUNT(*), 2) AS null_pct_customer_category,
  ROUND(COUNT(*) FILTER(WHERE insurance_grade IS NULL) * 100.0 / COUNT(*), 2) AS null_pct_grade,
  ROUND(COUNT(*) FILTER(WHERE plate_province IS NULL) * 100.0 / COUNT(*), 2) AS null_pct_province
FROM '数据管理/warehouse/fact/policy/current/*.parquet' WHERE branch_code='SX'"
```

**判断标准**：
- 字段枚举与 SC 完全一致 ✅
- 字段值域有差异（如山西特有客户类型） → 在本文件 §1 对应子章节追加，并请山西方业务签字
- 字段非空率与 SC 偏差 < 1% ✅；偏差 ≥ 1% → 排查 ETL 是否漏字段
