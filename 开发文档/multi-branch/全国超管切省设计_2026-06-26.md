# 全国超管「切省 + 全国合并视图」设计（方案 B：前端省份切换器）

> **来源**：用户 2026-06-26 确认。山西 cutover 步⑥发账号完成后，提出 `xuechenglong`（薛成龙）应为**全国用户**——可切单省（四川/山西）查看，也可看**全国多省合并视图**；其他用户维持现有分级权限不变。
>
> **状态**：设计已定，**实现待新会话执行**（本文件 = 实现依据 + 接力契约）。
> **关联 ADR**：[全国多省架构决策_2026-06-19.md](./全国多省架构决策_2026-06-19.md)（D1 = 单数据湖 + `branch_code` 列 + 行级安全 RLS；终局确认"总部需看跨省合并视图"）。本设计是该终局需求在**用户态/权限层**的首个落地。

## 2. 现状约束（为什么不能简单清空 branchCode）

> 核实于 2026-06-26，三道现有安全闸决定了实现路径：

1. **`server/src/middleware/permission.ts:88-94`**：RLS-on 时 `branchCode` 缺失 → **fail-closed 401**。所以全国超管**不能**靠"清空 branchCode 看全国"。
2. **`server/src/middleware/permission.ts:100-105`**：`branchCode` 必须 `^[A-Z]{2}$`，否则 403。
3. **`server/src/middleware/__tests__/permission.test.ts:181-187`** 不变量：*"所有 preset 用户都有 branchCode，无人缺省"*（fail-closed 前提）。

**结论**：全国超管**保留 `branchCode='SC'` 作默认省**（满足上述不变量），叠加新字段 `visibleBranches` 表达"可切省集合"。**不破坏任何现有不变量**。

## 3. 数据模型（新增字段）

### 3.1 `PresetUser` / store / JwtPayload 新增 `visibleBranches?: string[]`

- 语义：该用户**可切换/可合并**的省集合。
- `undefined` 或 `[]` → 普通用户（仅 `branchCode` 单省，行为不变）。
- `['SC','SX']` → 全国超管，可切 SC、切 SX、或看 ALL（= 该集合合并）。
- `xuechenglong`：`branchCode='SC'`（默认省）+ `visibleBranches: ['SC','SX']`。

> **加省时只改这里**：第 3 省上线后把 `'GD'` 等加进全国超管的 `visibleBranches`，无需动 permission 逻辑（IN 列表数据驱动）。

## 4. 请求参数 `targetBranch`

- 新增**全局查询参数** `targetBranch`（camelCase），值 ∈ `{'SC','SX','ALL'}`（省份枚举数据驱动，'ALL'=合并）。
- **登记**：`server/src/config/route-param-contracts.ts` 的 `commonFilterSchema`（所有 query 路由通用），否则 `scripts/route-catalog/validate-params.ts` 覆盖率检查会拦截。
- 前端从 `BranchContext.selectedBranch` 注入到每个 `/api/query/*` 请求。

## 5. permission.ts 改造（**安全核心**）

`server/src/middleware/permission.ts:59-110`，在现有 `baseFilter`（role 三态，**不动**）之后、RLS 注入段重写：

```text
const { role, organization, branchCode, visibleBranches } = req.user
const requested = req.query.targetBranch  // 用户可控，绝不信任原值

if (isBranchRlsEnabled()) {
  let branchClause
  if (visibleBranches?.length) {
    // —— 全国超管路径 ——
    if (requested === 'ALL') {
      // 合并视图：IN (可见省集合)，配合 GROUP BY branch_code 自然汇总
      branchClause = `branch_code IN (${visibleBranches.map(quoteUpper2).join(',')})`
    } else if (requested && visibleBranches.includes(requested)) {
      branchClause = `branch_code = '${requested}'`
    } else {
      // 无参/非法参 → 回落本人默认省（保守，不默认 ALL）
      branchClause = `branch_code = '${branchCode}'`
    }
  } else {
    // —— 普通用户路径：忽略 targetBranch（防越权）——
    if (!branchCode) throw 401
    if (!/^[A-Z]{2}$/.test(branchCode)) throw 403
    branchClause = `branch_code = '${branchCode}'`
  }
  req.permissionFilter = baseFilter === '1=1' ? branchClause : `(${baseFilter}) AND ${branchClause}`
} else {
  req.permissionFilter = baseFilter
}
```

> **安全不变量（必测）**：普通用户（`visibleBranches` 空）传 `targetBranch=SX` **一律被忽略**，强制本人 `branchCode` → 不能越权看他省。`visibleBranches` 里的每个值同样要过 `^[A-Z]{2}$` 校验再拼 SQL。

## 6. cache key 改造（防切省串读）

- `server/src/routes/query/shared.ts:160-178` `buildRouteCacheKey`：cache key 第二段已含 `req.permissionFilter`，全国超管切省后 permissionFilter 已不同（`='SX'` vs `IN('SC','SX')` vs `='SC'`）→ **天然区分**。
- 但第三段 `branchSegment = b=${branchCode}`（`shared.ts:175`）对全国超管恒为 `b=SC`，不够。**改为含 effectiveBranch / targetBranch 维度**，双保险。
- `server/src/services/cache-warmer.ts:36-50` `getWarmVariants`：加 `ALL` 合并变体 + 各 `visibleBranches` 单省变体预热（否则全国超管首次切省冷启动慢）。

## 10. 风险与护栏

1. **安全核心**：`targetBranch` 用户可控 → permission.ts 必须用**服务端 token 里的 `visibleBranches` 白名单**校验，普通用户传参一律忽略。绝不信任请求原值拼 SQL。
2. **fail-closed 不破坏**：xuechenglong 保留 `branchCode='SC'`，permission.test.ts:181 不变量不变。
3. **cache 串读**：切省必须区分 cache key（§6）+ 前端清缓存（§7）。
4. **deploy.yml 覆盖 ecosystem**（memory `feedback_deploy_yml_ecosystem_overwrite`）：本功能**纯代码**，不动 ecosystem 环境变量，无此风险。
5. **governance**：`bun run governance`（RouteCatalog 参数契约对账 + 能力矩阵镜像）+ `bun run verify:full` 必过。
6. **指标口径不变**：本功能只改"看哪个省的数据"，不改任何指标公式 / RLS 之外的 SQL。
