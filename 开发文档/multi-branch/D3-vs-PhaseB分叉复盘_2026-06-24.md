## 一、结论先行

**这不是 AI 私自跑偏，而是"用户已批准、但 SSOT 从未收敛"的决策分层问题。** ADR §1 的 D3（前缀）从未更新，而真实方向（子目录终局）只活在 §11 日志 + 一份独立规划文档里。两条互斥轨道在 2026-06-23 同一天被两个 loop 会话并行落地，谁也没带上谁的上下文 —— 撞在 sync 层（B3 退役 #753 前缀函数，而 #783 promote 仍建在其上）。

## 四、撞车机制：计划里是串行，执行中变并行

规划写的是 **"前缀 NOW（本期维持 #753）/ 子目录 LATER（Phase B，gated，问用户）"**。但执行把两段压在同一天并行跑：

- **#783 会话**：建 SX cutover promote 在前缀上 —— **符合"本期隔离层维持 #753"**，本身没错。
- **Phase B B1/B2/B3 会话**：抢跑子目录替代 —— **也是被授权的 Phase B**。
- 两个会话都"对"，但**没有任何机制让 #783 会话知道 B3 正在抽掉它脚下的前缀地板**。
- 加重情节：被 B3 退役的那个前缀 rsync filter，**此前已造成过一次删四川数据的 P0**（子目录方案 §2.2 自述）—— B3 退的是个已知危险件，#783 却还在它上面盖楼。

## 六、系统性根因（喂回 loop 优化线 —— 本复盘是 dispatch 架构轴缺口的实锤）

1. **dispatch 没有架构/程序轴**：两个会话在同一个紧耦合程序上扇出，唯一约束「前缀 NOW、子目录 LATER、别同时建」只活在 §11 散文 + 规划文档里，#783 会话没带上，dispatch 也拦不住。
2. **ADR §1 不是真 SSOT**：决策收敛靠人记得去翻 §11，没人翻 → 沉默撒谎。
3. **contract-token preflight 本可机械拦下**：B3 删 `buildRsyncBranchFilterArgs`，grep main 见 #783 仍引用 → 应阻断 + ack。**这一条机械闸，是本轮唯一能在执行期掐断撞车的东西。**

## 七、收敛动作（本复盘触发）

1. **收敛 §1**：ADR 新增 **D6**（0a 隔离层终局=子目录，退役前缀）+ D3 行加超越标记 + §11 2026-06-24 校准日志（遵 §9 不静默改写）。
2. **完成隔离层子目录化**：B3 落地（`950267c6` 复活）+ 退役/隔离 #783 前缀 promote（SX 未 promote，安全）+ 重建子目录版 SX promote（B5 cutover 前）。
3. **残余扁平站点**：基本已填平，仅剩 sync-vps（=B3）+ web 上传（已知限制）。
4. **loop 机制**（独立 follow-up）：本分叉作为 worked example，驱动 dispatch 架构轴（`program/decisionStatus`）+ contract-token preflight 立项。
