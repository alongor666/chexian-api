## 4. 硬护栏（不可越线）

1. 🔴 **GATED**：严禁触发把山西数据推生产。`current/SX/` 保持空、`validation/SX/` 隔离。山西真正上线是独立 GATED cutover（RLS-on → SX 进 `current/` → sync VPS → 发账号），须用户显式拍板，会话永不自动跨越。「只有 2 个省」≠「SX 已上线」。
2. **沉默失败优先级最高**：子目录失明 → 0 分片/0 行 → 假「通过」比报错危险。每个发现/枚举点必须有「子目录分片被枚举」正向断言。
3. **单源短路安全范式**：仅 SC 时逐字节/逐语义与现状一致。
4. **修补不拆除**：退役 #753 前缀须确保子目录方案已逐字节验证等价后才删旧函数，不可先删后补。
5. **部署链禁 auto-merge**：B2/B3 多半碰 `sync-vps.mjs`/`daily.mjs`（部署链敏感）→ 按 `.claude/pr-checklist.md §4` 禁 auto-merge，人工选窗口合并并盯 CI 前 5 分钟。B1/B4/B5 若不碰部署链可走 auto-merge 门。
6. **合并状态用 `gh pr view --json state`（==MERGED）权威核实**，禁信 shell 回显。
7. **禁直推 main**：分支 + PR。
