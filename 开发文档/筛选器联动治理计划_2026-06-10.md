## Phase 0：交叉销售止血（生产事故级，BACKLOG `0f01e6`）

**问题**：点「交强/商业」「气/油」「自卸/牵引/普货」chip → 共享 parser 往 `CrossSellDailyAgg` 注入它没有的列（`insurance_type`/`fuel_type`/`vehicle_model`）→ DuckDB Binder Error → 整页 HTTP 400。其中险类还被双重应用（路由已有正确的 `buildCrossSellAggInsuranceClause` premium>0 口径）。

**评审修订（🔴1 + 🔴2，已坐实于代码）**：
1. **禁止 mutate `req.query`**：`buildCrossSellAggInsuranceClause(req.query.insuranceType)` 在 `cross-sell.ts:109/156/198/370` 与 `bundles/cross-sell.ts:93` 全部位于 parser 调用**之后**直接读 `req.query`——若在 parser 前 delete，险类子句拿到 undefined → 交/商 chip 静默失效，且"WHERE 不含 insurance_type"的测试仍是绿的（抓不到）。修法 = 构造**净化副本**传 parser，险类子句继续读原始 `req.query`（符合 immutability 红线）。
2. **取消 gas/oil → isNev=false 语义降级**：主站 SSOT（`server/src/utils/filter-params.ts:218-223`）气 = `is_nev=false AND fuel_type LIKE '天然气%'`，油是其补集；降级后「气」=「油」= 非新能源全集，用户拿到错误数据且无感知，比 400 更危险。改法 = 前端**同 PR** 隐藏交叉销售页的气/油与自卸/牵引/普货 chip（`QuickFilterBar` 已有 `hideVehicleType` prop 先例 :39，加同款很薄），后端剥离仅作**防御层**（防直接 API 调用 400），不做语义映射。
3. **electric 完全不剥离**（实施核实，比评审建议的"electric→isNev 映射"更简）：SSOT 对 electric 只产出 `is_nev = true`，agg 表**有** `is_nev` 列 → parser 原生正确，无需映射也无需剥离，连 electric+isNev 并存的冲突边界都不存在。
4. **strip 只作用 agg 分支**：`/cross-sell-org-trend`（:238-274）走 PolicyFact（原生支持三列），不剥离；`/cross-sell-heatmap`（:286-345）是 agg/PolicyFact 双分支（:318-324 按 groupByDimension/drillFilter 选 clause），仅 agg 分支传净化副本——需把 parser 调用移到 `usePolicyFactHeatmap` 判定之后（drillFilter 解析不依赖 parser 输出，重排安全）。

**核心代码**（净化副本，纯函数）：

```ts
/**
 * CrossSellDailyAgg 物化表不含 insurance_type / fuel_type / vehicle_model 列，
 * 共享 parser 按这些参数注入 WHERE 会触发 DuckDB Binder Error（HTTP 400）。
 * 返回净化副本传给 parser；不修改 req.query —— 险类子句
 * buildCrossSellAggInsuranceClause 仍从原始 req.query.insuranceType 读取。
 * gas/oil（依赖 fuel_type）与 dump/tractor/general（依赖 vehicle_model）
 * 为防御性剥离（防直接 API 调用 400），不做语义映射；前端已隐藏对应 chip。
 * electric 不剥离：parser 对其产出 is_nev = true，agg 有该列，与 SSOT 严格等价。
 */
export function sanitizeAggQuery(query: Request['query']): Request['query'] {
  const out = { ...query };
  delete out.insuranceType; // 由 buildCrossSellAggInsuranceClause（premium>0 口径）处理
  if (out.fuelCategory === 'gas' || out.fuelCategory === 'oil') {
    delete out.fuelCategory;
  }
  const vqf = out.vehicleQuickFilter;
  if (vqf === 'dump' || vqf === 'tractor' || vqf === 'general') {
    delete out.vehicleQuickFilter;
  }
  return out;
}
```

// ...
const insuranceClause = buildCrossSellAggInsuranceClause(req.query.insuranceType); // 仍读原始 query
```

import {
  CUSTOMER_CATEGORIES,
  CAT_NON_COMMERCIAL_PERSONAL, CAT_MOTORCYCLE, CAT_RENTAL,
  CAT_NON_COMMERCIAL_ENTERPRISE,
} from '@/shared/config/customer-categories';

/** 车型 chip → 续保域 customer_category（评审 🟡4：必须从注册表常量派生，禁手写字符串——
 *  原稿手写注释把"挂车"误写成不存在的"拖拉机"即硬编码易错的现行犯。
 *  续保域无 tonnage_segment/vehicle_model，仅客户类别可表达的 chip 在本页生效；其余按 ✅D3=B 隐藏） */
const VEHICLE_TO_CATEGORY: Record<string, string[]> = {
  home_car: [CAT_NON_COMMERCIAL_PERSONAL],
  motorcycle: [CAT_MOTORCYCLE],
  rental: [CAT_RENTAL],
};
/** 营/非 → 类别枚举展开：从注册表 11 类按前缀机械派生，与主站 LIKE '营业%'/'非营业%' 语义逐字一致
 *  （摩托车/特种车/挂车不带前缀，两边均不归入） */
const BUSINESS_NATURE_CATEGORIES: Record<string, string[]> = {
  commercial: CUSTOMER_CATEGORIES.filter(c => c.startsWith('营业')),
  non_commercial: CUSTOMER_CATEGORIES.filter(c => c.startsWith('非营业')),
};

function useNonTimeFilterParams(): Record<string, string> {
  const { filters } = useGlobalFilters();
  const full = buildFilterParams(filters); // 统一函数（唯一事实源）
  const out: Record<string, string> = {};
  // 1) 直通白名单（后端原生支持的参数）
  for (const k of ['orgNames', 'salesmanNames', 'customerCategories',
                   'coverageCombinations', 'isNev', 'isNewCar', 'isTransfer', 'isRenewal'] as const) {
    if (full[k] !== undefined) out[k] = full[k];
  }
  // 2) fuelCategory → 续保域中文 fuelCategories
  if (full.fuelCategory) {
    const map: Record<string, string> = { oil: '油', electric: '电', gas: '气' };
    const v = map[full.fuelCategory];
    if (v) out.fuelCategories = v;
  }
  // 3) 车型/企客/营非 → customerCategories。主站语义 = 各自独立 AND 条件 → 交集；
  //    续保后端只有单一 IN 参数，故在前端求交集。
  const catSet = new Set<string>(out.customerCategories ? out.customerCategories.split(',') : []);
  const mapped: string[] = [];
  if (full.vehicleQuickFilter && VEHICLE_TO_CATEGORY[full.vehicleQuickFilter]) {
    mapped.push(...VEHICLE_TO_CATEGORY[full.vehicleQuickFilter]);
  }
  if (full.enterpriseCar === 'true') mapped.push(CAT_NON_COMMERCIAL_ENTERPRISE);
  if (full.businessNature && BUSINESS_NATURE_CATEGORIES[full.businessNature]) {
    mapped.push(...BUSINESS_NATURE_CATEGORIES[full.businessNature]);
  }
  if (mapped.length > 0) {
    const merged = catSet.size > 0 ? mapped.filter(c => catSet.has(c)) : mapped;
    // 评审 🟡4：交集为空 = 主站会返回空结果（两个 AND 条件互斥）。
    // 禁止 fallback 回 mapped（等于静默丢弃高级面板的选择，跨页口径漂移）；
    // 传不可能匹配的占位值复现"空结果"语义。
    out.customerCategories = merged.length > 0 ? merged.join(',') : '__none__';
  }
  return out;
}
```
