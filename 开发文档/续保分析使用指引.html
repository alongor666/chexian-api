核心业务概念

续保率计算公式
-- 核心公式 到期日 = 起保日期 + 1年 - 1天 应续保单（分母） = 上一年起保的保单，且到期日 ≤ 统计截止日期 已续保单（分子） = 应续保单中 续保单号不为空 的保单 续保率（件数） = 已续件数 / 应续件数 × 100% 续保率（保费） = 已续保费 / 应续保费 × 100%
"-1天"的业务含义（易踩坑）

「到期日 = 起保日 + 1年 - 1天」这个 -1天非常关键，会导致每月1日起保的保单在上个月最后一天到期：

起保日期到期日期实际到期月份
2025-01-012025-12-3112月（不是1月！）
2025-02-012026-01-311月（不是2月！）
2025-02-022026-02-012月
2025-03-012026-02-282月（不是3月！）
2024-02-292025-02-282月（闰年→非闰年）
2025-12-312026-12-3012月
实际操作建议 使用「到期月份」筛选时，选择的是保单实际到期的月份（而非起保月份）。例如：想看所有在2月到期的保单续保情况，选「2月」即可，系统已自动换算。
两种统计模式对比
模式一：按到期月份统计（选了具体月份）
  • 分母：该月所有到期保单（含未来）
  • 适合：查看某月完整续保情况
  • 截止日期：❌ 不起作用
  • 场景：追踪1月到期保单、提前看2月预警
模式二：按截止日期统计（选"全部"）
  • 分母：[年初, 截止日期] 内到期保单
  • 适合：年度累计续保率追踪
  • 截止日期：✅ 生效
  • 场景：今日整体续保率、特定时间段统计

Tab 一：续保下钻分析

第1步：设置统计截止日期
📅 统计截止日期： 2026-03-15 起始日期固定为 2026-01-01

截止日期决定「应续件数」的分母范围——只有到期日 ≤ 截止日期的保单才纳入统计。 默认为系统最新数据日期，可手动调整为历史日期以进行回溯分析。

例：今天是3月15日，截止日期设为3月15日 系统统计 2026年1月1日 ～ 2026年3月15日 之间到期的保单（即2025年1月至3月初起保的保单）。 尚未到期的3月16日后到期保单不纳入分母，避免分母虚大导致续保率虚低。
第2步：配置筛选条件
筛选控制条
到期月份： 全部 1月 2月 3月 ...12月
A

仅套单 可选

勾选后只统计「是否交商统保 = 套单」的保单，即商业险+交强险同时投保的套餐单。 适合分析主力续保业务，剔除散单干扰。

B

仅自留续保 可选

勾选后只统计「续保模式 = 自留」的保单，剔除流转、合作渠道等其他模式。 适合考核业务员自身续保能力。

C

到期月份 关键筛选

  • 全部（默认）：使用截止日期模式，统计年初至今所有到期保单
  • 选某月（如3月）：切换为月份模式，统计该月所有到期保单（截止日期不再生效）

A

当日续保率

仅统计当天到期的保单，分母为「当日应续件数」。 如果当天没有到期保单，显示 0/0（不会产生无效续保率）。

B

当月续保率（月累计）

使用窗口函数累计计算：截至该行日期，当月所有已到期保单的续保率。 每行数据都是「截至当日」的月度累计值，随日期推移分母逐渐增大。

C

当年续保率（年累计）

从 1月1日 累计到当行日期的年度整体续保率，是最重要的考核指标。 黄色 ★ 行的「当年续保率」即为当前时点的年度续保率实时值。

如何找到「当前实际续保率」？ 找到 ★ 黄色标记行（最新签单对应到期日），查看该行的「当年续保率」列，即为截至今日的年度续保率。 ★ 行之后的日期行，当日应续=0，月/年累计值不变（显示截止位置的累计情况）。
为什么某月显示「无数据」？ 例如「1月 (无数据)」，说明上一年（2025年）1月没有符合当前筛选条件的起保保单， 因此该月无应续件数。请检查机构/业务员筛选条件，或确认数据文件是否包含该时间段数据。

角色权限说明

👑 管理员（admin）
  • 下钻起点：全公司层级
  • 可查看所有机构数据
  • 可自由回退到「全公司」层级（面包屑第一项可点击）
  • 下钻路径：公司→机构→团队→业务员→险别
🏢 机构用户（org_user）
  • 下钻起点：直接锁定到自己的机构
  • 仅可查看本机构及下属团队、业务员
  • 面包屑「全公司」不可点击（权限锁定）
  • 下钻路径：[本机构]→团队→业务员→险别
数据过滤说明 机构用户登录后，系统自动在所有查询中附加机构过滤条件（org_level_3 = 本机构），确保跨机构数据不可见。 此过滤由后端权限服务（access-control.ts）强制执行，前端无法绕过。

常见问题解答

Q1：续保率为什么和我印象中的数字不一样？
最常见原因：
① 截止日期设置：系统默认截止日期为最新数据日期，只统计已到期保单。保单未到期则不纳入分母，会导致续保率高于月末全量统计。
② 筛选条件：勾选了「仅套单」或「仅自留续保」会缩小统计范围。
③ 到期月份选择：选了具体月份（如3月），截止日期不生效，统计全月数据。
Q2：为什么3月选择「2月到期月份」时，有些2025年2月起保的保单不在结果里？
这是「-1天」规则导致的。2025年2月1日起保的保单到期日是2026年1月31日（1月！）而非2月。 系统按到期日月份筛选，而非起保月份。2月到期的保单实际是2025年2月2日及之后起保的保单。
Q3：续保明细表某个月显示「暂无续保数据」是什么原因？
说明上一年（分析年份-1年）该月没有符合当前筛选条件的起保保单。 可能原因：① 该机构/业务员当月没有保单 ② 数据文件不包含该时间段 ③ 筛选条件过严。 建议先取消所有机构/业务员筛选后重试。
Q4：页面顶部的「数据口径」切换（签单/起保）对续保分析有影响吗？
没有影响。续保率计算强制使用起保日期口径，已在代码层面隔离（剥离 policy_date_start / policy_date_end 筛选参数）。 切换口径选择器不会改变续保分析结果。
Q5：四象限图中圆点的大小代表什么？越大越好吗？
圆点大小代表「应续件数」，即该对象的业务体量。体量大不代表好，要结合象限位置判断： 在「重点提升区」的大圆点（高体量、低续保率）才是最需要优先干预的对象。
Q6：为什么有些日期行的当日应续=0，但月/年累计数字不变？
正常现象。当日没有到期保单（0应续），因此当日续保率无意义，但月累计和年累计是滚动累积值， 不会因当日为0而归零——它们显示的是截至该日的历史累计情况。★ 标记行之后的日期就是这种情况。
