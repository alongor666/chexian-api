# 前端极简架构规划（车险经营分析体验）

## 一、结论先行（Governing Thought)

**车险经营分析前端不缺功能、缺收敛。** 页面层已经完成一轮合并（报表/专项/成本三处合并、六条旧路由重定向），是基本健康的；真正的复杂度沉积在**组件层和文档层**：22 个零引用死文件、六类"同一件事多套实现"的活重复、以及一份与现实严重脱节的功能索引。因此极简架构的路径不是重写，而是**三个动作**：

1. **删**：清除全部经双口径（生产入口 + 测试入口）验证的死代码（本轮完成）；
2. **立**：确立"每类可视化能力只允许一个实现"的唯一实现红线，登记收敛路线（本轮登记、后续分批）；
3. **正**：把功能索引恢复为与代码一致的极简地图，让索引重新成为可信导航（本轮完成）。

这与项目"修补不拆除"红线一致：不动任何活链路，只删除有证据的死物、只做文档纠偏，收敛类改造以需求账本任务分批推进。

## 二、情境—冲突—问题—答案（SCQA）

- **情境**：平台已覆盖车险经营分析主要场景，16 条活跃路由、约 430 个前端源文件。
- **冲突**：功能按"页面为单位"逐次生长，同一类可视化（KPI 卡、热力图、趋势图、排序表格、图表生命周期管理、PDF 导出）在不同页面各自实现，历史合并后留下的旧实现无人回收；索引文档仍在描述已被删除的模块（营销战报、交互式 SQL 查询、系数监控、设置面板、幻灯片周报等）。
- **问题**：如何在不打断经营分析日常使用的前提下，让前端回到"一条决策主线、一套共享组件、一份可信索引"的极简形态？
- **答案**：见第一节三个动作；分解与证据见下。

## 三、MECE 之一：经营分析体验按"决策五问"组织（页面层地图）

车险经营决策者的问题空间可以完备且互斥地分为五问，现有 16 条路由全部可归位，无缺口、无重叠：

| 决策问题 | 回答它的页面（路由） | 状态 |
|---|---|---|
| **1. 经营全貌怎么样？** | 保费分析看板 `/dashboard` · 首页报告门户 `/home` · 图表账本 `/chart-ledger` | 健康 |
| **2. 增长从哪里来、计划能不能达成？** | 业绩分析 `/performance-analysis` · 增长分析 `/growth` · 保费达成 `/reports` | 健康（已并掉 3 条旧路由） |
| **3. 成本亏在哪里？** | 成本分析 `/cost`（含综合分析子视图） · 赔案明细 `/claims-detail` · 费用率发展 `/expense-development` · 摩托车成本模型 `/moto-cost` | 健康（已并掉综合分析路由） |
| **4. 客户留得住、转得动吗？** | 续保追踪 `/renewal-tracker` · 报价转化 `/quote-conversion` · 客户来源去向 `/customer-flow` | 健康 |
| **5. 专项机会与资源在哪里？** | 专项分析 `/specialty`（驾意险交叉销售 + 货车） · 维修资源 `/repair` | 健康（已并掉 2 条旧路由） |

（支撑性路由：登录、数据导入、权限管理，不属于分析体验本身。）

**规划结论**：页面层不需要再合并或新增——五问已闭环。后续新分析需求一律先回答"归入五问中的哪一问、能否进入既有页面作为子视图"，答不上来才允许开新路由。

### C. 文档失实（本轮纠偏）

`src/features/INDEX.md` 仍登记营销战报、交互式 SQL 查询、系数监控、设置面板、幻灯片周报等**已不存在**的模块，且 400 余行按时间流水追加，无法作为导航使用。本轮重写为与代码一致的极简索引（变更史交给 git log）；`src/widgets/INDEX.md`、`src/shared/INDEX.md` 同步剔除被删文件条目。

## 五、极简架构（目标态）

分层沿用 `ARCHITECTURE.md §2.2`（L1 `shared/`+`widgets/` 共享层 → L2 `features/*` 特性层，governance「分层依赖边界」闸已在拦），本规划在其上补充**三条极简红线**：

1. **唯一实现红线**：每类可视化能力（KPI 卡 / 矩阵热力图 / 趋势图容器 / 排序长表 / 导出）在 L1 只允许一个实现；L2 只做取数、口径与装配。新增第二套实现视同违规（先 grep L1 再动手，等同 CLAUDE.md §0"先搜再写"在组件层的落地）。
2. **页面五问红线**：新分析入口必须先归位"决策五问"，优先进既有页面作子视图，开新路由需在需求账本说明为何五问装不下。
3. **索引即地图红线**：`INDEX.md` 只保留"模块 → 职责 → 入口文件"的当前态映射，禁止流水账式追加（历史由 git 承载）。

```
决策五问（体验层）
  全貌 │ 增长 │ 成本 │ 客户 │ 专项        ← 16 条路由，MECE，已闭环
────────────────────────────────────────
L2 features/*（取数 + 口径 + 装配）      ← 每域自治，不横向互引
────────────────────────────────────────
L1 shared/ + widgets/（唯一实现）        ← KPI卡/热力图/图表容器/长表/格式化/样式系统
────────────────────────────────────────
apiClient → /api/query/*（后端聚合）     ← 既有契约不动
```
