# BACKLOG 反复冲突根治：不提交派生视图（2026-07-09）

> 决策人授权：`[policy-override]`（允许改动 append-only 的 `.claude/rules/backlog-eventlog.md` 正文）。
> 母分支：`claude/pr-1005-1008-conflicts-43460d`。触发：PR #1005-1008 反复因 BACKLOG 冲突被卡。

## 1. 根因（观测，非推断）

事件日志重构（PR #522）已把**真相** `BACKLOG_LOG.jsonl` 做成纯追加 + `merge=union` →
结构上不冲突。但它**仍把渲染出来的派生视图** `BACKLOG.md` / `BACKLOG_ARCHIVE.md` 提交进 git，
并给它们也设了 `merge=union` 当安全网。

`merge=union` 只对**纯追加**生效。而看板渲染**不是纯追加**：每次 `add/status/note` 都会改写
计数表头（`活跃任务速查（39 项）`、`P2（19 项）`）、重排、重新分组。两个分支各自渲染出的看板
被 union 机械并在一起 → 产出一个**结构错乱、且 ≠ 折叠(日志)** 的视图。随后 governance 的
**陈旧守卫**（`视图 == 折叠(日志)`）判 CI 红 → PR BLOCKED → 必须 rebase + 重渲 + 重推；
期间任一别的 PR 先合并 → 又陈旧 → **追尾循环**。这就是"BACKLOG 变成推进瓶颈"的真身。

实证：1005-1008 的**唯一重叠文件面**就是这 3 个账本文件（1007 的 sync-vps 与 1008 的 src/
完全不相交）。用新式 `git merge-tree`（尊重 union）测，`BACKLOG.md` 显示"自动合并"无冲突标记
——痛点确实在"union 成功但陈旧被守卫拦下"，而非硬冲突。

## 2. 方案 A：拔掉被提交的派生视图（删机制 > 加机制）

一旦视图不进 git，就**没有被提交的视图可陈旧**，陈旧守卫失去存在理由 → 删除。
契合本项目自己的哲学（backlog-eventlog.md §9）："给 governance 新增闸前先问：能否用
结构性消除让该闸不必存在。"

| # | 文件 | 动作 |
|---|------|------|
| 1 | 版本库 | `git rm --cached BACKLOG.md BACKLOG_ARCHIVE.md`（停止跟踪，本地保留） |
| 2 | `.gitignore` | 追加 `/BACKLOG.md`、`/BACKLOG_ARCHIVE.md` |
| 3 | `.gitattributes` | 删 `BACKLOG.md/ARCHIVE merge=union` 两行 + 更新注释；`BACKLOG_LOG.jsonl merge=union` 保留 |
| 4 | `scripts/check-governance.mjs` | 删陈旧守卫；证据链改"折叠日志→内存查证据"；必需文件/冲突扫描去掉两视图 |
| 5 | `scripts/manage-plans.mjs` | 读账本前若视图缺失则惰性渲染兜底（防静默空索引） |
| 6 | `package.json` | 加 `backlog:render` = `bun scripts/governance-backlog-curate.mjs --apply`（按需一键出看板） |
| 7 | `scripts/backlog/lib.mjs` | 视图头部自述文案去掉"陈旧守卫"提法（改为"本地渲染，gitignored"） |
| 8 | `.claude/rules/backlog-eventlog.md` | 更新模型（视图不进 git）；append §10 记录本次演进 + `[policy-override]` 归因 |

**渲染函数 `renderBacklog/renderArchive` 保留**（`backlog.mjs`/`curate` 仍写本地文件；
`tests/backlog-compact-render.test.ts` 锁定不受影响）。

## 3. 为何 governance 不再需要文件

- 证据链：`loadLog()` + `fold()` 直接在内存任务模型上查终态任务的 `docs/code/evidence(+notes)`
  ——比解析渲染表格**更正确**（校验真相而非派生物）。
- 事件日志结构/孤儿校验：本就只读 `BACKLOG_LOG.jsonl`，不动。
- 于是**全新 clone 不渲染视图也能过 governance**；看板是人类便利品，`bun run backlog:render` 随时生成。

## 4. 效果

PR 从此**只碰 `BACKLOG_LOG.jsonl`**（union，追加天然可交换，结构上零冲突/零陈旧）。
**合并顺序从此永远无关紧要**，"BACKLOG 冲突"整类问题结构性消失。

## 5. 落地顺序

方案 A 作为一个 PR 合入 → 1005-1008 各自 rebase（丢弃视图文件改动、留日志 + 代码）→ 全绿。

## 6. 诚实边界 / 代价

- `BACKLOG.md` / `BACKLOG_ARCHIVE.md` 不再能在 GitHub 网页/仓库根目录直接浏览（决策人已确认"无所谓"）。
  CLAUDE.md 等对 `BACKLOG.md` 的链接在 GitHub 上会悬空，但本地渲染后可读；SSOT 链接指向 `BACKLOG_LOG.jsonl`。
- 不新增任何 CI Action（纯 A，非 A+）；保持最省心稳态。
