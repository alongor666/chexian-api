## 0. 结论摘要（先行）

1. **缺口实证**：生产 PM2 此前**无任何开机自启机制**——无 `pm2-*.service` systemd 单元、无 `@reboot` cron、`/root/.pm2/dump.pm2` 停在 2026-05-23。守护进程仅因 double-fork 挂在 init(pid 1) 下存活；**VPS 一旦因云厂商维护/内核升级/故障重启，`chexian-api` 不会自动拉起，全站宕机直到人工 `pm2 resurrect`**。
2. **已修复（静态验收全绿）**：安装并 enable `pm2-root.service`（`ExecStart=pm2 resurrect`），并 `pm2 save` 把**当前**进程列表刷新进 `dump.pm2`（含 `chexian-api` + `social-work-memory-app`）。开机时 systemd 将执行 `pm2 resurrect` 自动拉起两个应用。
3. **未做（避雷）**：全程**未 reload/restart `chexian-api`**（规避项目 memory 记录的「VPS bcrypt reload 502 地雷」）。`pm2 startup` 只装引导单元、`pm2 save` 只快照，均不触碰运行进程。
4. **剩余唯一确证 = 一次受控重启**（有秒级停机），须用户排维护窗，**禁止擅自重启生产**。故本卡状态置 **PARTIAL**，附「待重启实证」。
5. **可逆**：`pm2 unstartup systemd` 可完全撤销本变更。

## 2. 操作记录（逐条状态变更，一次一条）

环境实证：node/pm2 位于 `/root/.nvm/versions/node/v22.22.0/bin/`（非交互 shell 无此 PATH，故经 `bash -lc` 登录 shell 执行）。

1. **只读复核当前态**：确认无 pm2 unit、`dump.pm2` 旧、`chexian-api` online（见 §1 左列）。
2. **安装引导单元**：`pm2 startup systemd -u root --hp /root`
   - pm2 自动写入 `/etc/systemd/system/pm2-root.service` 并 `systemctl enable pm2-root`（创建 `multi-user.target.wants/pm2-root.service` 符号链接）。
   - 单元关键项：`User=root`、`ExecStart=/root/.nvm/.../pm2 resurrect`、`PM2_HOME=/root/.pm2`、`WantedBy=multi-user.target`。
3. **刷新进程快照**：`pm2 save` → `Successfully saved in /root/.pm2/dump.pm2`（11029→15349 字节，纳入当前 online 的 chexian-api）。
4. **静态验收**：§1 右列三项全绿 + `dump.pm2` 进程名核验通过。

## 4. 回退

本变更完全可逆：

```bash
ssh chexian-vps 'bash -lc "pm2 unstartup systemd"'   # 移除 pm2-root.service 并 disable
```

`dump.pm2` 的刷新无副作用（本就是当前真实进程列表），无需回退。
