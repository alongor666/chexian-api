# 地域/机构硬编码审计框架（多省全国平台）

> 背景：平台已从四川单省演进为多省（SC/SX，未来更多省）。代码里残留大量"四川兼容垫片"与未参数化硬编码。
> 目标：把每一处地域/机构相关代码归入四象限，分清**变量**（应参数化/配置化）vs **通用**（跨地域不变），据此消除硬编码。

## 一、四象限判定（每处硬编码必须归入其一）

| 象限 | 定义 | 处置 | 危害 | 样本 |
|---|---|---|---|---|
| **A. 应变量·已参数化但默认回落 SC** | 接了 branchCode 参数，但 `?? 'SC'` / `?? ...SC` 静默默认四川 | **改 fail-closed**：漏传 branchCode→报错，不静默查四川 | 中（漏配静默错误，难发现） | `kpi-detail.ts:53 SAME_CITY_ORGS_BY_BRANCH[branchCode ?? 'SC']` |
| **B. 应变量·未参数化（真硬编码）** | 完全没接 branchCode，写死四川 schema/机构/列名 | **参数化**：抽成 branchCode 入参或按省配置表 | 高（其他省直接坏） | quote-conversion/customer-flow 的 `policy_date` 列；performance-drilldown 对 SX 400 |
| **C. 通用·写死（正确）** | 全国统一的业务规则，本就不该按省变 | **保持不动**（误改反而错） | 无 | 车险口径/公式、标准四分类、客户11类、指标注册表 L1-L3、`branch_code` 列名本身 |
| **D. 灰色·待业务确认** | 不确定全国统一还是按省 | **登记 + 问业务**，先不改 | 待定 | 固定成本参数、NCD/定价系数表、节假日、计划口径是否分省 |

**核心区分（用户诉求"变量 vs 通用"）**：
- **变量** = A + B：省份代码(SC/SX/610/618)、省份名、机构名/清单、省份专有 schema、数据路径省份段、省份默认值。
- **通用** = C：业务口径/公式、全国统一枚举（四分类/11类客户/险别组合）、字段结构、原子指标。
- **变量的反模式**：`?? 'SC'`（A 象限）——多省时代应一律改为显式必传 + fail-closed。

## 二、危害等级

- **致命**：默认 fallback 到 SC（漏配静默查四川，数据错且无报错）；SQL 假设 SC schema 致他省整功能 400/列缺失。
- **高**：机构名/清单写死（新省新机构不显示/不可查）；省份专有逻辑无分支。
- **中**：已参数化但默认 SC；半参数化（签名接了但内部分支写死）。
- **低**：测试/注释/文档里的 SC 字面（不影响运行，规范问题）。

## 三、每个审计 Agent 的输出（写 findings/<域>.json）

```
{
  "domain": "<域名>",
  "summary": { "A": n, "B": n, "C": n, "D": n, "致命": n, "高": n },
  "top": [ "最高危 5 条一句话" ],
  "findings": [
    { "file": "...", "line": n, "snippet": "...",
      "quadrant": "A|B|C|D", "category": "省份代码|机构名|默认fallback|数据路径|省份schema|业务参数|口径",
      "severity": "致命|高|中|低", "isVariable": true|false,
      "sxBroken": true|false,   // 该硬编码是否已导致 SX/他省故障
      "fix": "一句修复方向" }
  ]
}
```

## 四、不要动的白名单（C 象限·通用，误改即错）

- 车险业务口径/公式（赔付率分子分母、出险率年化、推介率分母…）
- 全国统一枚举：标准四分类、客户 11 类、险别组合、风险等级 A-G/X
- 指标注册表 L1-L3 原子指标（`server/src/config/metric-registry/`）
- 字段注册表的字段**结构/列名**（`branch_code` 列本身是通用的，列的**值** SC/SX 才是变量）
- 业务规则字典口径定义

## 五、审计域划分（5 域并行）

1. **SQL 生成器域** `server/src/sql/**`（最高危，他省故障根源；重点查谁接了 branchCode、谁没接、谁默认 SC）
2. **鉴权/路由/配置域** `server/src/{middleware,services,routes,config}/**`
3. **前端域** `src/**`（机构/省份/客户分类/筛选器/亮灯阈值硬编码）
4. **ETL/数据管道域** `数据管理/{pipelines,daily.mjs,integrations,config}/**`
5. **运维脚本域** `scripts/**`（release/sync-vps/sentinel/multi-branch 的省份假设）
