## 一句话结论

- **文件级检索/审计**可降到 haiku（零损失省约 4×）；**符号级检索**必须 ≥ sonnet（haiku 召回仅 74% 且是静默遗漏）。
- 便宜模型的真实风险不是"答错"而是"静默漏"与"效率方差"——给 haiku 的任务必须配确定性验收闸（测试/基准比对）。
- **长文档对外摘要**是唯一明显按档位分层的任务，值得用 opus。
- 固定开销（每次子代理调用约 4 万令牌）主导小任务成本——**合并小任务比降档更省**。

完整路由策略表见报告第四节。

## 方法可信度

- 设计先经 Opus 审查，跑前抓出 3 个会让结论作废的缺陷（基准口径未固化、单跑无方差、成本不拆输入输出）并当场修正。
- 主观任务由 Opus 盲评：答卷匿名乱序、评审员持基准与量规、逐条核实引用真伪。
- 提示词逐字节相同（sha256 记账）、执行顺序随机化、工作目录中性编号防身份泄露。
- 诚实边界：样本量小（每格 1-3 次），结论为探索性方向；成本为代理指标（不拆缓存读折扣）。
