## 3. 进化总原则（V2 四条继续有效，V3 增两条）

1. （V2）先看见真相，再谈优化——V3 阶段 0 = 修引擎自身断流。
2. （V2）每个进化项必须带 harness / 验收 oracle。
3. （V2）自进化要能减，不只能增——本轮含 2 项撤项/降级决策。
4. （V2）诚实边界——与 owner 既有指令有张力的项显式列出，须 owner 拍板。
5. （V3 新）**机制必须覆盖「非 loop 路径」**——E 系列的记账/回填约定只在 loop 收尾步生效，常规 PR 路径同样产生 entry，闸必须对两条路径一视同仁（D1/D2 的教训）。
6. （V3 新）**度量跟着默认路径走**——协议开关反转（如 codex 默认关闭）时，必须同步评估「原度量是否随之失明」，为新默认路径配等价度量（D4 的教训）。
