## 1. P0 清单（2，建议立即处置）

### 🔴 P0-1 · SQL 注入 — `cost/earned-premium.ts`
- **证据**：`server/src/sql/cost/earned-premium.ts:65,68` 把 `policyMonth`/`orgLevel3` 原值零转义拼入 `policy_month='${policyMonth}'` / `org_level_3='${orgLevel3}'`；路由 `routes/query/cost.ts:33` 声明 `policyMonth: z.string().optional()`（无格式约束），`orgLevel3` 源自 `commonFilterSchema` 的 `z.string().optional()`；链路全程零转义，且 `services/duckdb.ts:74 duckdbService.query()` **不调用 `validateSQL`**（该校验只守 AI/NL2SQL 路径）。
- **影响**：任意已认证用户 `GET /api/query/cost?type=earned&cutoffDate=2026-01-01&policyMonth=x'UNION SELECT…` 可执行任意 SQL。
- **修复**：`escapeSqlValue(policyMonth)` 包裹（与同文件其他路径一致，一行）或路由加 `^\d{4}-\d{2}$` Zod 正则 + `orgLevel3` 走 `buildStringCondition`。

### 🔴 P0-2 · 跨机构越权读取 — `routes/reports.ts`
- **证据**：`server/src/routes/reports.ts:138-175` 两个 GET handler 仅挂 `authMiddleware`，**无 `permissionMiddleware`、无 organization/owner 归属校验**；现有防护仅覆盖路径遍历 + reportId 白名单。报告由 `diagnose-*` skill 平铺存入 `REPORTS_DIR`（文件名为业务名+hash，无机构隔离）。
- **影响**：任意已登录用户（含三级机构 org_user）凭文件名/reportId 读取**跨机构** HTML 诊断报告（含保费/赔付/出险敏感数据），违反 §10 行级安全。当前 `server/data/reports/` 为空，生产数据就绪后显性触发。
- **修复**：①文件名/目录嵌 `org_level_3` 归属段 + handler 比对 `req.user.organization`；或 ②维护「报告→owner/org」元数据映射，serve 前查归属；③叠加 `permissionMiddleware` 取 `req.user.role` 分级（org_user 仅本机构、branch_admin 全部）。**红线：只修补不拆除**。
