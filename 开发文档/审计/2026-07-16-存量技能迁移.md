## 4. chexian-api 侧收口

### 4.1 决策⑤：软链是否进 git

**采用**：项目侧不在 git 里保留任何软链或实体文件，消费完全走 `sync-skills` 装到
`~/.claude/skills/` 的全局软链。

**理由**：
1. 与仓内既有全部共享技能（`chexian-daily-loop` / `chexian-ir-diagnosis` / `chexian-channel` 等）
   消费模式完全一致，不新增第二套机制。
2. 软链是绝对路径（`~/.claude/skills/<name>` → `<技能仓 clone 路径>/skills/<name>`），进 git 会
   把开发者本机路径写死进版本库，不可移植。
3. 新机器接入路径已是既有惯例：clone 技能仓 + 跑 `sync-skills.sh link`，无需为这 14 个技能引入
   新的引导步骤。

### 4.3 治理闸适配

`scripts/governance/skill-frontmatter.mjs` 原只扫描 `.claude/skills/*.md` 扁平文件。本次扩展：

- `.claude/skills/` 为空或不存在 → 优雅通过（迁移后的预期常态）。
