# 定价红线分析（Pricing Redline）

> 核心问题：**赔付和费用不变，定价系数至少要多少才能不亏？**
> 本质：反事实定价分析（counterfactual pricing analysis）——固定成本端，反推收入端红线。

## 公式内核

```
盈亏平衡系数 = 当前实际系数 × (变动成本率 / 100%)
```

| 场景 | 当前系数 | 变动成本率 | 盈亏平衡系数 | 含义 |
|------|---------|-----------|-------------|------|
| 微利 | 1.05 | 95% | 1.00 | 刚好不亏 |
| 亏损 | 1.05 | 110% | 1.16 | 系数需提高10% |
| 严重亏损 | 1.35 | 214% | 2.89 | 系数需翻倍，实际不可能 |

**当盈亏平衡系数 > 系数上限时，意味着该板块靠定价无法救回，只能拒保或限额。**

## 系数上限规则（RED LINE）

系数上限因车辆能源类型和保险起期而异，**不是固定值**：

| 车辆类型 | 生效日期 | 系数上限 |
|---------|---------|---------|
| **燃油车** | 长期有效 | **1.5** |
| **新能源车** | 2025-09-07 之前 | **1.35** |
| **新能源车** | 2025-09-07 起 | **1.4** |
| **新能源车** | 2026-03-07 起 | **1.45** |

**判定可救性时，必须按保险起期选择对应上限**：
- 分析 2024 年数据：新能源用 1.35，燃油用 1.5
- 分析 2025 年数据：新能源用 1.4（简化取高值），燃油用 1.5
- 分析 2026 年数据：新能源用 1.45，燃油用 1.5

SQL 中动态计算上限：
```sql
CASE
  WHEN 是否新能源 = true AND YEAR(保险起期) <= 2024 THEN 1.35
  WHEN 是否新能源 = true AND YEAR(保险起期) = 2025 THEN 1.4
  WHEN 是否新能源 = true AND YEAR(保险起期) >= 2026 THEN 1.45
  ELSE 1.5  -- 燃油车
END AS coeff_ceiling
```

## 标准维度（6 维）

| # | 维度 | SQL 字段/表达式 | 适用场景 |
|---|------|----------------|---------|
| 1 | 新转续过户 | `CASE WHEN 是否新车=true THEN '新车' WHEN 是否过户车=true THEN '过户' WHEN 是否续保=true THEN '续保' ELSE '转保' END` | 始终 |
| 2 | 三级机构 | `三级机构` | 始终 |
| 3 | 风险等级 | `COALESCE(车险风险等级, '无评分')` | 有评分数据时 |
| 4 | 厂牌车型 | `厂牌车型` | 车型维度分析时（Top N） |
| 5 | 经代渠道 | `COALESCE(经代名, '直销')` | 渠道分析时（Top N） |
| 6 | 吨位分段 | `吨位分段` | 货车类别时 |

**可选扩展维度**：能源类型、车价段、车龄段、被保险人年龄、险别组合、车牌归属地。

## 执行流程

### Step 1: 生成红线矩阵 SQL

对每个维度 × 每个年份，计算：

```sql
SELECT
  {维度字段} AS dim_value,
  YEAR(保险起期) AS yr,
  COUNT(*) AS policy_count,
  ROUND(SUM(保费)/10000, 1) AS written_premium_wan,
  -- 成本端（固定）
  ROUND(SUM(COALESCE(已报告赔款,0)) / NULLIF(SUM(保费),0) * 100, 1) AS loss_ratio,
  ROUND(SUM(COALESCE(费用金额,0)) / NULLIF(SUM(保费),0) * 100, 1) AS expense_ratio,
  ROUND(
    SUM(COALESCE(已报告赔款,0)) / NULLIF(SUM(保费),0) * 100 +
    SUM(COALESCE(费用金额,0)) / NULLIF(SUM(保费),0) * 100,
  1) AS variable_cost_rate,
  -- 当前定价
  ROUND(AVG(CASE WHEN 商车自主定价系数 > 0 THEN 商车自主定价系数 END), 4) AS current_coeff,
  -- 红线系数 = 当前系数 × (变动成本率 / 100)
  ROUND(
    AVG(CASE WHEN 商车自主定价系数 > 0 THEN 商车自主定价系数 END) *
    (SUM(COALESCE(已报告赔款,0)) + SUM(COALESCE(费用金额,0))) /
    NULLIF(SUM(保费), 0),
  4) AS breakeven_coeff,
  -- 缺口 = 红线 - 当前
  ROUND(
    AVG(CASE WHEN 商车自主定价系数 > 0 THEN 商车自主定价系数 END) *
    (SUM(COALESCE(已报告赔款,0)) + SUM(COALESCE(费用金额,0))) /
    NULLIF(SUM(保费), 0) -
    AVG(CASE WHEN 商车自主定价系数 > 0 THEN 商车自主定价系数 END),
  4) AS coeff_gap
FROM raw_parquet
WHERE {筛选条件} AND YEAR(保险起期) IN ({年份列表})
GROUP BY dim_value, yr
HAVING COUNT(*) >= 10  -- 最小样本量
ORDER BY dim_value, yr
```

### Step 2: 分类判定

对每个维度值，按红线系数判定可救性：

| 红线系数 | 判定 | 颜色 | 行动建议 |
|---------|------|------|---------|
| ≤ 当前系数 | 已盈利 | 🟢 | 维持或适度降价抢市场 |
| 当前 < 红线 ≤ ceiling×0.85 | 可调 | 🔵 | 提系数即可扭亏 |
| ceiling×0.85 < 红线 ≤ ceiling | 勉强可调 | 🟡 | 需提至接近上限，关注竞争力 |
| 红线 > ceiling | **不可救** | 🔴 | 靠定价无法覆盖，必须拒保/限额 |
| 无商业险系数 | 纯交强 | ⚪ | 无法自主定价，需从承保端管控 |

> **ceiling 取值**：燃油车=1.5，新能源车按保险起期=1.35/1.4/1.45（见上方系数上限规则）

### Step 3: 输出红线报告

#### 格式一：红线矩阵（按维度）

```markdown
## 定价红线矩阵：{维度名}

| {维度} | 年份 | 保单数 | 保费(万) | 赔付率 | 费用率 | VCR | 当前系数 | 🎯红线系数 | 缺口 | 判定 |
|:---|:---:|---:|---:|---:|---:|---:|---:|---:|---:|:---|
| 新车 | 2024 | 330 | 168 | 95% | 0.0% | 95% | 1.35 | 1.28 | -0.07 | 🟢已盈利 |
| 新车 | 2025 | 940 | 432 | 189% | 0.0% | 189% | 1.35 | 2.55 | +1.20 | 🔴不可救 |
```

#### 格式二：行动地图（汇总视图）

```markdown
## 行动地图

### 🟢 已盈利（维持）
- {列表}

### 🔵 微调即可（+5-10%系数）
- {列表}

### 🟡 需大幅调价（系数提至1.15-1.35）
- {列表}

### 🔴 定价不可救（必须拒保/限额）
- {列表及止损金额估算}
```

#### 格式三：反思清单

每次输出红线矩阵后，必须回答以下问题：

```markdown
## 反思

1. **有多少板块红线 > 1.35？** → 这些板块占总保费多少？
2. **调价空间有多大？** → 红线在 1.15-1.35 的板块，提系数后市场竞争力还剩多少？
3. **费用端有压缩空间吗？** → 如果费用率降 5pp，红线会降到可救范围吗？
4. **时间维度**：红线是收敛还是发散？→ 趋势恶化则定价追不上成本
5. **哪些维度组合最毒？** → 交叉 2-3 个维度定位最精准的拒保画像
```

## 交叉红线（高阶用法）

当单维度不够精准时，叠加 2 个维度生成交叉红线：

```sql
-- 例：机构 × 新转续
SELECT 三级机构, {新转续表达式} AS biz_type, YEAR(保险起期) AS yr,
  ... (同上 KPI + 红线计算)
GROUP BY 1, 2, 3
HAVING COUNT(*) >= 10
```

交叉红线用于**精准定位拒保画像**：
- "青羊机构 + 新车 + 8-12万车价" → 红线 2.55，不可救
- "自贡机构 + 续保" → 红线 0.95，盈利

## 限制与注意

| 限制 | 说明 |
|------|------|
| 仅适用于商业险 | 交强险无自主定价系数，纯交强板块标记为 ⚪ 不计算红线 |
| 系数上限假设 | 按燃油/新能源与保险起期动态确定（燃油 1.5，新能源 1.35→1.4→1.45）；各公司/产品实际上限可能不同，需用户确认 |
| 赔款未充分发展 | 最新年度赔款可能尚未报全（IBNR），红线可能低估，需标注 ⚠️ |
| 费用率口径 | 费用金额/签单保费，不含固定成本（人力/租金/系统），实际盈亏需更高系数 |
| 最小样本量 | 默认 ≥10 单才计算红线，避免个案偏差 |

## 领域知识（RED LINE）

- **商车自主定价系数**仅适用于商业险，不适用于交强险。定价红线分析的分母应为"商业险保费"
- **定价系数 ≠ 赔付因果**：提高系数会筛掉价格敏感客户，改变组合结构，实际效果需动态评估
- **系数上限不是固定值**：燃油车=1.5，新能源车=1.35→1.4→1.45（按保险起期），必须动态计算
- **满期赔付率 vs 签单赔付率**：红线计算默认用签单保费做分母（与系数直接挂钩），满期口径仅供参考
- **新能源系数上限逐步放宽**：监管认可新能源高风险特性，允许提价。但即使1.45上限仍远不够覆盖200%+赔付率
