# 摩托车经营诊断（v1.0）

> 摩托车 = 交强险120元/件 + 捆绑人身险。**不能只看交强赔付率**，必须用完整成本模型算真实盈亏。
> 管理层三问：**算上人身险赚不赚？哪些细分在赚？能不能扩大盈利池？**

---

## 诊断路由边界（先判定）

执行前先按 `/diagnose-router` 分流；只要用户问题命中**摩托车 + 经营 / 盈亏 / 赔付 / 成本模型**，本命令优先于 `/diagnose-agent` 和 `/diagnose-segment`。

若用户只问摩托车在两个 cutoff 间的 cohort 发展，先用 `/diagnose-cohort-comparison` 做时间变化，再用本命令解释真实盈亏线；不得用普通车险口径直接下结论。

## 摩托车业务本质（RED LINE）

```
摩托车交强险 ≠ 普通车险

1. 交强险120元/件，费率监管制定，无法调价
2. 每单捆绑销售人身险（意外险），这才是利润来源
3. 车险费用率 = 0（交强险不支付渠道佣金）
4. 人身险通过佣金激励中介销售交强险
```

**禁止事项**：
- ❌ 建议"调价/提高定价系数/风险评分定价/核保收紧"
- ❌ 用100%作为盈亏平衡线（综合成本率框架下 100% 才是盈亏线）
- ❌ 忽略人身险利润只看交强赔付率

**应聚焦**：
- ✅ 用完整成本模型(交强+人身-佣金-管理费)算真实盈亏
- ✅ 按维度找盈利/亏损细分
- ✅ 渠道管理（经代质量筛选）
- ✅ 承保选择（品牌/车型/车龄白黑名单）

## 完整成本模型（综合成本率框架 v2.0）

> 2026-04 升级：从"盈亏平衡赔付率"改为"综合成本率/利润额"框架，与通用车险诊断一致。
> 参数来源：`数据管理/config/fixed-cost-params.json`（含时间属性）
> RED LINE：所有率值必须从绝对值计算，禁止率值直接相加。

### 机构分类

| 类型 | 机构 | 人身险保费 | 人身险佣金率 |
|------|------|-----------|------------|
| **A类** | 宜宾、泸州、自贡 | 200元/件 | 65% |
| **B类** | 天府、乐山、高新、武侯、德阳、资阳、达州、新都等 | 100元/件 | 60% |

> 人身险赔付率 5%（5年均值）、佣金率为当前水平。均有时间属性，变动时更新 fixed-cost-params.json。

### 综合成本率分别计算框架

摩托车综合成本率分**三块**计算（车险 / 人身险 / 合计），全部从绝对值出发：

#### 车险部分（交强险120元/件）

| 成本项 | 分类 | 计算方式 |
|--------|------|---------|
| 交强赔付 | 变动-赔付 | 来自数据源（diagnose_vehicle.py 聚合） |
| 车险费用 | 变动-费用 | 来自数据源（≈0，交强险不付渠道佣金） |
| 附加税费 | 固定 | SUM(交强保费) × 1.5% |
| 推动费 | 固定 | SUM(交强保费) × 0.15% |
| 管理费 | 固定 | Σ(各机构交强保费 × 管理成本比例) |

```
车险综合成本额 = SUM(赔付) + SUM(费用) + SUM(附加税费) + SUM(推动费) + SUM(管理费)
车险综合成本率 = 车险综合成本额 / SUM(交强满期保费)
利润额(车)    = SUM(交强满期保费) - 车险综合成本额
```

> 车险部分由诊断脚本自动计算（combined_cost_ratio / profit_amount 字段）。

#### 人身险部分（A类200元/件，B类100元/件）

| 成本项 | A类 | B类 |
|--------|-----|-----|
| 赔付(约定) | 保费×5% = 10元 | 保费×5% = 5元 |
| 佣金(约定) | 保费×65% = 130元 | 保费×60% = 60元 |
| 附加税费 | 保费×1.5% = 3元 | 保费×1.5% = 1.5元 |
| 推动费 | 保费×0.74% = 1.48元 | 保费×0.74% = 0.74元 |
| 管理费 | 保费×管理成本比例 | 保费×管理成本比例 |

```
人身险综合成本额 = Σ(各机构: 保单数 × 人身险保费 × (5% + 佣金率 + 1.5% + 0.74% + 管理成本比例))
人身险综合成本率 = 人身险综合成本额 / 人身险总收入
利润额(人)      = 人身险总收入 - 人身险综合成本额
```

> 人身险部分需 Phase 2 手动计算（人身险数据不在 parquet 中）。

#### 合计

```
利润额(合)      = 利润额(车) + 利润额(人)
综合成本率(合)  = 1 - 利润额(合) / 总收入
单均利润        = 利润额(合) / 保单数
```

**判定标准**：综合成本率(合) ≤ 100% → 盈利，> 100% → 亏损。

## Phase 2: 成本模型研判

**读取机构下钻 JSON 后，必须执行成本模型计算**：

### Step 1: 分类机构
```python
A_TYPE = {'宜宾', '泸州', '自贡'}  # 200元人身险, 佣金65%
B_TYPE = {其余所有机构}              # 100元人身险, 佣金60%
```

### Step 2: 逐机构计算综合成本率与利润额
```
对每个机构（从机构下钻 JSON 读取）:
  1. 判断 A/B 类 → 确定人身险保费、佣金率（fixed-cost-params.json）
  2. 读取该机构管理成本比例
  3. 车险部分: 读取诊断脚本输出的 combined_cost_ratio / profit_amount
  4. 人身险部分: 保单数 × 人身险保费 × (5%+佣金率+1.5%+0.74%+管理成本比例)
  5. 利润额(合) = 利润额(车) + 利润额(人)
  6. 单均利润 = 利润额(合) / 保单数
  7. 判定: 综合成本率(合) vs 100%（≤100%盈利，>100%亏损）
```

### Step 3: 输出综合成本率表
必须包含：机构、类型、保单数、车险综合成本率、人身险综合成本率、综合成本率(合)、利润额、单均利润、判定。

## Phase 3: 多维毒性分析（按需触发）

### 标准9维度
被保险人年龄、车牌归属地、车型(按排量)、新旧车、车价、车龄、业务员、经代、能源

### 毒性指数
```
毒性指数 = 赔付贡献度% / 保费占比%
```
- **> 1.3**：毒瘤（保费小赔付大）
- **< 0.8**：盈利池

### 盈亏判定必须用真实盈亏线

对每个维度的每个值，判定盈亏时：
- 不能用100%
- 用加权平均盈亏线 = A类保费占比×123.7% + B类保费占比×111.5%
- 或按该维度值在各机构的分布，分别计算

## 指标体系

与 `/diagnose-agent` 相同，但增加：

| 指标 | 公式 |
|------|------|
| 综合收入/单 | 交强险120 + 人身险(200或100) |
| 综合变动成本/单 | 交强赔付 + 人身险佣金 + 人身险赔付 + 工资 + 推动费 |
| 管理费/单 | 综合收入 × 管理成本比例（逐机构，见 fixed-cost-params.json） |
| **综合成本率(车)** | 车险综合成本额 / 交强满期保费（绝对值除法） |
| **综合成本率(人)** | 人身险综合成本额 / 人身险收入（绝对值除法） |
| **综合成本率(合)** | 1 - 利润额(合) / 总收入 |
| **利润额(合)** | 利润额(车) + 利润额(人) |
| **单均利润** | 利润额(合) / 保单数 |
