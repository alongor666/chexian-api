# 两阶段经营诊断（v6.0）

> 核心理念：脚本是聚合引擎，不是诊断者。Claude 读结构化 JSON 做研判，不够时追加下钻。
> 管理层三问：**赚不赚？亏在哪？要不要继续？**

## 文件组织规则（RED LINE）

**每个分析项目一个独立文件夹**，所有产出文件放入同一目录：

```
数据分析报告/
├── {项目名}_诊断_{years}/
│   ├── {title}_经营诊断_{years}_截至{date}.md    ← Phase 1 主报告
│   ├── {title}_诊断_{years}_截至{date}.json      ← Phase 1 结构化数据
│   ├── drilldown_*.json                          ← Phase 2 下钻数据
│   ├── dim_*.json                                ← Phase 3 维度分析
│   ├── toxic_segments.json                       ← Phase 3 毒瘤板块
│   ├── profitable_segments.json                  ← Phase 3 盈利池
│   └── analyze_dimensions.py                     ← Phase 3 分析脚本
```

**命名规则**：`{分析对象}_诊断_{起始年}-{结束年}/`，如 `摩托车诊断_2022-2026/`

脚本执行时统一用 `--output` 参数指向项目文件夹。

## 领域知识路由（RED LINE）

| 客户类别 | 使用技能 | 原因 |
|----------|---------|------|
| **摩托车** | **`/diagnose-motorcycle`** | 独有成本模型（交强险+人身险捆绑），独立盈亏线 |
| 其他 | `/diagnose-agent`（本技能） | 商业险+交强险，可用定价系数调节 |

**检测到"摩托车"关键词时，必须转到 `/diagnose-motorcycle`，不在本技能内处理。**

## Pre-flight 确认协议（RED LINE）

**每次运行前必须确认以下 4 项**，模糊时主动让用户选择：

| # | 确认项 | 说明 | 默认 |
|---|--------|------|------|
| 1 | **分析对象** | 机构？经代？车型？客户类别？组合筛选？ | 必须明确 |
| 2 | **年份范围** | 哪几年？含不含 2021？ | 最近 3 年 |
| 3 | **对比口径** | 同期对比(ytd) 还是全年(full)？最新年不完整时必须问 | ytd（推荐） |
| 4 | **输出目的** | 看全貌？聚焦某个问题？给谁看？ | 全貌 |

**场景判定**：用户需求 → 脚本选择

```
"XX经代/代理公司"        → diagnose_agent.py
"XX机构"                 → diagnose_vehicle.py --filter "三级机构 = 'XX'"
"XX车型/厂牌"            → diagnose_vehicle.py --filter "厂牌车型 LIKE '%XX%'"
"XX客户类别"             → diagnose_vehicle.py --filter "客户类别 = 'XX'"
"XX客户类别+新能源"      → diagnose_vehicle.py --filter "客户类别 IN (...) AND 是否新能源 = TRUE"
"对比 A 和 B"            → 分别运行两次
```

## Phase 1: 聚合引擎（自动执行）

确认筛选条件后，执行脚本产出 Markdown + 结构化 JSON：

```bash
python3 数据管理/pipelines/diagnose_vehicle.py \
  --filter "{WHERE条件}" \
  --title "{标题}" \
  --years {起始年}-{结束年} \
  --compare {ytd|full} \
  --output-json
```

**产出文件**：
- `{title}_经营诊断_{years}_截至{date}.md` — 11 板块完整报告（附录）
- `{title}_诊断_{years}_截至{date}.json` — 分层结构化数据

**JSON 分层**（Phase 2 首轮只读 L0+L1+findings，< 500 token）：
- `L0_identity`：规模身份（总保费、保单数、主导车型）
- `L1_health`：最新年健康指标 + 趋势方向 + 亮灯预警
- `L2_deep`：按维度拆分的 KPI（新转续/风险评分/能源/年度）
- `findings`：规则引擎自动发现（结构化 level/tag/text）
- `next_steps`：建议下一步

## Phase 2: 诊断研判（Claude 执行）

**读取 JSON 后，按以下清单执行诊断**：

### Step 1: 定性判断（读 L0 + L1）
用一句话回答"赚不赚？"

**承保品质（变动成本率）：**
- L1.latest_year.variable_cost_rate < 85% → 健康盈利
- 85-94% → 微利，需关注
- > 94% → 亏损

**真实盈亏（综合成本率，含固定成本）：**
- L1.latest_year.combined_cost_ratio ≤ 99% → 盈利
- 99-105% → 微亏
- > 105% → 严重亏损
- L1.latest_year.profit_amount_wan → 利润额绝对值（万元）

### Step 2: 归因排序（读 L2 + findings）
找到亏损/异常的 top 2-3 个维度：
- L2.by_vehicle_type 中哪个类型 variable_cost_rate 最高？
- L2.by_risk_grade 中哪个等级亏损最严重？
- findings 中 🔴 级别的发现有哪些？

### Step 3: 交叉验证（判断是否需要追加下钻）
对 top 归因，检查是否有充分佐证：
- **充分** → 直接出结论
- **不足** → 调用 `--drilldown` 追加聚合（≤2 轮 × 3 个维度）

```bash
# 追加下钻示例
python3 数据管理/pipelines/diagnose_vehicle.py \
  --filter "{同样的WHERE}" \
  --drilldown "厂牌车型" \
  --years {years} --compare {compare} --output-json
```

**可用下钻维度**：`厂牌车型`、`三级机构`、`经代名`、`业务员`、`客户类别`、`险别组合`

### Step 4: 输出诊断报告

**格式：SCQA 结构（情境→冲突→问题→回答）**

```markdown
## 诊断结论

{1 段话，直接回答"赚不赚、亏在哪、要不要继续"}

## 核心病灶

1. **{病灶名}**：{证据描述} [来源: L2.by_vehicle_type]
2. **{病灶名}**：{证据描述} [来源: L2.by_risk_grade]

## 行动建议

### 立即行动
- {建议1}

### 短期（1-3月）
- {建议2}

### 持续监控
- {建议3}

---

> 📎 完整数据明细：{Markdown 报告文件名}
> 📎 结构化数据：{JSON 文件名}
```

### 反事实检查（防止确认偏差）

在出结论前，自问：
- "如果我的归因是错的，还有什么其他解释？"
- "是否有被我忽略的维度可能翻转结论？"
- 若存在合理反事实 → 追加一轮 --drilldown 验证

## Phase 3: 多维毒性分析（按需触发）

当用户要求"找出保费占比低但赔付影响度高的板块"或"多维度分析"时触发。

### 毒性指数公式

```
毒性指数 = 赔付贡献度% / 保费占比%
```

- **毒性 > 1.3**：保费贡献低但赔付贡献高 → 毒瘤板块
- **毒性 0.8-1.3**：正常比例
- **毒性 < 0.8**：保费贡献高但赔付贡献低 → 盈利池

### 标准维度（9 维）

用户可指定任意子集；默认用户要求"全面分析"时跑全部 9 维：

| # | 维度 | 字段/表达式 | 分组策略 |
|---|------|------------|----------|
| 1 | 被保险人年龄 | `被保险人年龄分组` | 原值 |
| 2 | 车牌归属地 | `LEFT(车牌号码, 2)` | 映射城市名 |
| 3 | 车型 | `厂牌车型` | 按排量/类型分组(110/125/150/200+/电动/三轮/轻便) |
| 4 | 新旧车 | 新转续过户 | 4类 |
| 5 | 车价 | `新车购置价` | 分段(≤3千/3-5千/5-8千/0.8-1.5万/1.5-3万/3万+) |
| 6 | 车龄 | `初次登记年月` | 分段(0-1/2-3/4-5/6-8/9-12/12+) |
| 7 | 业务员 | `业务员` | 原值 |
| 8 | 经代 | `经代名` | 原值 |
| 9 | 能源 | `是否新能源` | 2类 |

### 执行方式

可复用项目文件夹下的 `analyze_dimensions.py` 脚本，或直接 DuckDB 查询。每个维度产出 `dim_{维度名}.json`，最终汇总为：
- `toxic_segments.json` — 毒性>1.3 且赔付率>120% 的板块
- `profitable_segments.json` — 毒性<0.8 且赔付率<100% 的板块

### 输出格式

Phase 3 报告附在 Phase 2 诊断报告之后，结构：

```markdown
## 多维毒性分析

### 🔴 毒瘤板块（保费小 + 赔付大）
| 排名 | 维度 | 值 | 保费占比 | 赔付贡献 | 毒性指数 | 赔付率 |
...

### 🟢 盈利池（保费贡献 > 赔付贡献）
| 排名 | 维度 | 值 | 保费占比 | 赔付贡献 | 毒性指数 | 赔付率 |
...

### 交叉定位
将多个维度的毒瘤特征叠加，定位最精准的毒瘤画像。
```

## 参数说明

### diagnose_vehicle.py

| 参数 | 说明 |
|------|------|
| `--filter` | SQL WHERE 条件（必填） |
| `--title` | 报告标题 |
| `--years` | 年份范围，如: 2024-2026 |
| `--compare` | `ytd`=同期对比，`full`=全年对比 |
| `--output-json` | 额外输出结构化 JSON（Phase 2 必需） |
| `--drilldown <列名>` | 追加聚合模式：按指定列 GROUP BY，仅输出 JSON |
| `--no-summary` | 跳过 Markdown 中的诊断总结（JSON 仍产出） |
| `--sections` | 仅运行指定板块，如: 1,5,9 |
| `--skip` | 跳过指定板块，如: 3,4 |
| `--output` | 输出目录（默认 数据分析报告/） |

### diagnose_agent.py（经代公司专用）

| 参数 | 说明 |
|------|------|
| `--org` | 三级机构名称（必填） |
| `--agent` | 经代公司名称（必填，支持模糊匹配） |
| `--years` | 分析年份列表（默认 2025 2026） |
| `--compare` | `ytd`=同期对比，`full`=全年对比 |

## 指标体系

| 指标 | 公式 |
|------|------|
| 满期边际贡献额 | 满期保费 × (1 - 赔付率 - 费用率) |
| 预估边际贡献额 | 签单保费 × (1 - 赔付率 - 费用率) |
| 变动成本率 | 满期赔付率 + 费用率 |
| 边际贡献率 | 100% - 变动成本率 |
| 满期赔付率 | 已报告赔款 / 满期保费 |
| 费用率 | 费用金额 / 签单保费 |
| 满期出险率 | 年化赔案 / 保单数 × 100 |
| 案均赔款 | 已报告赔款 / 赔案件数 |
| 商车定价系数 | AVG(商车自主定价系数)，仅商业险 |
| **固定成本额** | 附加税费额 + 推动费额 + 管理费额（绝对值聚合） |
| **综合成本率** | 综合成本额 / 满期保费（绝对值除法，非率值相加） |
| **利润额** | 满期保费 - 综合成本额 = 边际贡献额 - 固定成本额 |

> 固定成本参数来自 `数据管理/config/fixed-cost-params.json`，诊断脚本自动计算。

## 亮灯规则

| 指标 | 🟢正常 | 🔵关注 | 🟡预警 | 🔴危险 |
|------|--------|--------|--------|--------|
| 变动成本率 | ≤85% | 85-91% | 91-94% | >94% |
| 边际贡献率 | ≥15% | 9-15% | 6-9% | <6% |
| **综合成本率** | **≤99%** | **99-101%** | **101-105%** | **>105%** |
| 满期赔付率 | ≤60% | 60-70% | 70-75% | >75% |
| 满期出险率 | ≤8% | 8-10% | 10-12% | >12% |

## 口径选择指南

| 场景 | 推荐口径 | 原因 |
|------|---------|------|
| 看增长趋势、YoY 变化 | **同期(ytd)** | 避免全年 vs 季度的不可比 |
| 看绝对规模、利润总额 | **全年(full)** | 保费/赔款等绝对值更完整 |
| 最新年 ≥11月 | **全年** | 数据基本完整 |
| 最新年 ≤Q1 | **同期** | 必须同期 |
