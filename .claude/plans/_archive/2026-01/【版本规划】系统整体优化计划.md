**目标**: 减少至 ~300 行

```

├─ 状态管理 (~100 行) → useKpiData Hook
├─ 数据获取 (~200 行) → useTrendData Hook
├─ 筛选器逻辑 (~100 行) → useFilterState Hook
├─ KPI 渲染 (~150 行) → KpiSection 组件
├─ 趋势图渲染 (~200 行) → TrendSection 组件
└─ 表格渲染 (~177 行) → TableSection 组件

重构后结构 (~300 行)
├─ PremiumDashboard.tsx (主组件，组装各部分)
├─ hooks/
│   ├─ useKpiData.ts
│   ├─ useTrendData.ts
│   └─ useFilterState.ts
└─ components/
    ├─ KpiSection.tsx
    ├─ TrendSection.tsx
    └─ TableSection.tsx
```

**具体步骤**:

   - 返回: `{ filters, updateFilter, resetFilters }`

4. 创建 `components/KpiSection.tsx`
   - 纯展示组件
   - Props: `{ kpiData, loading }`
   - 使用 `EnhancedKpiCard`

5. 创建 `components/TrendSection.tsx`
   - 纯展示组件

6. 创建 `components/TableSection.tsx`

7. 重构 `PremiumDashboard.tsx` 主组件

##### 3.1 日志迁移（替换 186 个 console 调用）

**策略**: 渐进式迁移，按模块逐步替换

**优先级排序**:
```
P1 (高优先级，~80 个)
├─ src/shared/duckdb/ (Worker 通信日志)
├─ src/services/ (数据服务日志)
└─ src/features/dashboard/ (主要功能日志)

P2 (中优先级，~60 个)
├─ src/charts/ (图表渲染日志)
└─ src/features/sql-query/ (SQL 查询日志)

P3 (低优先级，~46 个)
├─ src/widgets/ (小部件日志)
└─ 其他零散日志
```

**替换步骤**:
1. 按模块创建专用 logger
   ```typescript
   // 示例
   import { createLogger } from '@/shared/utils/logger';
   const logger = createLogger('DuckDBClient');

   // 替换
   console.log('数据加载成功') → logger.info('数据加载成功')
   console.error('查询失败', err) → logger.error('查询失败', err)
   ```

2. 每完成一个模块，运行测试确保正常

3. 更新文档，记录已迁移的模块

**验证标准**:
- ✅ 所有 console 调用已替换
- ✅ 日志输出正常
- ✅ 生产环境 debug 日志已过滤

##### 3.2 类型迁移（替换 79 个 any 类型）

**策略**: 按影响范围从大到小逐步替换

**优先级排序**:
```
P1 (高影响，~40 个)
├─ src/features/dashboard/PremiumDashboard.tsx
│   └─ kpis: any → KpiData
│   └─ trendData: any[] → TrendDataPoint[]
│   └─ tableData: any[] → TableDataRow[]
├─ src/features/dashboard/Dashboard.tsx
└─ src/services/DataService.ts

P2 (中影响，~25 个)
├─ src/widgets/charts/*.tsx
└─ src/features/sql-query/*.tsx

P3 (低影响，~14 个)
└─ 其他零散 any 类型
```

**替换步骤**:
1. 按优先级逐个文件处理
   ```typescript
   // 示例
   const data: any[] = result.toArray();
   ↓
   const data: TrendDataPoint[] = result.toArray().map(row => ({
     time_period: row.time_period,
     total_premium: row.total_premium,
     // ...
   }));
   ```

2. 每完成一个文件，运行 TypeScript 类型检查

3. 更新测试，确保类型安全

**验证标准**:
- ✅ TypeScript 编译无错误
- ✅ 所有测试通过
- ✅ IDE 类型提示完整

##### 3.3 性能优化

**优化点清单**:

1. **React 渲染优化**
   ```typescript
   // 大型组件使用 memo
   export const ExpensiveComponent = React.memo(({ data }) => {
     // ...
   });

   // 复杂计算使用 useMemo
   const processedData = useMemo(() => {
     return heavyComputation(data);
   }, [data]);

   // 回调函数使用 useCallback
   const handleClick = useCallback(() => {
     // ...
   }, [dependency]);
   ```

2. **代码分割和懒加载**
   ```typescript
   // 大型组件延迟加载
   const TruckAnalysisPanel = lazy(() => import('./TruckAnalysisPanel'));
   const RenewalAnalysisPanel = lazy(() => import('./RenewalAnalysisPanel'));
   ```

3. **数据获取优化**
   - 避免重复查询
   - 使用缓存机制
   - 批量请求合并

4. **图表渲染优化**
   - 大数据量时使用采样
   - 延迟初始化
   - 按需加载 ECharts 模块

**验证标准**:
- ✅ Lighthouse 性能分数 > 90
- ✅ 首屏渲染时间 < 2s
- ✅ 交互响应时间 < 100ms
