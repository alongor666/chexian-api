## 🚨 关键问题修复（用户反馈）

### 问题1：maxDataDate 不随日期口径更新

**发现的问题**：
```typescript
// ❌ 当前计划的逻辑（有BUG）
const loadDataMetadata = async () => {
  const [maxPolicyDate, maxStartDate, years] = await Promise.all([
    duckdbClient.getMaxDataDate('policy_date'),
    duckdbClient.getMaxDataDate('insurance_start_date'),
    duckdbClient.getAvailableYears('policy_date'),  // ❌ 只查签单日期
  ]);

  const currentCriteria = filters.dateCriteria || 'policy_date';
  const resolvedMaxDate = currentCriteria === 'insurance_start_date'
    ? maxStartDate
    : maxPolicyDate;

  setMaxDataDate(resolvedMaxDate);  // ❌ 只设置一次，切换口径时不更新
};
```

**问题场景**：
1. 用户上传数据，默认选择"签单日期"
2. `maxDataDate` = 签单数据最大日期（如 `2026-12-15`）
3. 用户切换到"起保日期"
4. ❌ BUG：`maxDataDate` 仍然是 `2026-12-15`（签单日期的最大值）
5. ❌ 结果：用户看到的起保日期截止日期是错误的

---

### 问题2：availableYears 不随日期口径更新

**发现的问题**：
```typescript
// ❌ 当前计划的逻辑（有BUG）
const years = await duckdbClient.getAvailableYears('policy_date');  // ❌ 只查签单日期
setAvailableYears(years);
```

**问题场景**：
- 数据中：
  - 签单日期年份：[2024, 2025, 2026]
  - 起保日期年份：[2025, 2026, 2027]
- 用户切换到"起保日期"时
- ❌ BUG：年度选项仍然显示 [2024, 2025, 2026]（签单日期的年份）
- ❌ 结果：用户无法选择2027年的起保数据

---

### 问题3：元数据刷新策略未闭环

**根本原因**：
- 元数据查询只在数据加载时执行一次
- 切换日期口径时，`maxDataDate` 和 `availableYears` 没有重新计算
- 导致UI显示的日期范围与实际数据口径不匹配

---

## ✅ 修复方案：双口径缓存 + 动态切换

### 方案设计

**核心思路**：
1. **一次性查询两个口径的元数据**（避免重复查询）
2. **缓存两个口径的元数据对象**
3. **根据当前 `dateCriteria` 动态选择对应的元数据**

#### Step 1.2: 新增DuckDB元数据查询方法
**文件**: `src/shared/duckdb/client.ts`

```typescript
class DuckDBClient {
  /**
   * 查询指定日期字段的元数据（最大日期 + 可用年份）
   * @param dateField - 日期字段名（'policy_date' | 'insurance_start_date'）
   * @returns 该日期字段的元数据
   */
  async getDateMetadata(dateField: string): Promise<DateMetadata> {
    const [maxDateResult, yearsResult] = await Promise.all([
      // 查询最大日期
      this.query(`
        SELECT MAX(CAST(${dateField} AS DATE)) AS max_date
        FROM PolicyFact
        WHERE ${dateField} IS NOT NULL
      `).promise,

      // 查询可用年份
      this.query(`
        SELECT DISTINCT YEAR(CAST(${dateField} AS DATE)) AS year
        FROM PolicyFact
        WHERE ${dateField} IS NOT NULL
        ORDER BY year DESC
      `).promise,
    ]);

    const maxDateRows = maxDateResult.toArray();
    const yearsRows = yearsResult.toArray();

    const maxDate = maxDateRows[0]?.max_date || new Date().toISOString().split('T')[0];
    const availableYears = yearsRows.map((row) => Number(row.year));

    return { maxDate, availableYears };
  }

  /**
   * 查询双口径元数据（并行查询，一次性返回）
   * @returns 双口径元数据缓存
   */
  async getDualDateMetadata(): Promise<DualDateMetadata> {
    const [policyMetadata, insuranceMetadata] = await Promise.all([
      this.getDateMetadata('policy_date'),
      this.getDateMetadata('insurance_start_date'),
    ]);

    return {
      policy: policyMetadata,
      insurance: insuranceMetadata,
    };
  }

  /**
   * @deprecated 使用 getDateMetadata 代替
   * 查询数据中日期字段的最大值
   */
  async getMaxDataDate(dateField: string = 'policy_date'): Promise<string> {
    const metadata = await this.getDateMetadata(dateField);
    return metadata.maxDate;
  }

  /**
   * @deprecated 使用 getDateMetadata 代替
   * 查询数据中存在的年份列表（降序）
   */
  async getAvailableYears(dateField: string = 'policy_date'): Promise<number[]> {
    const metadata = await this.getDateMetadata(dateField);
    return metadata.availableYears;
  }
}
```

export function buildWhereClauseFromFilters(
  filters: AdvancedFilterState,
  dateField?: string
): string {
  const conditions: string[] = ['1=1'];

  // 推断日期字段（优先级：参数 > filters.dateCriteria > 默认值）
  const resolvedDateField =
    dateField || filters.dateCriteria || 'policy_date';

  if (filters.policy_date_start) {
    conditions.push(
      `${resolvedDateField} >= '${sanitizeDate(filters.policy_date_start)}'`
    );
  }
  if (filters.policy_date_end) {
    conditions.push(
      `${resolvedDateField} <= '${sanitizeDate(filters.policy_date_end)}'`
    );
  }

  // ...其他逻辑保持不变
  return conditions.join(' AND ');
}
