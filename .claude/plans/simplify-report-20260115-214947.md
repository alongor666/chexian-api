## 详细分析

### 1. useCoefficientMonitor.ts (935行) - 最高优先级

**问题诊断**:
- 77个分支语句（if/else/switch），圈复杂度极高
- 单一 Hook 承担过多职责（数据获取、状态管理、计算逻辑）
- 内部存在3处21行的重复代码

**重构方案**:

```typescript
// ❌ 当前：单一巨型 Hook
export function useCoefficientMonitor(options) {
  // 935行代码...
}

// ✅ 建议：拆分为多个职责单一的 Hook
export function useCoefficientMonitor(options) {
  const queries = useCoefficientQueries(options);      // 查询构建
  const data = useCoefficientData(queries);            // 数据获取
  const compliance = useCoefficientCompliance(data);   // 合规计算
  return { ...data, ...compliance };
}
```

**预计收益**: 减少 400+ 行，复杂度降低 60%
