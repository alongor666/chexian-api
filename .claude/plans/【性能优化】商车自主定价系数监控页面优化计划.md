# 商车自主定价系数监控页面优化计划

## 概述

优化商车自主定价系数监控页面，提升查询性能、修正计算逻辑、新增缺口保费列、拆分成4个周期子表。

**预期性能提升**：5-10倍（将N+1查询优化为单次UNION ALL查询）

---

## 核心变更点

### 1. 性能优化
- **问题**：`useCoefficientMonitor.ts:273-322` 为每条数据单独执行周查询（N+1问题）
- **解决**：新增SQL批量查询函数，使用UNION ALL合并为单次查询

### 2. 计算逻辑修正
- **当周与阈值比**：从 `当周系数 / 监管阈值` 改为 `当周系数 - 监管阈值`
- **新增缺口保费列**：`(监管阈值 - 当周系数) × 当周保费`

### 3. UI拆分
- 将单表拆分为4个垂直堆叠的子表：1-7日、8-14日、15-21日、22-月末
- 仅显示有数据的周期，无数据则提示

### Phase 1: SQL层优化（核心性能优化）

#### 文件：`src/shared/sql/coefficient.ts`

**新增函数**：`generateWeekBatchQuery()`

```typescript
/**
 * 生成所有周数据的UNION ALL批量查询（性能优化版）
 *
 * 优化点：
 * 1. 将N+1查询合并为单次UNION ALL查询
 * 2. 在SQL层直接按周期分组（1-7日、8-14日、15-21日、22-月末）
 * 3. 返回周期名称字段供应用层分组
 */
export function generateWeekBatchQuery(
  dateField: string,
  cutoffDate: Date,
  additionalWhere: string = '1=1'
): string {
  const year = cutoffDate.getFullYear();
  const month = cutoffDate.getMonth();
  const lastDay = getLastDayOfMonth(year, month);

  // 定义4个周期
  const periods = [
    { name: '1-7日', start: 1, end: 7 },
    { name: '8-14日', start: 8, end: 14 },
    { name: '15-21日', start: 15, end: 21 },
    { name: `22-${lastDay}日`, start: 22, end: lastDay }
  ];

  // 生成所有UNION子查询（成都、全省、各机构）
  const unionParts: string[] = [];

  // 辅助函数：生成单个周期的查询
  const generatePeriodQuery = (periodName: string, startDate: Date, endDate: Date, orgFilter: string = '') => {
    return `
      SELECT
        '${periodName}' AS period_name,
        ${orgFilter ? `'${orgFilter}' AS org_level_3,` : 'org_level_3,'}
        ${orgFilter ? `'${orgFilter === '成都' ? 'chengdu' : 'province'}' AS region_group,` : `${generateOrgGroupCase()} AS region_group,`}
        is_nev,
        CASE WHEN ${NON_COMMERCIAL_PERSONAL_CONDITION}
             THEN 'non_commercial_personal'
             ELSE 'all'
        END AS customer_category_group,
        is_new_car,
        SUM(premium) AS week_premium,
        SUM(premium) / NULLIF(SUM(premium / commercial_pricing_factor), 0) AS week_factor,
        COUNT(DISTINCT policy_no) AS week_count
      FROM PolicyFact
      WHERE ${COMMERCIAL_BASE_CONDITION}
        AND ${dateField} >= '${formatDate(startDate)}'
        AND ${dateField} <= '${formatDate(endDate)}'
        ${orgFilter === '成都' ? `AND (${ORG_GROUPS.SAME_CITY.map(c => `org_level_3 LIKE '%${c}%'`).join(' OR ')})` : ''}
        ${orgFilter === '全省' ? '' : `${orgFilter ? `AND org_level_3 = '${orgFilter}'` : ''}`}
        AND ${additionalWhere}
      ${orgFilter ? 'GROUP BY is_nev, CASE WHEN ${NON_COMMERCIAL_PERSONAL_CONDITION} THEN \'non_commercial_personal\' ELSE \'all\' END, is_new_car' : 'GROUP BY org_level_3, is_nev, CASE WHEN ${NON_COMMERCIAL_PERSONAL_CONDITION} THEN \'non_commercial_personal\' ELSE \'all\' END, is_new_car'}
    `;
  };

  // 为每个周期生成3类查询（成都、全省、各机构）
  for (const period of periods) {
    const startDate = new Date(year, month, period.start);
    const endDate = new Date(year, month, period.end);

    // 成都聚合
    unionParts.push(generatePeriodQuery(period.name, startDate, endDate, '成都'));

    // 全省聚合
    unionParts.push(generatePeriodQuery(period.name, startDate, endDate, '全省'));
  }

  // 机构明细查询（需要遍历实际机构）
  // 注意：这部分在fetchData中动态生成，因为需要获取实际机构列表

  return `
    WITH week_data AS (
      ${unionParts.join('\n      UNION ALL\n      ')}
    )
    SELECT * FROM week_data
    ORDER BY
      CASE org_level_3
        WHEN '成都' THEN 1
        WHEN '全省' THEN 2
        ELSE 3
      END,
      period_name,
      org_level_3,
      is_nev DESC,
      customer_category_group,
      is_new_car
  `;
}
```

**2.2 重构 fetchData() 函数**

**关键变更**：
- 删除第273-322行的N+1查询循环
- 使用单次UNION ALL查询获取所有周数据
- 按周期分组数据（1-7日、8-14日、15-21日、22-月末）

**伪代码结构**：
```typescript
// 初始化4个周期的Map
const periods = [
  { name: '1-7日', start: 1, end: 7 },
  { name: '8-14日', start: 8, end: 14 },
  { name: '15-21日', start: 15, end: 21 },
  { name: '22-月末', start: 22, end: lastDay }
];

const resultsByPeriod = new Map<string, Map<string, Partial<CoefficientRow>>>();
periods.forEach(p => resultsByPeriod.set(p.name, new Map()));

// 阶段1：查询当天、当月、当年数据（保持原逻辑，但填充到所有周期）
for (const { range, period } of queries) {
  // 查询并填充到每个周期的Map
  for (const [, periodMap] of resultsByPeriod.entries()) {
    // 填充 day/month/year 数据
  }
}

// 阶段2：单次UNION ALL查询获取所有周数据（性能优化核心）
const weekSql = generateWeekBatchQuery(dateField, cutoffDate, additionalWhere);
const weekTable = await duckdbClient.query(weekSql).promise;
const weekRows = weekTable.toArray();

// 按周期分组填充周数据
for (const weekRow of weekRows) {
  const periodName = weekRow.period_name;
  const periodMap = resultsByPeriod.get(periodName);
  // 填充 weekFactor, weekPremium, weekCount
}

// 阶段3：处理每个周期的数据（计算阈值、缺口保费等）
const periodGroupsData: PeriodGroupData[] = [];
const finalData: CoefficientRow[] = [];

for (const [periodName, periodMap] of resultsByPeriod.entries()) {
  const periodRows: CoefficientRow[] = [];

  for (const [, row] of periodMap.entries()) {
    // 获取阈值规则
    const thresholdRule = getThresholdRule({...});

    // 计算合规状态
    let compliant: boolean | null = null;
    let thresholdRatio: number | null = null;
    let gapPremium: number | null = null;

    if (thresholdRule && row.weekFactor !== null) {
      compliant = isCompliant(row.weekFactor, thresholdRule);
      // 修改计算逻辑：当周系数 - 监管阈值
      thresholdRatio = row.weekFactor - thresholdRule.threshold;
      // 新增：计算缺口保费
      gapPremium = (thresholdRule.threshold - row.weekFactor) * (row.weekPremium ?? 0);
    }

    // 构造 CoefficientRow
    const coefficientRow: CoefficientRow = {
      // ... 填充所有字段 ...
      periodName: periodName,
      gapPremium: gapPremium,
    };

    periodRows.push(coefficientRow);
    finalData.push(coefficientRow);
  }

  // 排序
  periodRows.sort(...);

  // 计算周期日期范围
  const [startDate, endDate] = calculatePeriodRange(periodName, year, month);

  periodGroupsData.push({
    periodName,
    startDate,
    endDate,
    hasData: periodRows.length > 0,
    rows: periodRows,
  });
}

setData(finalData);
setPeriodGroups(periodGroupsData);
```

**2.3 新增辅助函数**

```typescript
/**
 * 计算周期的日期范围
 */
function calculatePeriodRange(periodName: string, year: number, month: number): [Date, Date] {
  const lastDay = getLastDayOfMonth(year, month);

  if (periodName === '1-7日') {
    return [new Date(year, month, 1), new Date(year, month, 7)];
  } else if (periodName === '8-14日') {
    return [new Date(year, month, 8), new Date(year, month, 14)];
  } else if (periodName === '15-21日') {
    return [new Date(year, month, 15), new Date(year, month, 21)];
  } else {
    return [new Date(year, month, 22), new Date(year, month, lastDay)];
  }
}
```

### Phase 3: 计算逻辑修正

#### 文件：`src/shared/config/coefficient-thresholds.ts`

**3.1 修改 calculateThresholdRatio() 函数（第269-271行）**

```typescript
/**
 * 计算当周与阈值比（修改为减法）
 *
 * @param weeklyFactor 当周系数
 * @param rule 阈值规则
 * @returns 差值（当周系数 - 监管阈值）
 */
export function calculateThresholdRatio(weeklyFactor: number, rule: ThresholdRule): number {
  // 修改前：return weeklyFactor / rule.threshold;
  // 修改后：
  return weeklyFactor - rule.threshold;
}
```

**3.2 新增 calculateGapPremium() 函数**

```typescript
/**
 * 计算缺口保费
 *
 * @param weeklyFactor 当周系数
 * @param weeklyPremium 当周保费
 * @param rule 阈值规则
 * @returns 缺口保费（(监管阈值 - 当周系数) × 当周保费）
 */
export function calculateGapPremium(
  weeklyFactor: number,
  weeklyPremium: number,
  rule: ThresholdRule
): number {
  return (rule.threshold - weeklyFactor) * weeklyPremium;
}
```
