报价数据的"是否报价"字段口径有问题：当前源 xlsx 中用"是/否"标记，但正确逻辑应该是**有上年保单号（续保单号）才算已报价**。

**Why:** 报价清单的本质是"为哪些上年保单发起了续保报价"，判定依据是续保单号是否存在，而非一个简单的布尔标记。当前的"是/否"标记不可靠。

**How to apply:**
- Quotes 域的拆分逻辑（split_existing.py / transform.py --domain quotes）目前基于 `是否报价=True` 过滤，待口径修正后需同步调整
- 这是**用户亲自处理的事项**，涉及源数据层面的修正，AI 无法替代
- 修正前不要基于当前报价数据做业务结论
