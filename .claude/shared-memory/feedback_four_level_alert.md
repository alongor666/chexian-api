诊断报告亮灯从三级升级为四级：

| 级别 | 符号 | 含义 |
|------|------|------|
| 🟢 | 正常 | 指标健康 |
| 🔵 | 关注 | 接近预警，需关注趋势 |
| 🟡 | 预警 | 超预警线，需干预 |
| 🔴 | 危险 | 超危险线，紧急 |

**阈值配置**（关注/预警/危险）：
- 变动成本率：85% / 91% / 94%
- 边际贡献率：15% / 9% / 6%（越低越差）
- 满期赔付率：60% / 70% / 75%
- 满期出险率：8% / 10% / 12%
- 案均赔款(货车)：8,000 / 10,000 / 12,000 元

**Why:** 三级体系（正常/预警/危险）粒度不够，无法区分"接近预警但尚可"和"远离预警很安全"。四级体系让管理层更精确判断优先级。

**How to apply:** diagnose_vehicle.py 已使用四级。diagnose_agent.py 仍是旧三级（下次迭代统一）。前端如需亮灯也应使用此四级体系。
