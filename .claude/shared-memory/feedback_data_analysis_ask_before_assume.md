---
name: 数据分析需求必须精准确认
description: 所有数据分析技能/agent在需求模糊时必须先让用户选择，不能自行假设口径
type: feedback
---

数据分析技能和 agent 必须智能识别用户需求，模糊时主动让用户做选择，不能自行假设。

**Why:** 用户发现诊断报告用全年 vs Q1 数据做 YoY 增长率对比，口径不一致导致数据失真。AI 自动选择了一个口径但没有提示用户，浪费了整轮分析。

**How to apply:**
- 涉及时间口径（全年 vs 同期）、指标定义、筛选条件等可能有多种合理选择的场景，必须让用户选择
- CLI 工具：加交互提示或参数选项（如 `--compare ytd|full`）
- Agent/skill：在开始分析前先确认关键假设
- 示例场景：YoY 用同期还是全年？赔付率用满期还是签单口径？保单数用去重还是不去重？
- 如果用户已明确指定，直接执行不再确认
