四分类判定逻辑（SQL CASE WHEN 顺序敏感）：

```sql
CASE
    WHEN 是否新车 THEN '新车'
    WHEN 是否过户车 THEN '旧车过户'
    WHEN 是否续保 THEN '旧车续保'
    ELSE '旧车转保'          -- 非新车、非过户、非续保 = 转保
END
```

**判定优先级**：新车 > 过户 > 续保 > 转保（兜底）

**牵引车实测分布**：
- 旧车转保：10,833 件（60.4%）← 主体
- 新车：4,220 件（23.5%）
- 旧车续保：1,843 件（10.3%）
- 旧车过户：1,027 件（5.7%）

**Why:** 新转续过户是车险经营分析的核心维度，不同类型的赔付率和费用率差异巨大（旧车转保赔付率 101% vs 旧车续保 41%）。

**How to apply:** diagnose_vehicle.py 板块 2 使用此分类。注意"旧车转保"是兜底类，可能含少量未标记的数据。
