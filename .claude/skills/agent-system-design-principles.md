# Agent 系统设计第一性原理（Agent System Design Principles）

> 来源：拆解 super-powers-team-suite（DM/DI/DG）的设计本质后抽象出的母原理。目的不是抄那套框架，而是**从公理推导出适合本场景的设计**。
>
> 语义独立性：纯通用工程思想，抹掉 chexian/车险仍成立（过 rule-promotion-gate ✅），可上提共享仓；§4 是本项目落地映射。

## 0. 根公理：所有设计都在对冲"LLM 是什么"

LLM = **高能力 / 高方差 / 无状态 / 谄媚 / 上下文受限** 的推理器。每个形容词是一类必然失效，设计就是对冲单：

| LLM 本性 | 必然失效 | 对冲 |
|---------|---------|------|
| 高方差 | 同输入时对时错 | 质量门 + 强制验证 |
| 无状态 | 跨步骤遗忘、上下文漂移 | 状态外置（写盘，不留对话）|
| 谄媚 | 替自己的错找理由、声称"应该通过" | 独立验证 + 证据优先 |
| 上下文受限 | 任务一大就涣散、越界 | 分解 + 角色隔离 + 最小工作集 |

**FP0：不要相信这个组件，要围着它的已知失效模式做工程。** 纪律下在「流程/产物」上，不下在「提示词更努力」上——这是分水岭。

## 1. 所以然：这是"用不可靠节点造可靠系统"的老学问换载体

- **容错分布式系统**：状态外置 + 日志 + 检查点 + 单一调度 → 黑板的 `INDEX+gen+inbox` 是微型带日志内核；"只有 coordinator 派发" = 强制星型拓扑防递归爆炸。
- **丰田生产方式/精益**：标准作业 + 工位检验 + 检查清单 + **安灯绳** + 检验与生产分离 + 书面工单 + 工长派工。
- 对齐：`stop-on-mismatch`=安灯绳 · 质量门=工位检验 · 黑板=工单 · coordinator=工长 · 前后两扫=来料+出厂检验 · 独立验证=检验员≠生产工。
- **取经方向**：设计 agent 系统时，去这两门成熟学科取经，别重新发明。

## 2. 六条派生原理（附经典原型，便于识别"非新东西"）

| # | 原理 | 一句话 | 原型 |
|---|------|--------|------|
| ① | 状态外置 | 上下文是负债不是记忆；结论后续要用就落盘 | OS 内存/磁盘分离、事件溯源 |
| ② | 分解+角色隔离 | 给每个 agent 窄而深的高信噪比工作集；禁互调、单一控制点 | Actor 模型、康威定律、内核/用户态 |
| ③ | 独立对抗验证 | 信任是制造的：检查者不继承制造者假设、只认证据 | 三权分立、code review、TDD |
| ④ | 有界授权+偏差浮现 | 装断路器，让矛盾变可见信号而非被悄悄吸收 | 安灯绳、保险丝、fail-fast |
| ⑤ | 投入配风险 | 模型没分寸感，分寸外部编码（≤3行签收 vs 完整ADR）；最小爆炸半径 | 基于风险的测试、最小变更 |
| ⑥ | 门=强制 | 被"鼓励"的质量必被跳过，成"门"的才被执行 | CI/CD、Definition of Done |

## 3. 生成式配方（怎么推导自己的设计，而非套框架）

1. **先列你场景里 LLM 的具体失效模式**（如车险：口径幻觉、把"查询失败"当"无数据"、跨步骤忘记已定维度）。
2. **每个失效配一个对冲**——从 §2 六条里**选**，不是全上。
3. **机制落在流程/产物**，不落在"请你仔细一点"。
4. 验收：能从公理推导出设计 = 内化；只能照抄 DM/DI/DG = 没内化。

## 4. 最高原理：可靠性强度匹配失效代价（否则成新教条）

这些机制不是免费的，是保险，**保费 = 文件 I/O + 派发开销 + 门的延迟与 token**。对小任务上全套 = 过度工程（讽刺地违反 simplicity-first）。

> **FP-top：可靠性机制的强度应匹配失效的代价。** 失效便宜处（改文案）用轻流程；失效昂贵处（核心赔付率口径、生产部署链）才上重机制。不为存配置跑 Paxos，不为 bugfix 过 8 道门。

**本项目映射**：`deploy-chain-sop`（失效昂贵）重、数据分析 skill 轻——已在直觉里用这条，本 skill 把它显式化为可复用推导起点。

## 5. 关联

- 本套原理的具体落地件：`dev-stop-on-mismatch`(④) · `silent-failure-guard`(③证据观) · `rule-promotion-gate`(③独立验证) · `adr-tiered-response`(⑤) · `code-search-routing`(⑤按问题选刀)。
- 评估"要不要引入某 agent 框架"时：先用 §0/§4 判它对冲的失效模式在你场景是否昂贵到值回保费。
