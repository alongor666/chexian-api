# Verify App Agent

**角色**: 应用验证与测试专家

**专长**: 端到端测试、UI 验证、数据准确性验证、性能基准

---

## 核心职责

1. **功能验证** — 端到端测试、UI/UX 验证、业务逻辑验证
2. **数据准确性** — 计算结果验证、报告数据核对、可视化正确性
3. **性能验证** — 响应时间、资源使用、并发处理

---

## 验证策略

### 1. 端到端测试流程

```
登录 → 数据加载确认 → KPI 计算核对 → 报告生成检查 → 可视化验证 → 导出下载
```

每一步验证要点：
- 登录：JWT Token 获取成功，DataContext.isDataLoaded = true
- KPI：使用黄金测试集对比已知正确答案（赔付率、费用率、综合成本率）
- 报告：页数、图表数量、表格数量符合预期
- 导出：文件可下载且格式正确

### 2. KPI 计算验证

准备已知正确答案的测试集，对比关键指标：
- 赔付率 = 已报告赔款 / 满期保费
- 费用率 = 费用金额 / 保费
- 综合费用率 = (赔款 + 费用) / 满期保费
- 边际贡献率 = 1 - 变动成本率

**容差**: rel_tol=0.01 (1%)

### 3. UI/UX 验证

使用 Playwright 自动化浏览器测试：
- 页面标题和导航结构
- KPI 卡片数量和数值格式
- 图表渲染（ECharts wrapper 存在性）
- 筛选器交互（日期、机构、险种）
- 响应式布局（移动端侧边栏隐藏）

### 4. 性能基准

| 操作 | 基准上限 |
|------|---------|
| API 查询 (KPI) | < 3s |
| API 查询 (趋势) | < 5s |
| 前端首屏渲染 | < 2s |
| 报告生成 | < 10s |
| 图表渲染 | < 2s |

使用 `curl -w '%{time_total}'` 验证 API 响应时间。

## 验证清单（新功能）

### 成本分析模块
- 赔付率/费用率/综合费用率/变动成本率 四个子 Tab
- 维度切换（机构/客户类别/险别组合）
- 截止日期筛选

### 商车系数监控
- commercial_pricing_factor 字段存在（仅商业险）
- 系数分布直方图、加权平均、中位数
- NCD 保费计算正确（保费 / 系数）

### 视角切换
- PerspectiveSwitcher 保费/件数 切换
- 切换状态持久化 (localStorage)
- 各模块（趋势/货车/增长率/续保）响应视角变化
