## 项目概述

驾意险推介率日报是一个自动化数据分析项目，用于每天自动生成驾意险业务分析报告，帮助管理层及时了解各三级机构的交叉销售业绩。

---

## 核心功能

### 1. 数据分析维度
- **驾意险推介率**：反映业务员交叉销售的积极性
  - 计算公式：driver_count / auto_count × 100%（车架号去重）
  - 目标值：≥ 30%
  - 优秀值：> 40%

- **驾意险件均保费**：反映驾意险保单价值
  - 计算公式：driver_premium / driver_policy_count
  - 基准值：> 200 元

- **驾意险件数**：反映绝对业绩
  - 统计周期：最近14天
  - 目标：环比增长 > 5%

### 2. 报告结构

#### 第一部分：核心结论（结论先行）
1. **整体趋势分析**
   - 最近7天 vs 前7天对比
   - 推介率、件均保费、件数变化

2. **问题机构识别**
   - 推介率低于平均水平80%的机构
   - 需要重点关注和培训

3. **优秀机构表彰**
   - 推介率高于平均水平120%的机构
   - 总结名验并推广

#### 第二部分：详细分析
- 驾意险推介率分析
- 件均保费分析
- 驾意险件数分析

#### 第三部分：数据表格
- 表格1：驾意险推介率（按机构和日期）
- 表格2：件均保费（按机构和日期）
- 表格3：驾意险件数（按机构和日期）

## 维护说明

### 更新数据源
当数据源文件更新时，需要：
1. 确保新文件位于 `warehouse/fact/policy/current/` 目录
2. 文件命名遵循规范：`每日数据_YYYYMMDD_YYYYMMDD.parquet`
3. 脚本会自动选择最新的"每日数据"文件

### 修改报告模板
编辑 `daily_report_jiayi.py` 文件中的以下函数：
- `generate_analysis()` - 修改分析逻辑
- `generate_table()` - 修改表格格式
- `generate_report()` - 修改整体报告结构

### 调整分析周期
修改 `daily_report_jiayi.py` 中的配置：
```python
CONFIG = {
    'days_to_analyze': 14,  # 修改为其他天数
}
```
