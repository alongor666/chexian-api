# 数据管理/AGENTS.md

> 数据管道、Parquet、ETL、业务规则相关任务优先遵守这里的规则；根目录 `AGENTS.md` 仍然适用。

## 1. 事实来源

- 数据处理任务优先遵守 `.claude/data-knowledge-protocol.md`。
- 业务口径唯一事实源：`数据管理/knowledge/rules/车险数据业务规则字典.md`。
- Parquet 结构与字段知识参考：`数据管理/knowledge/ai/PARQUET_SCHEMA_KNOWLEDGE.md`。

## 2. 工作方式

- 先搜现有实现、脚本和知识文档，再改 ETL、统计口径或数据产物。
- 不确定的业务定义、因果关系、分母分子口径，先确认再改，不要自行推断。
- 修改数据管道时，优先保持局部修补，避免无必要的大范围重构。

## 3. 验证要求

- 修改 SQL 生成器、Parquet 逻辑、字段映射或数据快照时，必须做源数据级验证。
- 涉及“是否报价”“终端来源”“渠道”“出险率”“推介率”等口径时，验证必须对照业务规则字典。
- 数据文件的本地源位于 `数据管理/warehouse/`，不要和 `server/data/` 的运行时文件混用。

## 4. 路径与生成

- 禁止硬编码路径；优先复用现有配置和脚本。
- 需要改动数据字典、索引或知识文档时，完成后同步更新受影响的说明文件。
