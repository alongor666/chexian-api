# 续保巡检报告撰写框架

> 本文件是 Claude 撰写续保巡检报告的规范。每次 /patrol 执行后，Claude 基于此框架 + 巡检数据撰写 `report.md`。
> 框架可根据用户反馈持续迭代优化。

---

## 一、报告定位

- **读者**：分公司管理层、机构负责人
- **目的**：发现续保经营中的问题和机会，给出可执行的行动建议
- **风格**：数据先行、结论明确、不假设因果、业务判断标注 ⚠️

## 二、三口径体系（必须）

每个维度的分析都必须展示四个口径的指标值，按以下顺序：

| # | 口径 | 定义 | 用途 |
|:-:|:-----|:-----|:-----|
| 1 | 当月 | `expiry_month = 当前月份`（自然月） | 当月经营快照，反映最近一个月的续保进展 |
| 2 | 窗口期 | `expiry_date ∈ (latest_data_date, latest_data_date + 30]` | 行动预警（即将到期，需立即触达） |
| 3 | 截至统计日 | `expiry_date <= latest_data_date - 1` | 年度累计实际绩效（续保窗口已开启的全部保单） |
| 4 | 当年累计 | 无日期限制 | 全景展望（含远期，被未到期保单稀释） |

**latest_data_date**：PolicyFact 当年最新 policy_date（如 2026-04-10）。

**四口径各有侧重**：
- 当月 — 聚焦本月，月中数据未完成时续保率偏低是正常的
- 窗口期 — 未来 30 天预警，关注报价覆盖是否到位
- 截至统计日 — 年度真实绩效基准
- 当年累计 — 被远期稀释，仅作全年展望

**必须在报告开头解释四口径含义**，避免读者误读。

## 三、客户类别优先级（必须按此顺序）

1. 非营业个人客车（体量最大，占 80%+）
2. 非营业货车（个人货车）
3. 非营业企业客车（含机关客车）
4. 营业货车（企业货车）
5. 营业公路客运（体量小，简要分析）

不分析的类别：摩托车、挂车、拖拉机（已在应续口径中排除）、营业出租租赁（体量小且续保模式特殊）、特种车（体量小）。

## 四、核心指标

| 指标 | id | 说明 |
|:-----|:---|:-----|
| 续保率 | renewal_rate | `已续件数 / 应续件数` |
| 报价覆盖率 | quote_coverage_rate | `已报价件数 / 应续件数` |
| 报价转化率 | quote_conversion_rate | `已续且已报价件数 / 已报价件数` |
| 流失保费 | premium_at_risk | `未续保保单保费总额 / 10000`（万元），衡量流失经济影响 |
| 应续件数 | policy_count | 中立指标，用于判断统计显著性 |

**流失保费**：指未续保保单的保费总额。如果这些客户最终不续保，公司将损失这些保费。

## 五、报告结构（必须包含以下章节）

### 第一章：三口径总览
- 三口径汇总表（整体 + 五类别各一行）
- 三口径含义解释（一句话）
- 关键结论（1-2 句）

### 第二章：五大类别逐一分析
每个类别包含：
1. 三口径指标表
2. 1-2 句定性判断（与整体对比、趋势、瓶颈在触达还是转化）
3. 机构排名表（已到期口径，按续保率升序）
4. 标杆机构 vs 末位机构点评

### 第三章：跨类别发现
- 新旧车、过户车、险别等配置外维度的发现
- 每个发现：现象 + 数据 + 可能原因（标注 ⚠️）

### 第四章：行动建议
- 按 P1/P2/P3 排序
- 每条包含：行动项 + 目标 + 预期影响（量化）

### 元信息
- 报告末尾注明：数据日期、查询次数、AI 撰写声明、⚠️ 免责

## 六、撰写规则

### 必须做
- 所有数字右对齐，文字左对齐
- 率值用百分比（1 位小数），保费用万元（整数）
- 件数用整数
- 偏离大于 5 个百分点才算"偏低"
- 标杆机构和末位机构都要点名
- 行动建议必须量化预期影响

### 禁止做
- ❌ 假设因果关系（"因为…所以续保率低"）
- ❌ 使用"风险保费"（改用"流失保费"）
- ❌ 使用英文（指标名、维度名全部中文）
- ❌ 只展示一个口径（必须三口径）
- ❌ 把全年口径当实际绩效解读
- ❌ 对 <30 件的分组做结论（统计不显著）

## 七、下钻查询模板

```sql
-- 某类别 × 机构（已到期口径）
SELECT org_level_3, COUNT(*) AS cnt,
  SUM(is_renewed::INT) AS renewed,
  ROUND(SUM(is_renewed::INT)*1.0/COUNT(*), 4) AS renewal_rate,
  ROUND(SUM(is_quoted::INT)*1.0/COUNT(*), 4) AS quote_coverage,
  ROUND(SUM(CASE WHEN NOT is_renewed THEN total_premium ELSE 0 END)/10000, 0) AS lost_premium_wan
FROM read_parquet('数据管理/warehouse/fact/renewal_universe/latest.parquet')
WHERE customer_category = '{cat}'
  AND expiry_date <= '{latest_data_date_minus_1}'
GROUP BY org_level_3
HAVING COUNT(*) >= 30
ORDER BY renewal_rate ASC
```
