## 🎯 核心业务概念

### 1. 险类体系 (险类)

**定义**: 保险产品的分类维度,区分强制保险和商业保险

**值域**:

- `交强险` (76.2%): 机动车交通事故责任强制保险,国家强制购买
- `商业保险` (23.8%): 自愿购买的商业险种

**业务规则**:

- ✅ 一个保单可能同时包含交强险和商业险(通过 `险别组合`区分)
- ✅ 交强险保费固定(与车型无关),商业险保费可变
- ✅ `商车自主定价系数` 仅对商业险生效

### 2. 险别组合 (险别组合)

**定义**: 同一保单下包含的险别组合类型

**值域**:

- `单交` (54.3%): 仅交强险
- `交三` (23.0%): 交强险 + 第三者责任险
- `主全` (22.7%): 交强险 + 商业险全险种
- `其他` (0.0%): 其他组合

**业务规则**:

- ✅ `单交` = 仅交强险,无商业险
- ✅ `交三` = 基础组合,保额较低
- ✅ `主全` = 全险组合,包含车损险、三者险、盗抢险等
- ✅ 批改时 `险别组合`可能变更(如从 `主全`改为 `单交`)

### 3. 统保类型 (是否交商统保)

**定义**: 交强险和商业险是否在同一保单(统保优惠)

**值域**:

- `单交` (54.3%): 仅交强险
- `套单` (44.1%): 交强险 + 商业险一同在我司投保
- `单商` (1.6%): 仅商业险(交强险不在我司投保)
- `其他` (0.0%): 特殊情况

### 4. 客户类别体系 (客户类别)

**定义**: 按车辆使用性质和车主类型分类

**值域** (11个类别):

| 类别               | 占比  | 业务特征                  |
| ------------------ | ----- | ------------------------- |
| `非营业个人客车` | 58.8% | 个人自用轿车,SUV等        |
| `摩托车`         | 28.6% | 二轮/三轮摩托车           |
| `非营业货车`     | 4.8%  | 个人/企业自用货车         |
| `非营业企业客车` | 4.8%  | 公司公务车                |
| `营业货车`       | 2.9%  | **🎯 重点分析对象** |
| `营业出租租赁`   | 0.1%  | 出租车/网约车             |
| `特种车`         | 0.1%  | 工程车、消防车等          |
| `营业公路客运`   | 0.1%  | 长途大巴                  |
| `挂车`           | 0.0%  | 半挂车                    |
| `非营业机关客车` | 0.0%  | 政府机关车辆              |
| `营业城市公交`   | 0.0%  | 公交车                    |

**业务规则**:

- ✅ **营业货车**是专项分析重点(对应 `吨位分段`字段)
- ✅ 营业性车辆保费高于非营业车辆(风险更高)
- ✅ 摩托车保费最低,单交占比高
- ✅ 特种车、挂车需要特殊核保规则

## 🔑 关键字段详解

### A. 主键字段

#### 保单号 (policy_no)

- **数据类型**: STRING
- **唯一性**: 100% (607,455个唯一值)
- **非空率**: 100%
- **格式**: `6102101030120240000857` (22位数字)
- **业务含义**: 保单唯一标识
- **业务规则**:
  - ✅ 同一保单号可能对应多条记录(交强险+商业险分开记录)
  - ✅ 通过 `保单号 + 险类`组合去重获取唯一保单

#### 续保单号 (renew_policy_no)

- **数据类型**: STRING
- **唯一性**: 221,002个唯一值
- **非空率**: 36.4% (221,002 / 607,455)
- **业务含义**: 上一年度保单号
- **业务规则**:
  - ✅ **空值 = 非续保**,非空值 = 续保
  - ✅ 通过 `是否续保`字段快速判断(True=有续保单号)
  - ✅ 续保率 = 36.4% (221,002 / 607,455)

#### 批单号 (endorsement_no)

- **数据类型**: STRING
- **唯一性**: 10,890个唯一值
- **非空率**: 1.8% (10,890 / 607,455)
- **格式**: `6102101030120240000857-001` (保单号-批单序号)
- **业务含义**: 批改业务凭证号
- **业务规则**:
  - ✅ **非空 = 该保单发生批改**(退保/过户/变更信息等)
  - ✅ 批改率 = 1.8%
  - ✅ 一个保单可能有多条批改记录(-001, -002, ...)

### B. 日期字段

#### 签单日期 (sign_date)

- **数据类型**: STRING (格式: YYYY-MM-DD)
- **唯一性**: 725个不同日期
- **非空率**: 100%
- **值域**: 2023-12-05 ~ 2026-01-08 (实际数据覆盖范围)
- **业务含义**: 保单签订日期
- **业务规则**:
  - ✅ 签单日期 <= 保险起期 (通常提前1-30天)
  - ✅ 签单日期用于统计**业绩**(按签单日归属)
  - ✅ 日期字符串需转换为 DATE 类型才能进行日期计算

#### 保险起期 (start_date)

- **数据类型**: STRING (格式: YYYY-MM-DD)
- **唯一性**: 817个不同日期
- **非空率**: 100%
- **值域**: 2023-12-29 ~ 2026-01-08
- **业务含义**: 保险责任开始时间
- **业务规则**:
  - ✅ 保险起期 - 签单日期 = 提前承保天数(通常1-30天)
  - ✅ 保险起期用于统计**业务量**(按生效日归属)
  - ✅ 续保判断: 当前保险起期 - 上一年度保险起期 ≈ 365天

### C. 金额字段

#### 保费 (premium)

- **数据类型**: FLOAT64
- **非空率**: 100%
- **值域**: -18,902.75 ~ 30,185.35
- **统计特征**:
  - 平均值: 822.98
  - 中位数: 672.59
  - 标准差: 1,119.86
- **业务含义**: 保险费(可能为负数)
- **⚠️ 关键业务规则**:
  - ✅ **负保费 = 批改退费**(退保/减费批改)
  - ✅ `保费 > 0` = 正常承保保费
  - ✅ `保费 < 0` = 批改退费金额
  - ✅ 统计总保费时需 **SUM(保费)** (正负抵消后为实收保费)
  - ✅ 统计毛保费时需 **SUM(ABS(保费))** (不考虑退费)

#### 新车购置价 (car_price)

- **数据类型**: FLOAT64
- **非空率**: 100%
- **值域**: 0 ~ 12,600,000
- **统计特征**:
  - 平均值: 137,238.59
  - 中位数: 75,900.00
  - 标准差: 249,339.55
- **业务含义**: 新车购买价格(车损险保额依据)
- **业务规则**:
  - ✅ 车损险保费与新车购置价正相关
  - ✅ 0值 = 摩托车/仅投保交强险(无车损险)
  - ✅ 高值车辆(>100万)通常需要特殊核保

#### 商车自主定价系数 (pricing_factor)

- **数据类型**: FLOAT64
- **非空率**: 23.8% (仅商业险有值)
- **值域**: 0.5 ~ 1.5
- **统计特征**:
  - 平均值: 0.91
  - 中位数: 0.85
  - 标准差: 0.18
- **业务含义**: 商业险自主定价系数(NCD系数)
- **业务规则**:
  - ✅ **仅商业险生效**,交强险该字段为空
  - ✅ 系数范围 [0.5, 1.5] (监管规定)
  - ✅ 系数 < 1.0 = 优惠(连续无赔款年份多)
  - ✅ 系数 > 1.0 = 上浮(出险次数多)
  - ✅ 系数 = 1.0 = 基准费率

- **业务规则**:
  - ✅ 车架号是车辆的物理唯一标识

  - ✅ 可结合保单号分析同一车辆的多次投保
  - ✅ 高基数字段，不适合直接分组分析

#### 是否报价 (is_quote)

- **数据类型**: BOOL
- **非空率**: 100%
- **值域**:
  - `False`: 82.1% (非报价记录)
  - `True`: 17.9% (报价记录)
- **业务含义**: 是否为报价记录（非正式保单）
- **业务规则**:

  - ✅ `False` = 正式保单记录

#### 赔案件数 (claim_cases)

- **数据类型**: INT64
- **非空率**: 96.1% (3.9%空值填充为0)
- **值域**: 0 ~ N（整数）
- **统计特征**:
  - 平均值: 0.09
  - 最大值: 需要进一步统计
- **业务含义**: 累计赔案件数量
- **业务规则**：
  - ✅ 用于计算**出险率** = 赔案件数 / 已赚暴露（earned exposure，**非**签单件数；见铁律 §4）
  - ✅ 用于计算**案均赔款** = 已报告赔款 / 赔案件数
  - ✅ 空值默认填充为 0，表示无赔案

> ⚠️ 本字段是赔付**频度**的分子，**不要**与赔付**金额**类指标（赔付率 / 变动成本率 / 综合费用率）的公式混挂。
> 那些公式的完整定义见 `已报告赔款 (reported_claims)` 和 `费用金额 (fee_amount)` 节。

#### 已报告赔款 (reported_claims)

- **数据类型**: FLOAT64
- **非空率**: 96.1% (3.9%空值填充为0)
- **值域**: 0 ~ 高值（元）
- **统计特征**:
  - 总计: 195,223,115.10元
- **业务含义**: 累计已报告的赔款金额（=已决赔款 + 未决赔款）
- **业务规则**（与 metric-registry 唯一事实源对齐）：
  - ✅ 用于计算**满期赔付率** = 已报告赔款 / 满期保费（闰年感知；`SUM(reported_claims) / SUM(premium × earned_days/policy_term)`）
  - ✅ 用于计算**变动成本率** 的赔付分量（同上：已报告赔款 / 满期保费），费用分量另算
  - ✅ 用于计算**综合费用率** = (已报告赔款 + 费用金额) / 满期保费
  - ✅ 结合 `赔案件数` 计算 **案均赔款** = 已报告赔款 / 赔案件数
  - ✅ 空值默认填充为 0，表示无赔款支出

> 📌 **率值聚合铁律**：所有以"已报告赔款"为分子的率值必须 `SUM(分子) / SUM(分母)`，禁止逐行赔付率再二次平均（见全局护栏）。

#### 费用金额 (fee_amount)

- **数据类型**: FLOAT64
- **非空率**: 100%
- **值域**: 0 ~ 高值（元）
- **统计特征**:
  - 总计: 83,737,838.03元
- **业务含义**: 累计费用金额（含各项手续费、服务费等）
- **业务规则**（与 `server/src/config/metric-registry/categories/cost.ts` 唯一事实源对齐）：
  - ✅ 用于计算**费用率** = 费用金额 / 签单保费（**专指此口径**，分母签单保费 `premium`）
  - ✅ 用于计算**变动成本率** = 已报告赔款 / 满期保费 + 费用金额 / 签单保费
    - 两个分量分母不同（满期 vs 签单），闰年感知；可超 100%（亏损）
  - ✅ 用于计算**综合费用率** = (已报告赔款 + 费用金额) / 满期保费
    - 与变动成本率的区别：分子费用统一除以满期保费（已赚口径），适合综合费用对标
  - ✅ 费用结构分析可优化成本控制

> ⚠️ **独立口径护栏（B273 决策，2026-05-01）**：
> 「费用率」「综合费用率」「变动成本率」是**三个独立指标**，公式互不相同（分子/分母都不同），
> **禁止把"费用金额/保费"这一行公式同时挂在三个指标名下**——这会让后续 AI/人类按错口径推理。
> 「综合成本率」是与综合费用率派生关系的另一独立指标，由综合分析侧定义，不可与上述三者互推。

### 1.1 续保分析统计逻辑（重要业务知识）

在实际业务中，**续保分析存在两种不同的统计模式**，AI 和开发者必须理解。

#### 【核心公式】到期日计算规则

```
到期日 = 起保日 + 1年 - 1天
```

**⚠️ 这个 "-1天" 非常关键！** 它导致每个月的1日起保的保单实际在**上个月最后一天**到期：

| 起保日 | 到期日 | 实际到期月份 |
|--------|--------|-------------|
| 2025-02-01 | 2026-01-31 | **1月** |
| 2025-02-02 | 2026-02-01 | **2月** |
| 2025-03-01 | 2026-02-28 | **2月** |
| 2025-03-02 | 2026-03-01 | **3月** |

**闰年影响**：
- 2024-02-29 起保 → 2025-02-28 到期（闰年到非闰年）
- 2024-03-01 起保 → 2025-02-28 到期

**SQL 实现**：
```sql
-- 正确：按到期日月份筛选
MONTH(DATE_ADD(CAST(insurance_start_date AS DATE), INTERVAL '1 year') - INTERVAL '1 day') = 2

-- 错误：按起保月份筛选（会漏掉3月1日起保的保单）
MONTH(CAST(insurance_start_date AS DATE)) = 2
```

#### 模式一：按到期月份统计

| 项目 | 说明 |
|------|------|
| **触发条件** | 用户选择特定"到期月份"（如"2月到期"） |
| **应续件数** | 上一年该月起保的**所有**保单（不管当前是否已到期） |
| **已续件数** | 这些应续保单中，已经成功续保的数量（含提前续保） |
| **cutoffDate** | ❌ 不应用（要看完整月份的数据） |

**使用场景**：
- 📊 **当月到期追踪**：现在是1月，看1月到期保单的续保率
- ⚠️ **次月到期预警**：现在是1月，提前看2月到期保单有多少已提前续保
- 📈 **历史月份回顾**：回顾去年某月的最终续保率

```sql
-- 示例：查询2月到期的应续件数（不受cutoffDate限制）
SELECT COUNT(*) as due_count
FROM RenewalTrackerFact
WHERE expiry_month = 2
```

#### 模式二：按到期日范围统计

| 项目 | 说明 |
|------|------|
| **触发条件** | 用户选择"全部月份" + 设置 cutoffDate |
| **应续件数** | 到期日在 [年初, cutoffDate] 范围内的保单 |
| **已续件数** | 这些应续保单中，已经成功续保的数量 |
| **cutoffDate** | ✅ 应用（限制到期日范围） |

**使用场景**：
- 📊 **年度累计续保率**：从年初到今天的整体续保情况
- 📅 **特定时间段统计**：某个日期范围内已到期保单的续保率

```sql
-- 示例：查询2026-01-01到2026-01-27到期的应续件数
SELECT COUNT(*) as due_count
FROM RenewalTrackerFact
WHERE CAST(expiry_date AS DATE) BETWEEN '2026-01-01' AND '2026-01-27'
```

#### 两种模式的关系

```
用户操作                    SQL 生成逻辑
─────────────────────────────────────────────────────────
选择"2月到期"          →    按月份筛选，忽略 cutoffDate
选择"全部月份"         →    应用 cutoffDate 的日期范围
─────────────────────────────────────────────────────────
```

**⚠️ 关键原则**：这两种模式**互斥**，不会同时生效。代码中通过 `if (cutoffDate && !dimension.dueMonth)` 实现。

**📍 代码位置**：`src/shared/sql/renewal-drilldown.ts`

- ✅ 毛保费 = SUM(ABS(保费)) (不考虑退费)

```sql
-- 统计实收保费
SELECT SUM(premium) FROM data  -- ✅ 正负抵消

-- 统计承保保费(不含退费)
SELECT SUM(premium)
FROM data
WHERE premium > 0  -- ✅ 仅统计正保费

-- 统计退费金额
SELECT SUM(ABS(premium))
FROM data
WHERE premium < 0  -- ✅ 仅统计负保费
```

```sql
-- 按保单号+险类去重
SELECT DISTINCT
    policy_no,
    险类
FROM data

-- 获取唯一保单数
SELECT COUNT(DISTINCT policy_no) AS unique_policies
FROM data
```

## 新增业务规则（v3.0, 2026-03）

### 四级亮灯体系

替代旧三级体系，用于诊断报告中的指标评级：

| 等级 | 符号 | 含义 |
|------|------|------|
| 正常 | 🟢 | 指标健康 |
| 关注 | 🔵 | 需要关注 |
| 预警 | 🟡 | 需要干预 |
| 危险 | 🔴 | 需要紧急处理 |

**阈值配置**（`diagnose_common.py` TH_* 常量）:

| 指标 | 关注 | 预警 | 危险 | 方向 |
|------|------|------|------|------|
| 变动成本率 | 85% | 91% | 94% | 越高越差 |
| 边际贡献率 | 15% | 9% | 6% | 越低越差 |
| 满期赔付率 | 60% | 70% | 75% | 越高越差 |
| 满期出险率 | 8% | 10% | 12% | 越高越差 |
| 案均赔款-货车 | 8,000 | 10,000 | 12,000 | 越高越差 |

### 满期保费公式（闰年感知）

```
policy_term = DATEDIFF('day', 保险起期, 保险起期 + INTERVAL 1 YEAR)
  → 365 天（非闰年）或 366 天（闰年，保险起期在闰年 2/29 前）
earned_days = LEAST(DATEDIFF('day', 保险起期, 截止日), policy_term)
earned_premium = 保费 × earned_days / policy_term
```

**注意**: 旧版固定 365 天会导致闰年保单的满期保费计算偏高。

### 满期出险率年化公式

```
annualized_cases = SUM(赔案件数 × policy_term / earned_days)
incident_rate = annualized_cases / COUNT(DISTINCT 保单号) × 100
```

未满期保单的赔案件数需按 `policy_term / earned_days` 放大到年化。

### 边际贡献额

| 指标 | 公式 | 用途 |
|------|------|------|
| 满期边际贡献额 | `满期保费 × (1 - 赔付率/100 - 费用率/100)` | 已赚部分的实际盈利 |
| 预估边际贡献额 | `签单保费 × (1 - 赔付率/100 - 费用率/100)` | 含未到期保单的预估盈利 |

**变动成本率** = 满期赔付率 + 费用率（注意两个分母不同：满期保费 vs 签单保费）

### 件均保费 vs 车均保费

| 指标 | 公式 | 说明 |
|------|------|------|
| 件均保费 | `SUM(保费) / COUNT(DISTINCT 保单号)` | 每张保单的平均保费 |
| 车均保费 | `SUM(保费) / COUNT(DISTINCT 车架号)` | 每辆车的平均保费（含交强险+商业险） |

- 一台车通常购买交强险和商业险两张保单，因此**车均保费 > 件均保费**
- 交强险保费固定、强制、全行业统一；商业险保费因车辆、险别组合而异
- 车均保费衡量"每辆车平均保费水平"，更贴近客户视角的保费贡献度
- 车架号为空时回退至保单号（`COALESCE(vehicle_frame_no, policy_no)`）

### 驾乘推介率口径

```
驾乘推介率 = 驾意险推介件数 / 商业险出单件数
```

- **分子**: COUNT(DISTINCT CASE WHEN is_cross_sell THEN 去重键 END)，去重键 = COALESCE(vehicle_frame_no, policy_no)
- **分母**: COUNT(DISTINCT 去重键)，仅含 `coverage_combination IN ('主全','交三')`
- **整体行 = 主全件数 + 交三件数**（排除纯交强/单交）

### 驾乘渗透率

```
驾乘渗透率 = 驾意险承保件数 / 商业险承保件数
```

### 率值治理规则

**禁止对子项率值做算术平均**，汇总时必须基于绝对值重算：

| 率值类型 | 聚合方式 |
|---------|---------|
| A 类（赔付率、费用率） | SUM(分子) / SUM(分母) × 100 |
| B 类（出险率） | SUM(annualized_cases) / SUM(policy_count) × 100（基于可加的年化赔案数） |
| 推介率 | SUM(driver_count) / SUM(auto_count) × 100（基于绝对值件数） |
| 定价系数 | 保费加权平均 |

### 维修资源分析口径

**数据域**：`repair_resource`（RepairDim，07_维修资源.xlsx）× `claims_detail`（ClaimsDetail，02_理赔明细.xlsx 新增"标的汽修厂"）

#### 1. 合作三态分类（coop_tier）

源字段 `cooperation_status` 枚举 → 三态映射：

| 源枚举 | coop_tier | 分类 |
|-------|-----------|------|
| `1生效中` | `active` | 已合作 |
| `0暂停合作` / `7已撤销` / `8失效` | `past` | 曾合作 |
| `3退回修改` / `5待复核` / `无合作` / NULL | `none` | 未合作 |
| ClaimsDetail 中出现但不在 RepairDim | `none_shadow` | 未合作（影子） |

#### 2. 网点匹配规则

- **JOIN Key**：`shop_code = SUBSTR(shop_name, 1, 8)`（前 8 位编码）
- **排除关键词**（非维修单位）：
  - `shop_name LIKE '%定损%'` → 剔除
  - `shop_name LIKE '%自选%'` → 剔除
  - `shop_name = '无'`（精确匹配）→ 剔除
  - 其他关键词待业务补充（`等`）

#### 3. 核心指标公式

| 指标 | 公式 | 层级 |
|------|------|------|
| **修保比** | `SUM(damage_assessment_amount) / NULLIF(SUM(net_premium), 0)` | L2（RepairDim 单表） |
| **本地资源占比** | `COUNT(DISTINCT CASE WHEN c.accident_district = r.district THEN c.claim_no END) / NULLIF(COUNT(DISTINCT c.claim_no), 0)` | L4（RepairDim LEFT JOIN ClaimsDetail） |
| **合作启用率** | `COUNT(DISTINCT CASE WHEN coop_tier='active' THEN shop_code END) / NULLIF(COUNT(DISTINCT shop_code), 0)` | L2 |
| **4S 占比** | `COUNT(DISTINCT CASE WHEN is_4s_shop THEN shop_code END) / NULLIF(COUNT(DISTINCT shop_code), 0)` | L2 |
| **导流潜力保费** | `SUM(p.premium) WHERE 保单对应赔案的 subject_shop_code 属于 past 或 none 或 none_shadow` | L4 |

**业务解读（修保比）**：
- `< 0.3` → 合作深度低，网点拿我们保费多但用我们定损少（可挖掘）
- `0.3-0.7` → 健康区间
- `> 0.7` → 定损偏重，保费贡献不足（业务员应推动）

#### 4. 时间口径

维修资源分析**同时展示**两个时间窗口：

| 口径 | 定义 |
|------|------|
| **自然年度 YTD** | `YEAR(accident_time) = YEAR(CURRENT_DATE)` |
| **滚动 12 月** | `accident_time >= MAX(accident_time) - INTERVAL 12 MONTH`（基准日 = `MAX(ClaimsDetail.accident_time)`，非 `CURRENT_DATE`，避免数据滞后造成空窗） |

#### 5. 未合作网点的地理归属（影子网点）

ClaimsDetail 中出现但 RepairDim 中不存在的网点（`coop_tier = none_shadow`），无 `province/city/district`。
归属规则：**以该 `subject_shop_code` 接到的赔案中"出险区县众数"作为地理归属**。

```sql
WITH shadow_geo AS (
  SELECT
    subject_shop_code,
    accident_district,
    COUNT(*) AS cnt,
    ROW_NUMBER() OVER (PARTITION BY subject_shop_code ORDER BY COUNT(*) DESC) AS rn
  FROM ClaimsDetail
  WHERE subject_shop_code NOT IN (SELECT DISTINCT shop_code FROM RepairDim)
  GROUP BY subject_shop_code, accident_district
)
SELECT subject_shop_code, accident_district AS shop_district FROM shadow_geo WHERE rn = 1;
```

#### 6. 散点图呈现口径

- **X 轴**：区县（按机构 × 区县分组排序）
- **Y 轴**：机构
- **颜色**：三态（绿=active / 黄=past / 红=none/none_shadow）
- **点大小**：`signed_premium` 签单净保费

#### 7. 业务目标

识别在**曾合作/未合作/影子**网点维修的客户保单，引导：
- **导流回流**：推动客户改用已合作网点 → 提升合作深度
- **新合作转化**：将高保费贡献的影子网点转为合作网点
