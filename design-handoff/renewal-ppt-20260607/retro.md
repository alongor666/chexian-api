## 3. 简报缺失归因

- **简报没写死「线框交接时若用户未选方案，落地方怎么办」**：本次我按简报逐页意图替每个页型选了最贴合方案并在回复里标注「可换」。建议简报 / handoff README 补一句：*「若交接时尚未选定方案，落地方按简报 §3 逐页意图选最贴合者，落地后列出所选方案供确认」*。
- **简报 §3 页3 要「四级亮灯阈值」，但源报告只给每格灯色、未定义数值阈值**：我的处置是**逐格照搬源报告灯色 + 只展示定性图例（达标/关注/预警/危险），不杜撰数值阈值**。建议简报补：*「亮灯一律用报告既有着色，禁止反推/杜撰数值阈值」*——否则落地方可能编一套阈值，反成新的「数据篡改」。
- **driver 4 页初次溢出（P3/P11/P12/P13）不是简报缺失**，是落地密度问题：窄列里的「指标口径」长句换行撑高、单表页填充不足需双表拼页。已在落地层解决（见 §4 通用经验）。

## 4. 分流回写

### 4.1 通用经验（任何项目适用）→ 回写 ui-redesign skill 仓库

> 路径：clone `alongor666/alongor666-skills` → 编辑 `skills/ui-redesign/SKILL.md` 或 `references/*.md` → push → 重装软链。

- [ ] **线框阶段交接的落地协议**：交接包可能停在 Wireframe（low-fi）阶段且 chat 以「请用户选方案」结尾。落地方应：① 按简报逐页意图选最贴合方案；② 落地后**显式列出所选方案供确认**（混搭可改）；③ 不阻塞等待。建议写进 ui-redesign Phase C。
- [ ] **多页数据 deck 用「数据驱动渲染」而非手写每格**：把所有数字放进**唯一的 JS const 数组**，渲染函数生成 DOM——文件内单一事实源，且能写**列求和 = 合计**的自洽脚本一键抓转写错。比逐格硬写更不易篡改。
- [ ] **driver.mjs 冷启动竞态**：driver 每次用全新 user-data-dir + 1500ms settle，**冷启动 Chrome 会在 settle 内没加载完 → 误判 0 个 `.page` → 落入「连续文档」模式报假 FAIL**。排查法：先 warm 一次 Chrome / 循环跑到落入 deck 模式 / 用固定 warm udd 的探针自测填充率。务必别把「0 页」误诊成真 bug。
- [ ] **16:9 填充率调法**：单表页易**欠填**（<80%）→ 双表拼一页冲到 80%+；窄侧栏长句**易溢出** → 改成单行密排（`·` 分隔）+ 降字号 + 收紧 line-height。
- [ ] **亮灯阈值红线扩写**：acceptance / 简报应写死「亮灯一律照搬源报告着色，禁止反推数值阈值」。

### 4.3 触发式进化（≥2 次重复）→ 当次必须回写

本次出现 ≥2 次的同类问题：

- **位置数组「索引漂移」bug 出现 2 次**：
  - `bulletRow(r[0],r[7],r[8])` 误把灯色字段 `r[8]` 当续保率传入 → `.toFixed` 崩（应 `r[9]`）。
  - `tableMatured` 合计行取 `sum[6]` 越界（合计数组无灯色列，续保率在 `sum[5]`）。
  - **根因同一个**：明细行数组**内嵌灯色字符串**（如 `[...,报价率,'ok',续保率,'danger']`），而合计数组**不含灯色列**，两者列序不对齐 → 索引极易漂。
  - **防御**：①（已做）写「列求和 = 合计」核验脚本，跑一次即暴露；② 更稳的做法是**用具名字段对象**（`{org,due,quoted,renewed,rate,light}`）取代位置数组，根除索引漂移。建议回写 ui-redesign：*「数据驱动渲染优先具名字段对象，禁止灯色与数值混在同一位置数组」*。

→ **回写承诺**：用户下次说「crystallize / 沉淀 skill」时，把 §4.1 全部 + §4.3 具名字段经验按 `chexian-crystallize-skill` 流水线回写 ui-redesign 仓库。

## 6. 写给下一次重做的「经验包」
