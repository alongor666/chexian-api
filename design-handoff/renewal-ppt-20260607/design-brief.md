## 1. 这是什么 · 受众 · 场景

- **产物**：四川分公司 2026 年 6 月「续保经营盯盘」演示稿，16:9 大屏。
- **受众**：分公司管理层（非技术）。看的是各三级机构续保表现、缺口归因、紧急派单。
- **场景**：① 大屏投影、管理层逐页讲解；② 会后留档自阅。所以要**结论一眼可见 + 数据可追**。
- **当前痛点**：现稿信息忠实但偏"报告纸感"，12~14 行的机构密集表在投影上偏挤、缺少视觉重点；想要更现代的数据产品观感。

## 2. 视觉方向：现代经营仪表盘（BI 数据产品感）

| 维度 | 要什么 | 避免什么 |
|---|---|---|
| **基调** | 深色或中性浅灰为基底、克制专业、数据产品质感 | 纸纹底、印章、做旧暖黄 |
| **强调色** | **单一品牌强调色**统领界面（建议靛蓝系或保留克制朱红，二选一）；用于标题条、KPI 高亮、主图主色 | 多个高饱和强调色互抢 |
| **状态色（红线，见 §4）** | 四级亮灯**必须保留四档可辨**：达标/关注/预警/危险。可换更现代色值（如 翠绿 / 天蓝 / 琥珀 / 玫红），但四档之间要清晰区分 | 把四档压成两色或同色深浅难辨 |
| **字体** | 现代无衬线（思源黑 / Inter 系），数字用等宽数字（tabular-nums）。标题可大、可粗 | 宋体装饰性标题、花体 |
| **组件** | KPI 卡片网格、横向条形图（机构排名）、对比/进度条、徽标 chips | 玻璃拟态、霓虹辉光、超大渐变、卡通圆角、emoji 滥用（亮灯点除外） |
| **图表** | 简洁扁平：条形 / 子弹图 / 进度条 / 小多图（small multiples） | 3D、阴影堆叠、过度动效装饰 |
| **留白与层级** | 每页一个核心信息，标题—主图—结论三段式，留白大气 | 把整页塞满数字 |

> Claude Design 的常见默认要主动压住：**别加大面积渐变、别加玻璃模糊、别把企业数据做成消费 App 的彩色泡泡**。这是给分公司一把手看的经营盘子，要稳、准、专业。

## 4. 红线（绝对不能动）

1. **数据零篡改**：7 张表的所有三级机构、所有数字（应续/已报价/已续保/未报价/流失/影响度/报价率/续保率）必须与源报告一致，主线图表化后**原值仍要能在附录全表里查到**。一个机构、一个数字都不能丢或改。
2. **四级亮灯语义**：达标(🟢)/关注(🔵)/预警(🟡)/危险(🔴) 四档色值可现代化，但**四档必须清晰可辨**；续保率亮灯只用于 ① 已到期、⑤ 当年已到期（已成熟最终留存），② 临期、③④ 未到期为**进度口径不亮灯**（这点必须在视觉上区分，别让进度值被误读成最终留存）。
3. **续保影响度口径**：= 该机构流失 ÷ 合计应续，**可加和**（各机构之和 = 整体续保缺口）。图表别破坏这个可加和直觉（用堆叠占比/帕累托更佳）。
4. **全中文**：正文、标题、图例、轴标签全部中文。英文术语（KPI / BI 等）首次出现给中文全称。**禁止英文缩写堆砌**。
5. **画幅与交互**：固定 16:9 = **1280×720px**；每页内容区可用高 **≈604px**，填充率 80–100%、零溢出、内容不跨页；保留**横向翻页**（← → / 序号条直达 / 全屏 / Esc 缩略图总览）与"一页一张 16:9 导出"。
