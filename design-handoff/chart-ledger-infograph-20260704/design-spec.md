# 图表账本 · 12 张读图指南（infograph）设计规格

> 2026-07-04 · 需求：每张图右上角一个提示符号，点击进入该图的 infograph（信息图解）。
> 本文件是「先设计后实现」的设计规格；实现 SSOT = `src/features/chart-ledger/infographs/`。

## 1. 定位与原则

- **infograph = 读图指南**：卡片左栏的「怎么看」是一句话速记；infograph 是**深读版**——图形解剖、读图步骤、判定规则、决策映射。教会一个第一次看这张图的人"怎么读、读到什么就做什么"。
- **不复述数据**：infograph 讲**图的方法论**（形态与规则），不含实时数据（数据在卡片本体，随筛选联动）。
- **零新增查询**：纯静态内容，不发任何请求。
- 设计语言延续账本：LEDGER 五色（teal/gold/coral/good/muted）只用于数据示意，UI 层走设计系统 token；数字等宽；无渐变网格/装饰。

## 2. 交互设计

| 项 | 设计 |
|----|------|
| 触发器 | 图表卡片（右栏白卡）**右上角** ⓘ 圆形按钮（28px，中性描边，hover 转主色）；`aria-label="查看读图指南：<图名>"` |
| 打开 | 点击 → 居中弹层（modal，`role="dialog" aria-modal`），背板半透明遮罩 |
| 关闭 | ① 右上 × ② ESC ③ 点击背板（背板本身是 button，**不挂 document 全局监听**——PR #481 时序教训） |
| 弹层规格 | `max-w-2xl`、`max-h-[85vh]` 内滚，`cardStyles.base` 皮肤，明暗双模式 |
| 滚动锁 | 打开时锁 body 滚动，关闭恢复 |
| 焦点 | 打开自动聚焦关闭按钮；Tab 循环不强制（v1 简化），ESC 兜底 |

## 3. 内容框架（12 张统一五段）

1. **这张图回答什么**（primary 左边线色块，一句话，与卡片"结论先行"的 warning 色块区分）
2. **图形解剖**（SVG 注解图 560×240：示意形态 + 区域/元素语义标注；细节图注列在 SVG 下方）
3. **读图三步**（编号步骤，快速路径）
4. **判定规则**（该图用到的阈值/统计规则 + 局限说明；口径 SSOT 指向 `ledgerMeta.LOSS_RATIO_THRESHOLD` 与 hook 内客户端规则）
5. **决策映射**（信号 → 经营动作（沿用 加码/复制/优化/整改/预警/暂停 双编码标签）→ 下一步）

## 4. 12 张 infograph 内容定义（摘要）

| 图 | 解剖图核心 | 读图三步 | 关键规则 | 决策映射要点 |
|----|-----------|---------|---------|-------------|
| 01 气泡矩阵 | 65% 金线切出上下两带 + 四区语义（右上=大而差/右下=压舱石/左下=小而优/左上=小而差），气泡大小=件数 | 看线上方有谁 → 找线上最大气泡 → 左下找可复制样本 | 65% 为展示阈值（SSOT ledgerMeta） | 右上→优化整改；左下→复制；右下→守住 |
| 02 异常散点 | 主群（teal 圆团）vs 离群（coral 菱形）+ 均值+2σ 竖线 | 看主群位置 → 看竖线右侧 → 离群先核实再定性 | 均值+2σ；样本<5 家不判（统计量不稳） | 离群→专项稽核；主群整体右移→费用政策复盘 |
| 03 热力图 | 色标 teal(50%)→coral(95%)；最热格、整行红、整列红三种形态 | 扫最红格 → 看所在行与列 → 区分组合/机构/险种问题 | 梯度 50→95 映射（lossRatioColor SSOT） | 单格红→限额该组合；整行红→机构核保收紧；整列红→险种定价上报 |
| 04 箱线图 | 五数解剖（最大/Q3/中位金线/Q1/最小）+ 箱高=中间50% + 上须长=长尾 | 比中位数 → 比箱高 → 看上须长度 | 机构数≥3 才出箱；取极差最大前 5 客群 | 上须特长→大案复核/反欺诈；整箱偏高→客群定价复审 |
| 05 频度趋势 | 双线错位示意：频度线领先赔付率线 2–3 周 | 看近 3 周方向 → 与起始比水平 → 抬头即预警 | 满期口径（已赚暴露分母）；先行指标 | 连续上行→提前介入核保/风控，不等赔付率恶化 |
| 06 发展三角 | 行=起保年度、列=满 N 月；对角线=数据边界；**纵向同列比**高亮 | 竖比同列 → 新年度高于老年度同期=提速 → 禁横比不同列 | 同列=同成熟度才可比 | 提速→复核准备金假设；深钻走赔付率发展诊断 |
| 07 漏斗 | 四层收窄 + "最陡收窄处"标注 | 找最陡层 → 算步进转化 → 干预卡在那一步 | 步进转化率=本层/上层 | 有效→优质陡→报价质量治理；优质→承保陡→价格竞争力核查 |
| 08 瀑布 | 首柱保费/中间红柱扣减/尾柱边际 + 柱顶=累计连线 | 看尾柱正负 → 比两根红柱谁长 → 对应率值比阈值 | 赔款=Σ(满期保费×赔付率)；费用=成本−赔款 | 赔款柱长→控赔优先；费用柱长→控费；尾柱负→止损组合 |
| 09 帕累托 | 降序柱 + 累计金线 + 80% 虚线切"关键少数/长尾" | 找 80% 交点 → 左侧=优先治理 → 右侧长尾不平均用力 | 只取承保边际为负者 | 关键少数→专项治理；前 1–2 家占比过半→单点攻坚 |
| 10 树图 | 面积=占比：压舱石大块 / 高风险敞口 / 机会小块 | 看最大块集中度 → 看高赔付险种面积 → 小块找机会 | 占比=险种保费/总保费 | 过度集中→结构调整；高风险敞口大→限额/再保 |
| 11 控制图 | 中线+上下限三线 + 正常波动带 + 破限点放大 | 先看有无破限 → 破限定位周次找事件 → 限内不动作 | 控制限=均值±2σ | 破上限→定点介入；连续单侧漂移→系统性预警 |
| 12 四象限 | 十字阈值线 + 四区四色四动作 | 先看右上 → 再看右下 → 按象限下发动作 | y=65%；x=当前筛选下增速均值（动态） | 右下加码/右上整改/左下复制/左上暂停 |

## 6. 验收要点

- ⓘ 在 12 张卡右上角均出现，不遮挡图表内容与 tooltip；
- 打开/ESC/×/背板三路关闭；打开时页面不滚动；
- 明暗双模式可读；SVG 数据色仅 LEDGER 五色系；
- `bun run typecheck` / `build` / `governance` / 单测全绿；零新增网络请求；
- 内容不复述业务口径之外的因果assertion（领域红线）。
