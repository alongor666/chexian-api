## 1. 我们是谁 · 谁会用这个

**项目**：chexian-api — 一个为**省级保险分公司的经营管理者**用的车险数据分析看板。技术栈：React 19 + TypeScript + Vite + Tailwind 3.4（`class` darkMode 策略） + ECharts 5 + React Router 7（Hash 路由）。

**目标页**：业绩分析（`/#/performance-analysis`）

**典型用户场景**：分公司经理早晨打开这一页，**一屏内要看懂**：
- 各三级机构、各业务员当前业绩进度（保费/件数）
- 谁达成、谁掉队、谁同/环比异常
- 异常背后是哪类业务、哪类客户

用户**每天都要在这里花 15-30 分钟**做下钻、排序、对比。这是一个**高频专业看板**，不是低频展示页。

## 2. 视觉调性（必须遵守，红线）

这是一个 **数据密集型经营看板**，不是 SaaS 营销页。

### 三个核心原则

| 原则 | 含义 |
|------|------|
| **数字即主角** | KPI 数字是焦点，标题/装饰是配角。KPI 用 30px Avenir Next，区块标题仅 14-18px |
| **颜色即语义** | 每种颜色只能传达业务含义，**禁止装饰性用色**。蓝=主操作/链接，绿=增长/达标，红=下降/超标，橙=接近阈值 |
| **密度即效率** | 信息密度优先于留白。表格行高紧凑（py-2），卡片内边距 p-3/p-4，**禁止网格线** |

### 整体氛围
**克制的专业感**——白底/深色 surface 浮着轻盈卡片，柔和阴影暗示层级但不喧宾夺主。大面积留白，颜色**只在传递语义时出现**。安静底色 + 精准语义色，让用户视线自然聚焦异常数据。

### 一定不要做（红线）
- ❌ 渐变背景、阴影装饰、彩色描边
- ❌ 大圆角（除徽章外，最大 `rounded-lg` = 8px；卡片用 8px，徽章用 full）
- ❌ 弹跳/闪烁/旋转动画（只能有 hover/transition 反馈）
- ❌ 图表网格线（`splitLine: { show: false }` 必须）
- ❌ 图表全量数据标签（必须智能筛选：极值/异常/均值/最新值/时间锚点）
- ❌ AI 默认审美陷阱：紫渐变、玻璃拟态、卡片悬浮过度
- ❌ 给数字加 emoji 装饰、给区块加图标徽章
- ❌ 大字标语式排版（这不是营销页）

## 3. 设计系统（必须基于此，不可发明）

> 你正在为一个**有完整设计系统的成熟项目**做视觉迭代，**不是从零起步**。所有 token 必须沿用，禁止发明色彩/字号/间距语言。

### 3.1 色板（CSS 变量驱动，自动 light/dark）

**5 主语义色**（每色含 DEFAULT/light/dark/bg/border/solid 6 变体）：
- `--c-primary` `#1890ff`（蓝）— 主操作、链接、强调、信息
- `--c-success` `#52c41a`（绿）— 增长、达标、正面
- `--c-warning` `#faad14`（橙）— 警示、接近阈值
- `--c-danger` `#ff4d4f`（红）— 下降、超标、负面
- `--c-neutral-500` `#8c8c8c`（灰）— 次要文本、辅助

**5 扩展色**：`--c-purple` `#722ed1`（成本）、`--c-indigo` `#4f46e5`（AI）、`--c-sky` `#0284c7`（信息）、`--c-orange` `#ea580c`（费用）、`--c-amber` `#d97706`（次警告）。

**中性色阶**：neutral-50 `#fafafa` → neutral-900 `#1f1f1f` 9 级。**静态值，dark mode 必须手动 `dark:` 前缀**。

**Dark mode 表面层级**：`--surface-0` 页面底（#0f0f10）→ `--surface-1` 卡片（#161618）→ `--surface-2` 嵌套/表头（#1c1c1f）→ `--surface-3` hover/交替行（#232326）。**深色模式无阴影**，靠 surface 明度递进 + 半透明边框传达层级。

**Dark mode 双轨机制**：
- 语义色（primary/success/warning/danger/purple 等）走 CSS 变量自动切换，**不写 `dark:` 前缀**
- 中性色（neutral 50-900）静态值，**必须手动 `dark:` 前缀**（如 `text-neutral-900 dark:text-neutral-100`）

### 3.2 字体（3 套必记）

- `font-sans`（默认）— `-apple-system, "SF Pro Text", "PingFang SC", "Microsoft YaHei"`，所有正文/标题/标签
- `font-kpi` — `"Avenir Next", "Century Gothic", "SF Pro Display"`，仅 KPI 大数字（30px / 24px）
- `font-numeric` — `"SF Pro Text", "Helvetica Neue"` + `tabular-nums`，**所有表格数字 + 图表标签**（必须等宽对齐）

### 3.3 字号阶梯（不可发明新尺寸）
xs 12px / sm 14px（**正文默认**） / base 16px / lg 18px / xl 20px / 2xl 24px（KPI 次级数字 / 大标题） / 3xl 30px（KPI 主数字）/ 4xl 36px（极少用）

### 3.4 间距（4px 基准）
xs 4px / sm 8px / md 16px / lg 24px / xl 32px / 2xl 48px。**模块间垂直堆叠用 `space-y-4`（16px）**，卡片标题与内容 `mb-4`，紧凑按钮组 `gap-2`。

### 3.5 圆角
`rounded-md` 6px（输入框/小按钮）/ `rounded-lg` 8px（**卡片/按钮 = 最常用**）/ `rounded-xl` 12px（大容器）/ `rounded-full`（徽章/药丸）

### 3.6 阴影（深色模式全部消失）
`shadow-sm` `0 1px 2px rgba(0,0,0,0.05)` — 默认卡片，几乎看不到但能感知。**禁止使用 shadow-lg/xl 来强调层级**——用边框或 surface 递进。

## 4. 当前页的 5 个区块（必须保留 + 优化视觉密度）

按 sticky anchor 顺序：

### 区块 1 · 热力图（performance-heatmap）
- **顶部控制条**：粒度 Tab（日/周/月/季/年）+ 增长口径 Tab（环比/同比）
- **维度 Tab**：三级机构 / 团队 / 业务员 / 客户类别 / 险别 / 能源 / 新续转 / 风险评分（8 个）
- **面包屑**：全部 > 已下钻的路径段（可点击回溯）
- **主体**：行=维度值（如某三级机构），列=最近 15 个时间段，单元格背景色按 `getHeatmapColor(rate)` 6 档色阶映射
- **侧卡片 HeatmapFocusPanel**：单元格点击后右侧抽出展示该单元格详情

### 区块 2 · 业绩概览（performance-summary）
- **展开维度 Tab**：不展开 / 油电 / 新转续 / 油电+新转续
- **表格**：行=险别组合（主全/交三/纯交/单交），列=保费(万元) / 计划 / 件数 / 达成率(%) / 增长率(%) / 新能源占比 / 续保率 / 过户率…
- **可展开行**：左侧 ▸/▾ 箭头展开子行（细分维度），子行背景 `bg-neutral-50/40`

### 区块 3 · 趋势分析（performance-trend）
- 两张 ECharts 折线图：**保费走势** + **件数走势**
- 按 `timePeriod` 粒度绘制
- 多年对照时年份色用 `getYearChartColor(year)`（2026 = `#3B82F6` 亮蓝）

### 区块 4 · 下钻分析（performance-drilldown）
- **维度选择器**（弹层）+ "重置分析" 按钮（仅在已下钻时浮现）
- **四象限分布图**：X=达成率（%）/ Y=增长率（%）/ 气泡=件数 / 颜色=象限（优秀/异常/预警/危险）
- **下钻表格**：列头可排序（升降序切换），每行维度值右侧有 `▼` 下钻菜单（DrilldownCell）

### 区块 5 · Top20 业务员（performance-top20）
- 与区块 4 同构表格，默认达成率升序（最差排最前）

## 6. 数值与排版铁律（车险特定，不可妥协）

| 指标 | 小数位 | 单位位置 | 示例 |
|------|--------|----------|------|
| 率值（赔付率/达成率/增长率） | **1 位** | `(%)` 在**列头**，单元格只放数字 | 列头 `达成率(%)`，值 `85.0` |
| 自主定价系数 | **4 位** | 无单位 | `0.8523` |
| 绝对值（保费/赔款/费用） | **整数** | `(万元)` 在**列头** | 列头 `保费(万元)`，值 `1,256` |
| 案均赔款、件均保费 | **1 位** | 元 | `8,432.5` |
| 件数 | **整数** | 件 | `1,234` |

**金额必须千分位**（`1,256` 而非 `1256`）。**空值统一显示 `-`**。**数字单元格右对齐 + `font-numeric tabular-nums`**（等宽）。

**表格排序铁律**：
- 主题指标**从差到好**（赔付率分析 → 降序；达成率 → 升序最差排最前）
- 无好坏标准 → 升序（保费规模）
- 文字叙述 → 表现最差排最前

## 8. 我们希望你做什么 · 不做什么

### 请你做
- **重做视觉表达**：信息层级、留白节奏、卡片密度、表格可读性、状态/趋势的视觉信号强度
- **提供 2-3 个备选方向**：例如"经典三栏 vs 模块化拼接 vs 顶部 KPI 概览 + 下方 tab 切换"
- **优化热力图**：当前 8 维度 Tab 横排 + 单元格点击侧卡，能否更紧凑/更优雅？
- **优化下钻反馈**：当前下钻深度只有面包屑提示，能否在视觉上更强地暗示"已下钻 N 层"？
- **保留全部 5 区块 + 30+ 交互**：任何视觉方案都必须能放下所有现有功能
- **导出格式**：standalone HTML（self-contained Tailwind / 纯 CSS），让 AI 能 1:1 落地到 React 组件

### 请你不要做
- ❌ **不要改数据维度或交互逻辑**：例如不要把"按险别组合分行"改成"按客户类别分行"
- ❌ **不要发明新色彩**：所有颜色必须能映射到上面的 5 主语义色 + 5 扩展色 + neutral 色阶
- ❌ **不要发明新字号**：必须落在 xs/sm/base/lg/xl/2xl/3xl 8 档
- ❌ **不要加任何渐变/玻璃拟态/彩色阴影**
- ❌ **不要去掉 dark mode**：必须给出 light + dark 两套渲染
- ❌ **不要给率值数字加 `%` 后缀**（`%` 只在列头出现）
- ❌ **不要堆 emoji**（仅 🟢🔵🟡🔴 四个亮灯保留）
- ❌ **不要让 KPI 数字小于 24px**（违反"数字即主角"原则）
- ❌ **不要去掉表格的 sticky 表头/冻结首列**（数据表至少 15 列 + 20+ 行）
