# 图表账本（/chart-ledger）重设计简报（用于 Claude Design）
> 用法：claude.ai/design 新建 → 上传 current-page.html 作起点 → 整段粘贴本文 → 导出 standalone HTML 存回本文件夹。

## 0. 美学方向定调（先读，避免跑偏）
这是一个**编辑型数据方法论长页**（半本"方法论杂志" × 半个"数据终端"）：12 类保险经营图表按业务链路组织，每张图由真实数据驱动、配"结论先行 + 怎么看 + 经营动作"。它的"高级感"来自**阅读节奏、图文对照的信息层级、结论句的视觉权重、状态可辨**，而非装饰。
> ⚠️ 给 Claude Design：**压制你"氛围感/视觉冲击"的默认本能**。本页严禁渐变网格/噪点纹理/戏剧化阴影/玻璃拟态/不对称重叠布局/彩虹色。唯一保留的本能 = 克制自信的排版（编辑级字号阶梯、数字等宽）+ 一种主色 + 锐利语义强调 + 发丝分隔。

## 1. 全局设计法则（取舍标尺）
1. **每卡 3 秒读懂结论**：每张图卡片的"结论先行"句是最高优先级信息，视觉权重必须 > 图表本身的装饰细节。
2. **每个数字要有参照系**：图内阈值线（65% 赔付率、UCL/CL/LCL、均值+2σ）是这套方法论的灵魂，任何视觉方案必须让参照线比现在更醒目，不能弱化。
3. **状态预注意编码**：动作标签（加码/复制=绿、优化=橙、整改/预警=红、暂停=灰）目前只有一个 1.5px 小圆点——请加强（色点必须叠形状/文字，色盲安全），让人扫一眼就知道哪几张图要"整改"。
4. **Tufte 数据墨水**：删非信息像素——图表无网格线（或极淡）、图例改线端直接标注、精确值进 tooltip。
5. **7±2 分块**：12 张卡按 5 阶段分组已符合分块，保持；可强化"阶段间"的视觉节奏差（例如阶段编号/标题的排版层级），但不要把卡片折叠掉。
6. **8px 栅格 + 段落感**：eyebrow（眉标）+ 留白 + 发丝分隔线做"章节段落感"；数字右对齐、tabular-nums。
7. **长页导航感**：这是一次 10+ 屏的长滚动阅读，当前 sticky 锚点导航没有"当前所在阶段"高亮——请设计 active 态（scrollspy 视觉），让读者始终知道自己在链路的哪一环。

## 一句话目标
为**省级保险分公司的经营管理者与业务骨干**重做"图表账本"页。这是**方法论参考页 + 真实数据看板的混合体**（不是营销页）。核心诉求：把"12 类图表怎么看、看完做什么动作"讲得比现在更清楚、更有阅读节奏；每张卡 3 秒内读懂结论。保留全部数据维度与交互前提下，提升信息层级、可读性与品质。

## 目标受众与核心用例
- 受众：分公司经营管理者、三级机构负责人、业务分析骨干；桌面宽屏为主（≥1280px），偶有笔记本；light + dark 双模式。
- 核心用例：
  1) **教学/对齐**：管理者用它给团队讲"我们怎么看图、图后面接什么动作"（自上而下通读）；
  2) **速查**：分析骨干直接锚点跳到某张图，看当前筛选下的真实结论（点导航→读结论句→看图）；
  3) **周会引用**：切换全局筛选（机构/时间窗）后引用某张卡的结论与要点。

## 当前状态与痛点
- 现状：见所上传 current-page.html（忠实还原当前页；顶部灰条是应用壳占位，重做范围外）。
- 痛点：
  1) 12 张卡片完全同构，长页扫读时**阶段之间缺乏视觉节奏**，容易迷失（导航也无 active 态）；
  2) 左栏叙述堆叠密（眉标/标题/结论/怎么看/要点/动作标签六层），**结论句的视觉权重不够**，橙色左边线太弱；
  3) 动作标签（加码/优化/整改/预警）**语义强度太弱**（1.5px 圆点），无法预注意扫到"哪几张图在报警"；
  4) 图表右栏统一 260px 高、卡片样式统一，**图内参照线（65%/UCL/±2σ）不够跳**，方法论的"阈值思维"没有被视觉放大；
  5) Hero 与 §0 框架区平铺直叙，作为"账本封面"缺少定场感（但不要做成营销 Hero）。

## 必须保留的功能区块与交互（红线级）
> 这是 Phase C 验收的硬基线。视觉可以变，结构与交互必须 1:1 保留：

1. **sticky 锚点导航**：6 项（§0 框架 / 01 渠道 / 02 承保 / 03 理赔 / 04 续保 / 05 财务），点击平滑滚到锚点（各 section `scroll-mt`），窄屏横向滚动；
2. **Hero 区**：eyebrow + 大标题（"业务动作 vs 图表形态"论点）+ 导语段（含链路五环节强调）；
3. **§0 三层框架**：目标层/链路层/动作层 3 列卡（内容一字不改）；
4. **5 个阶段 section**：编号 + 标题 + 导语（tagline），阶段内卡片顺序不变（01→1 卡、02→2 卡、03→3 卡、04→1 卡、05→5 卡）；
5. **12 张图卡片**，每张含全部 7 个信息元素：眉标（Chart NN · 分类）/ 标题 / **结论先行句（数据派生，随筛选变化）** / "怎么看"说明 / "真实数据结论"要点列表（2-4 条，数据派生）/ 经营动作标签（动作名+旁注）/ 图表 + 数据口径脚注；
6. **图表形态不可换**：01 气泡矩阵、02 异常散点（离群=菱形）、03 热力表格、04 箱线图、05 折线趋势、06 发展三角表格、07 漏斗、08 瀑布、09 帕累托（双轴柱+线）、10 树图、11 控制图（UCL/CL/LCL 三线）、12 四象限散点——形态是方法论本体，只能优化表达；
7. **图内参照系必须保留**：01/12 的 65% 赔付率虚线、02 的均值+2σ 离群着色、11 的三条控制线、12 的双阈值十字、09 的 80% 累计线语义；
8. **每张图的 loading / error / empty 三态**（当前为居中文案"载入中…/数据加载失败/当前筛选下无数据"）——新设计要给出这三态的视觉方案；
9. **全局筛选联动**：页面数据随顶部全局筛选（机构/日期等）重查询——设计上保留"数据来自真实查询（随全局筛选联动）"的提示语义；
10. **ECharts tooltip**（01/02/05/08/09/11/12 悬停出详情）——静态稿标注 hover 态即可；
11. **03/06 表格横向滚动**（窄屏 overflow-x-auto）+ 单元格 teal→coral 赔付率梯度着色（50%→95%）；
12. **收尾"最强反方观点"卡**：标题 + 段落 + 五个失败条件有序列表 + 底部口号行（内容一字不改）；
13. **卡片锚点 id**（chart-01…chart-12 可直链）；
14. **暗色模式**：全页 light/dark 双套（项目 `class="dark"` 策略）；
15. **响应式**：lg 以下卡片左右栏塌成单栏、导航横滚。

## 逐维度设计方向
- 排版：编辑感——中文标题用现有 font-sans（苹方/雅黑栈）加重字重与字距层级即可，**不要引入新字体**；数字一律 font-numeric（tabular-nums）。可以强化 eyebrow/编号的"账本感"（如 01-12 大号编号做视觉锚点），字号必须落在项目 8 档阶梯内。
- 配色：底色安静（白/浅灰 + dark surface 递进），语义色只在传递语义时出现。**图表账本专属数据色板保留**：teal `#13C2C2`（正常/主数据）、gold `#E8B339`（参照/中位/累计线）、coral `#F5615C`（异常/高赔付）、good `#52C41A`、muted `#8C8C8C`——这是 12 张图的既有视觉语言，请沿用并可以系统化（例如把 teal 升格为本页图表主色调性），但不得引入第六种数据色。
- 动效：克制——仅 hover/过渡（0.15-0.3s）；scrollspy 高亮切换可有淡入；禁弹跳/闪烁/视差。
- 背景/质感：白底浮轻卡（shadow-sm 级），dark 模式无阴影靠 surface 明度递进；分隔靠发丝线与留白，不靠色块。

## 设计系统约束（产出要能落地，来自本项目 src/shared/styles/index.ts + DESIGN.md）
- 语义色（CSS 变量，自动 light/dark）：primary `#1890ff`（链接/强调）、success `#52c41a`、warning `#faad14`、danger `#ff4d4f`、neutral-500 `#8c8c8c`；每色含 DEFAULT/light/dark/bg/border 变体。**页面 UI 层（非图表）只准用这套语义色**；图表数据层用上文 LEDGER 五色。
- 中性色阶 neutral-50→900 静态值，dark 需手动 `dark:` 前缀；dark 表面层级 surface-0 `#0f0f10` → surface-1 卡片 `#161618` → surface-2 `#1c1c1f` → surface-3 `#232326`。
- 字体栈：`font-sans`（正文/标题）、`font-kpi`（仅 KPI 大数字）、`font-numeric` + tabular-nums（所有表格数字/图表标签/编号）。
- 字号 8 档：12/14/16/18/20/24/30/36px，不可发明新尺寸（现页有 11px/13px/15px 微调属历史遗留，新稿请归档到 12/14/16）。
- 组件预设：卡片=白底 rounded-lg(8px) + border neutral-200 + shadow-sm（dark: surface-1 无阴影）；徽章=rounded-full 小号；间距 4px 基准，卡间 space-y-4/gap-3。
- 数值格式：**率值 1 位小数**；本页叙述文案与表格单元格中百分比**带 %**（如 `92.6%`——本页表格列头未标单位，与项目"列头标 (%)"的表格惯例不同，保持现状）；金额万元取整千分位（`38,420万`）；件数整数千分位；系数 4 位；空值 `—`。
- 排序既有语义保留：热力图行按行最大赔付率降序取前 8、帕累托按亏损降序、箱线按极差降序取前 5、树图按占比降序。
- 暗色模式：必须；语义色自动、中性色手动 dark: 前缀、图表色板五色在暗底上已验证可用。

## 后端字段契约速查表（数据驱动页强制 — 防"假设字段总是有值"踩坑）
> 每个展示字段：单位、是否可能 NULL、真实数据源。落地 AI 会假设字段非空、单位匹配——契约缺失=必踩坑。

| 字段 | 单位 | 可能 NULL/NaN? | 真实数据源 | 注意 |
|------|------|-----------|----------|------|
| total_premium | **元** | 是 | `/api/query/pivot`（metric-registry 原子指标） | hook 内 `wan()` ÷1e4 转万元后再进图；展示：元→`formatPremiumWan`，已万元→`formatWanDirect`，二者差 1 万倍，混用即归零 |
| earned_claim_ratio | **已是百分比**（65 = 65%） | 是 | 同上 | 禁再 ×100；**禁用 `formatChartValue`**（内部 ÷1e4，PR #880 事故：百分比全显示 0） |
| expense_ratio | 已是百分比 | 是 | 同上 | 同上 |
| earned_margin_amount | 元（可为负） | 是 | 同上 | 帕累托取负值的绝对值；瀑布：成本 = 满期保费 − 边际 |
| earned_premium | 元 | 是 | 同上 | 瀑布首柱 |
| policy_count | 件（可能 BigInt） | 是 | 同上 | `num()` 兜 bigint；formatCount 千分位 |

## 要避免的默认（黑名单 — 逐条点名模型已知默认）
- ❌渐变网格/噪点/颗粒/任意渐变背景（漏斗现有的 teal 单色渐变条属数据编码，可保留或改纯色）
- ❌玻璃拟态/戏剧化大阴影/装饰几何/自定义光标 ❌不对称·重叠·对角线布局（本页要栅格对齐）
- ❌彩虹色阶（热力/三角用现有 teal→coral 单调双色梯度） ❌通用字体(Arial/Inter/Roboto)做数字
- ❌把方法论页做成留白过度的营销落地页 ❌注意力消耗型动画 ❌给数字/标题加装饰 emoji（本项目仅 🟢🔵🟡🔴 四级亮灯可用 emoji，本页当前未用）
- ❌新增第六种图表数据色 ❌把 12 张卡折叠进 Tab/手风琴（折叠=破坏通读用例）

## 红线（不要做）
- ❌ 删任何数据维度、任何一张图、卡片的任何信息元素（眉标/结论/怎么看/要点/动作/脚注一个都不能少）
- ❌ 改叙述文案与业务口径（结论句/要点是数据派生模板，样式可改、句式结构别改）
- ❌ 用设计系统之外的色彩语言；率值多于 1 位小数；金额丢千分位
- ❌ 去掉 dark mode；去掉锚点导航或改成非 sticky
- ❌ 改动应用壳（侧边栏/全局筛选条）——不在本次范围
