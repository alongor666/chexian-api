<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| permission.ts -> snapshot-serve.ts | permissionFilter 由服务端 permission 中间件生成，非用户直接输入；但 scope 值用于构建文件路径 |
| snapshot-serve.ts -> 文件系统 | scope 值拼接进文件路径，潜在目录遍历风险 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-1-01 | Information Disclosure | permissionToScope → resolveSnapshotPath | mitigate | 返回 null + 提前 next()，未知权限永远不触碰文件系统（本 plan 核心修复） |
| T-1-02 | Tampering | permissionFilter 注入 `../` 路径 | accept | permissionFilter 由 permission.ts 服务端生成（L50-L68），不接受用户输入；org 名来自 regex 匹配 `'(.+?)'`，单引号限定范围不含路径分隔符 |
| T-1-03 | Information Disclosure | 快照文件权限 | accept | 快照文件在 server 进程内读取，不通过 static file serving 暴露；文件权限由 OS 控制 |
</threat_model>
