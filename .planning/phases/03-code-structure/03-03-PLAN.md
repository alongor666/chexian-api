    **Step 1: 创建 `src/shared/contexts/StableContext.tsx`**

    StableContext 负责:
    - `filterOptions: FilterOptions` — 从 API 加载一次
    - `salesmanTeamMap: Map<string, string>` — 从 API 加载一次
    - `maxDataDate?: string` — 从 API 加载一次
    - `availableYears?: number[]` — 从 API 加载一次
    - `isLoading: boolean` — 仅初始化期间变更
    - `initializeFilters: () => Promise<void>` — 稳定回调
    - `dualDateMetadata` 内部状态（不暴露，通过 maxDataDate/availableYears 派生）

    ```typescript
    import React, { createContext, useContext, useState, useCallback, useEffect, ReactNode } from 'react';
    import { createLogger } from '../utils/logger';
    import type { FilterOptions, DateMetadata, DualDateMetadata } from '../types/data';
    import { getMetadataByCriteria } from '../types/data';
    import { formatSalesmanName } from '../utils/formatters';
    import { useDataStatus } from './DataContext';
    import { apiClient } from '../api/client';

    const logger = createLogger('StableContext');

    export interface StableContextValue {
      filterOptions: FilterOptions;
      salesmanTeamMap: Map<string, string>;
      maxDataDate?: string;
      availableYears?: number[];
      isLoading: boolean;
      initializeFilters: () => Promise<void>;
      // 内部供 FilterProvider 使用
      _internal: {
        dualDateMetadata?: DualDateMetadata;
        isInitialized: boolean;
        latestInitResult: { maxDate: string; dataYear: number } | null;
      };
    }

    const StableContext = createContext<StableContextValue | null>(null);

    export const useStableContext = (): StableContextValue => {
      const context = useContext(StableContext);
      if (!context) {
        throw new Error('useStableContext must be used within a StableProvider');
      }
      return context;
    };

    interface StableProviderProps {
      children: ReactNode;
    }

    export const StableProvider: React.FC<StableProviderProps> = ({ children }) => {
      // ... 将 FilterContext.tsx 中的 filterOptions/salesmanTeamMap/dualDateMetadata/isLoading/isInitialized 状态
      // 和 loadFilterOptions/initializeFilters/getCurrentMetadata 逻辑移入此处
      // maxDataDate 和 availableYears 通过 getCurrentMetadata 派生（但此处无法访问 filters.date_criteria）
      // 解决方案：getCurrentMetadata 默认使用 'policy_date'（初始值），后续由 FilterProvider 通过 prop 或 effect 同步
      // 实际观察：getCurrentMetadata 使用 filters.date_criteria，但在初始化阶段 date_criteria 总是 'policy_date'
      // 简化方案：StableContext 中 maxDataDate 直接取 dualDateMetadata.policy.maxDate（因为 date_criteria 默认 policy_date）
      // availableYears 同理取 dualDateMetadata.policy.availableYears
    };
    ```

    **注意：** `dualDateMetadata` 上的 `getCurrentMetadata()` 依赖 `filters.date_criteria`（一个易变状态）。但分析 FilterContext.tsx 第 70-77 行：
    - `getCurrentMetadata` 仅在计算 `maxDataDate` 和 `availableYears` 时使用
    - `date_criteria` 切换场景极少（两个口径之一：policy_date 或 insurance_start_date）
    - 两个口径的 metadata 在当前实现中完全相同（第132-135行：`policy` 和 `insurance` 的 maxDate/availableYears 一模一样）

    因此 StableContext 直接用 `dualDateMetadata.policy.maxDate` 和 `dualDateMetadata.policy.availableYears` 即可，无需依赖 `filters.date_criteria`。

    **Step 4: initializeFilters 中的 setFilters 问题**

    当前 `initializeFilters` 在 FilterContext 中调用 `setFilters`（设置初始日期范围）。拆分后：
    - `initializeFilters` 在 StableProvider 中，无法直接调用 FilterProvider 的 `setFilters`
    - 解决方案：StableProvider 暴露 `_internal.latestInitResult`，FilterProvider 监听此值变化并同步 setFilters

    ```typescript
    // In FilterProvider:
    const { _internal } = useStableContext();

    useEffect(() => {
      if (_internal.latestInitResult && !isFilterSynced) {
        const { maxDate, dataYear } = _internal.latestInitResult;
        setFilters(prev => ({
          ...prev,
          date_criteria: 'policy_date',
          analysis_year: dataYear,
          policy_date_start: `${dataYear}-01-01`,
          policy_date_end: maxDate,
        }));
        setIsFilterSynced(true);
      }
    }, [_internal.latestInitResult]);
    ```
